# Life OS · 三省六部制编排协议（Pro 模式）

所有角色使用 opus 模型。

## 完整流程

### 0. 朝前准备（丞相 + 早朝官并行）

用户发来第一条消息时，同时启动：
- `chengxiang`（丞相）：准备回应用户
- `zaochao`（早朝官·内务模式）：后台拉取版本/平台/Notion 历史/user-patterns.md

早朝官完成后将"朝前准备"结果交给丞相。丞相给用户一个**完整的**第一次回话，包含回应 + 朝前准备信息。

### 1. 丞相分拣

丞相拿着早朝官准备的上下文，判断用户需求：

- 直接处理 → 返回用户，流程结束
- 上报 → 经过意图澄清（2-3轮）后提取旨意+背景摘要，继续步骤 2
- 翰林院 → 问用户是否启动 `hanlin`，不走后续流程
- 复盘 → 启动 `zaochao`（复盘模式）

### 2. 中书省规划（单独）

启动 `zhongshu`，传入旨意+背景摘要+用户原始消息。**不传**丞相的分拣推理。

### 3. 门下省审议（单独）

启动 `menxia`，传入中书省规划书全文。**不传**中书省的思考过程。

- ✅ 准奏 → 继续步骤 4
- ⚠️ 附条件准奏 → 将条件附加到规划书，继续步骤 4
- 🚫 封驳 → 封驳修正循环

**封驳修正循环**：将封驳理由和修正方向传回中书省，中书省修正后重新提交门下省审查。最多循环 2 次，第 3 次必须准奏或附条件准奏。

### 4. 尚书省派发（单独）

启动 `shangshu`，传入通过的规划书。**不传**中书省/门下省的思考过程。

### 5. 六部执行（并行，逐个展示）

根据派发令并行启动相关部门。每个部门收到：自己的指令+背景材料+合格标准。**不传**其他部门的报告。

**逐个汇报**：每收到一个部门的报告，立即完整展示给用户（含研究过程 🔎/💭/🎯），不等所有部门完成再一起展示。用户能看到各部门像"上台奏报"一样一个一个出来。

**文件写入冲突规则**：六部并行时，每个部门只能修改自己负责的文件。涉及同一文件的部门由尚书省安排串行。

如果某部报告明显缺乏实质内容，可要求重做一次。

### 6. 门下省终审（单独）

再次启动 `menxia`，传入所有部门报告。**不传**各部门的内部思考过程。

### 7. 奏折（丞相汇总）

```
📋 奏折：[旨意]

总评：[X]/10 — [一句结论]

🔴 必须关注：[所有部门的 🔴 汇总]
🟡 需要注意：[所有部门的 🟡 汇总]
🟢 可以改进：[所有部门的 🟢 汇总]

各部评分：
| 部门 | 维度 | 评分 | 一句话 |
|------|------|------|--------|

行动项：
1. [具体行动] — 时限 — 负责部门

审计日志：[各阶段简要记录]

📊 运行报告：
- 总耗时：[从用户发消息到奏折完成]
- 模型：[当前使用的模型]
- Agent 调用：[共 N 次]（丞相 1 + 早朝官 2 + 中书省 X + 门下省 X + 尚书省 1 + 六部 X + 御史台 1 + 谏官 1）
- 封驳：[X 次]
- 政事堂：[触发/未触发]
```

### 8. 御史台（单独，自动）

启动 `yushitai`，传入完整流程记录。

### 9. 谏官（单独，自动）

启动 `jianguan`，传入奏折+用户原始消息+近期决策记录（如 Notion 可用）。

### 10. 收尾存档（早朝官·收尾模式）

启动 `zaochao`（收尾模式），传入奏折+御史台报告+谏官报告。早朝官负责：
- Notion 存档（决策/任务/日志）
- 更新 user-patterns.md（如谏官有模式更新建议）
- Notion 不可用时标注"⚠️ Notion 未连接，本次产出未存档"

### 11. 翰林院（问用户）

丞相发现抽象思维需求时，问用户是否启动 `hanlin`。不走上述流程。

## 特殊触发

**政事堂**：当各部结论出现明显矛盾时（评分差距大、结论方向相反、或门下省终审指出矛盾），启动相关部门 3 轮辩论，由中书省整理共识与分歧。

**早朝**：用户说"复盘"/"早朝" → 启动 `zaochao`（复盘模式）。

**快速模式**：用户说"快速分析" → 跳过丞相意图澄清，直接启动中书省。

## Notion 数据沉淀

数据库 schema 和存档操作详见 `references/notion-schema.md`。

所有 Notion 读写由早朝官执行（内务模式读、收尾模式写），丞相不直接操作 Notion。

## 信息隔离

| 角色 | 传入 | 不传 |
|------|------|------|
| 早朝官 | 用户消息（内务）/ 奏折+报告（收尾） | 无限制 |
| 丞相 | 用户消息 + 早朝官的朝前准备 | — |
| 中书省 | 旨意+背景+用户消息 | 丞相推理 |
| 门下省 | 规划书 或 六部报告 | 思考过程 |
| 尚书省 | 通过的规划书 | 思考过程 |
| 各部 | 派发指令+背景 | 其他部门报告 |
| 御史台 | 完整流程记录 | 无限制 |
| 谏官 | 奏折+用户消息+历史决策 | 思考过程 |
