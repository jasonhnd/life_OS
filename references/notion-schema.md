# Notion 数据库 Schema

各 agent 读写 Notion 时参考本文件。使用 Notion MCP 工具操作。

## 核心数据库

### 🤔 决策

存储三省六部流程的决策记录和奏折。

| 字段 | 类型 | 说明 |
|------|------|------|
| 标题 | Title | 旨意（≤20字） |
| 流程类型 | Select | 简单决策 / 三省六部 |
| 启用部门 | Multi-select | 吏/户/礼/兵/刑/工 |
| 综合评分 | Number | X/10 |
| 封驳次数 | Number | 门下省封驳了几次 |
| 状态 | Status | 待执行 / 执行中 / 已完成 / 已搁置 |
| 日期 | Date | 决策日期 |
| Actions | Relation → 任务 | 关联产出的行动项 |

页面正文：奏折全文

### ✅ 任务

存储各部产出的行动项。

| 字段 | 类型 | 说明 |
|------|------|------|
| 标题 | Title | 任务名称 |
| 状态 | Status | 待办 / 进行中 / 已完成 |
| 优先级 | Select | P0 / P1 / P2 |
| 截止日期 | Date | 时限 |
| 负责部门 | Select | 吏/户/礼/兵/刑/工 |
| 来源决策 | Relation → 决策 | 关联产出此任务的决策 |
| Area | Relation → 六部 | 关联到六部 Area |

### 📓 日志

存储复盘简报、御史台报告、谏官报告。

| 字段 | 类型 | 说明 |
|------|------|------|
| 标题 | Title | 日志标题 |
| 类型 | Select | 早朝简报 / 御史台报告 / 谏官进谏 / 手动记录 |
| 日期 | Date | 记录日期 |
| 关联决策 | Relation → 决策 | 如果是某次流程的产出 |

页面正文：报告全文

### 🎯 目标

| 字段 | 类型 | 说明 |
|------|------|------|
| 标题 | Title | 目标名称 |
| 状态 | Status | 规划中 / 进行中 / 已达成 / 已放弃 |
| 周期 | Select | 周 / 月 / 季 / 年 |
| 截止日期 | Date | 目标期限 |
| Area | Relation → 六部 | 关联到六部 Area |
| 进度 | Number | 0-100% |

### 🏛️ 六部（Areas）

六个固定条目，作为其他数据库的关联锚点。

| 条目 | 说明 |
|------|------|
| 👥 吏部 · 人 | 人际关系、团队、社交 |
| 💰 户部 · 钱 | 财务、投资、预算 |
| 📖 礼部 · 学习与表达 | 教育、品牌、创作 |
| ⚔️ 兵部 · 行动 | 项目、执行、调研 |
| ⚖️ 刑部 · 规则 | 风控、合规、自律 |
| 🏗️ 工部 · 基建与健康 | 健康、环境、数字基建 |

## Agent 操作指南

### 丞相：查询历史

对话开始时，根据用户话题搜索相关历史：

```
1. 用 notion-search 搜索决策数据库，关键词=用户话题
2. 如果找到相关决策，读取奏折摘要
3. 在上报时附上历史上下文
```

### 早朝官：拉取复盘数据

```
1. 用 notion-search 查询任务数据库（状态=待办/进行中/已逾期）
2. 查询目标数据库（当前周期）
3. 查询最近的日志条目
4. 汇总为早朝简报
```

### Orchestrator：流程结束存档

```
1. 创建决策页面（标题=旨意，字段填充，正文=奏折）
2. 逐条创建任务（关联到决策和六部 Area）
3. 创建日志页面（御史台报告）
4. 创建日志页面（谏官报告）
```

如果 Notion MCP 不可用，在奏折末尾标注"⚠️ Notion 未连接，本次产出未存档"。
