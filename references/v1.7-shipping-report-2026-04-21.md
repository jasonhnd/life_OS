## 2. Two narratives

### 2a · COURT-START-001 fix (v1.6.3 line)

**The bug**: 2026-04-19 the user said "上朝" in the dev repo. ROUTER skipped the retrospective subagent entirely, simulated 18 steps in main context, and fabricated paths like `_meta/roles/CLAUDE.md § 0 Pre-Court Preparation` as authority. Documentation was complete; enforcement was zero. User reaction: "这样的 life os 如何拿给别人用？？？我无法接受。"

**The fix** (5 layers + 7-class violation taxonomy):
- L1 hook (`scripts/lifeos-pre-prompt-guard.sh`) injects HARD RULE reminder into ROUTER's context
- L2 Pre-flight Compliance Check requires `🌅 Trigger: ... → Action: Launch(...)` line before any tool call
- L3 Subagent Self-Check requires `✅ I am the RETROSPECTIVE subagent ...` first line
- L4 AUDITOR Compliance Patrol Mode 3 audits each session against 7 classes (A1/A2/A3/B/C/D/E) + writes violations to `pro/compliance/violations.md` (dev) or `_meta/compliance/violations.md` (user)
- L5 eval scenario `evals/scenarios/start-session-compliance.md` + `scripts/lifeos-compliance-check.sh` for regression testing

**Three patches** (a/b) closed real follow-up gaps:
- v1.6.3a: `scripts/setup-hooks.sh` now auto-registers UserPromptSubmit hook (was: install copied script but didn't wire). Hook regex tightened (first-line ≤ 100 chars + prompt ≤ 500 chars) to reduce false positives on pasted content. Class F (false positive) added to taxonomy.
- v1.6.3b: AUDITOR Mode 3 spec existed but **nothing called it**. Added Orchestration Code of Conduct rule #7 to pro/CLAUDE.md so orchestrator chains AUDITOR after every retrospective Mode 0 / archiver completion.

**Production verification status (2026-04-21)**: User upgraded use repo to v1.6.3, ran `上朝`, and L2 + L3 worked end-to-end (Pre-flight line emitted + Subagent self-check displayed). L4 not yet observed (depends on v1.6.3b orchestrator chain). L1 was inactive in user repo until they run setup-hooks.sh after v1.6.3a install. L5 eval not yet run with `claude -p`.

`pro/compliance/violations.md` has 6 entries: 4 COURT-START-001 (P0/P1) marked `partial (v1.6.3 shipped + L2+L3 production-verified 2026-04-21 + awaiting eval regression + 30d obs)`, 1 misattribution (P1) marked `true`, 1 Class F (P2) for paste-content false positive (mitigated by v1.6.3a).

### 2b · Cortex Pre-Router Cognitive Layer (v1.7 line)

**The why**: Before v1.7, Life OS touched long-term memory only at session boundaries. Between sessions, agents had no shared cognitive substrate. PLANNER planned without knowing how the user decided similar subjects in the past. The 16 agents were strong individually but nobody saw what other sessions learned.

**The architecture** (Step 0.5 Pre-Router):

```
user message
    ↓
Step 0.5 (historical alpha: config-gated; v1.7.2: always-on)
    ├─ hippocampus       → 3-wave session retrieval (5-7 sessions)
    ├─ concept-lookup    → concept-graph match (5-10 concepts)
    └─ soul-check        → SOUL dimension signals (top 5)
         ↓
    gwt-arbitrator        → salience formula → top-5 signals
         ↓
[COGNITIVE CONTEXT] block prepended to user message
    ↓
Step 1 ROUTER (with annotated input)
    ↓
... draft → review → execute ...
    ↓ (between REVIEWER + Summary Report)
narrator → wraps substantive claims with [source:signal_id] citations
narrator-validator (Sonnet) → audits citation discipline
```

**6 new subagents** in `pro/agents/`:
- `hippocampus.md` — cross-session memory retrieval (Wave 1: direct match, Wave 2: strong synapse traversal weight ≥ 3, Wave 3: weak synapse weight ≥ 1, cap 5-7 total)
- `concept-lookup.md` — concept-graph match (read INDEX, LLM judgment, cap 10)
- `soul-check.md` — SOUL dimension signals (tier_1_alignment / tier_1_conflict / tier_2_relevant / dormant_reactivation)
- `gwt-arbitrator.md` — salience formula `urgency*0.3 + novelty*0.2 + relevance*0.3 + importance*0.2`, cap 5 signals, floor 0.3
- `narrator.md` — wrap Summary Report substantive claims with `[hippo:s1, concept:c3, soul:d1]` citations
- `narrator-validator.md` — Sonnet-tier auditor (3 checks: citation resolution, missing citations, misapplication)

All 6 enforce information isolation (peer Pre-Router agent outputs rejected in input). All read-only — mutations happen only in archiver Phase 2.

## 3. Historical Opt-In Posture (superseded in v1.7.2)

Cortex used an **opt-in activation gate** in v1.7.0-alpha.2. Reasons:
1. Production validation pending — alpha is theory-tested but not yet exercised at scale in real user second-brains
2. Cost meaningful — ~$0.05-0.25/turn (Opus tokens across Pre-Router subagents)
3. INDEX.md needs to accumulate before retrieval has value (≥30 sessions baseline)

**Historical enablement command** (no longer an activation gate in v1.7.2):
```bash
echo "cortex_enabled: true" >> _meta/config.md
```

In v1.7.2, user messages flow through Step 0.5 by default. If anything fails
(bootstrap failure, subagent timeout, INDEX missing, GWT error), the system
gracefully degrades to v1.6.3 behaviour (raw message to ROUTER) — Cortex is
additive, never blocks.

## 8. Honest self-assessment

This was a **single-day burst** that took COURT-START-001 from incident to fix to verified, then leapt to a major architectural addition (Cortex). Quality risks:

- **Spec/code drift possible** — wrote 4865 lines of spec, then 6 subagents + Python tools + tests. They should be coherent; pytest covers the Python parts; subagent prompt quality is unverifiable without Claude API.
- **Cortex untested in production** — historical alpha gating was meant to shield users from possible bugs while validation happened.
- **Token cost unmeasured** — estimated $0.05-0.25/turn but real-world usage may differ.
- **i18n consistency**: maintained version badges + CHANGELOG entries trilingually but body content has occasional drift (e.g., some sections only have EN — see README "Still current from v1.6.2" line in zh/ja).

What's solid:
- 173 pytest tests, all green, 0.66s combined
- 5 git tags pushed
- All Python modules typecheck-clean (mypy not run, but ruff lint OK)
- All bash scripts syntax-clean (bash -n)
- Hook + compliance check tested end-to-end with subprocess-based pytest
- Information isolation enforced in Cortex spec + AUDITOR Mode 3 detection
- Historical alpha gating meant users saw zero behaviour change unless they opted in

What's unproven:
- Whether Cortex retrieval actually IMPROVES decisions (vs adding noise)
- Whether the cost is worth it for typical sessions
- Whether prompt cache stays warm with `[COGNITIVE CONTEXT]` injection

These need real-world validation. Step 5 above is the gate.
