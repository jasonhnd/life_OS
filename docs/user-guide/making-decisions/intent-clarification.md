# 意图澄清 · 为什么必须三轮

> 本文档是作者自用参考。写给懂系统内部的人看。

## HARD RULE

**任何复杂请求在升级到 PLANNER 之前，ROUTER 必须进行 2-3 轮意图澄清。不能跳过。**

这条规则写在 `pro/agents/router.md` 的 Intent Clarification 节，并且列为 anti-pattern 第一条：

> Do not skip intent clarification and escalate directly. This is a HARD RULE.

只有一个例外：**Quick Mode**（用户显式说"快速分析"/"quick"），ROUTER 直接走快车道，跳过澄清。其他情况一律不能跳。

## 为什么不可跳

### 用户提问的真正样貌

人类提问有一个普遍特征：**第一句话不是你真正想问的**。

真实例子：
- 用户说："我应该辞职吗？"
- 真正在问的是："我 38 岁、家里有两个娃、存款能撑 14 个月，现在这份工作让我每天下班只想躺着，但跳槽后薪水必然降 30%——我在找一个允许我不辞职的理由，还是在找一个允许我辞职的理由？"

如果 ROUTER 听到第一句就立刻升级到 PLANNER，PLANNER 拿到的 Subject 就是"辞职决策"。六领域基于"辞职决策"去分析——每个领域都给出技术上正确但**完全错过真正问题**的报告。最后奏折给"6/10 中性"，用户看完说"感觉没帮上忙"。

**三轮澄清就是在强制 ROUTER 把"第一句话"剥离到"第十句话"。**

### 情绪和决策交织时的风险

用户在情绪高峰时提的决策问题，**不是决策问题**，是情绪问题披着决策的外衣。

`pro/agents/router.md` 里的 Emotion Separation Protocol：

1. 先承认情绪："听起来你压力很大。"（一句话，不多说）
2. 分离："我们先把焦虑放一边，只看事实——现在的情况具体是怎样的？"
3. **不启动全朝议，直到情绪和事实分离**

为什么这一步必须在 ROUTER 这里做，而不能留给 PLANNER？因为 PLANNER 的岗位不是安抚情绪，是拆维度。如果情绪没剥离就进 PLANNER，PLANNER 给出的维度会把"情绪反应"当"真实偏好"来优化。

## 五种问题类型 · Step 0 分类

ROUTER 在第 1 轮之前先分类：

| 类型 | 信号 | 澄清焦点 |
|------|------|----------|
| 决策（Decision） | "A 还是 B？"、"我该不该做 X？" | 标准是什么？约束是什么？可逆吗？ |
| 规划（Planning） | "我想做 X"、"帮我计划 Y" | 目标是什么？资源？时间线？ |
| 迷茫（Confusion） | "不知道该怎么办"、"我迷失了" | 情绪状态？真正困扰你的是什么？这是决策还是情绪？ |
| 复盘（Review） | "看看进展怎么样"、"复盘一下" | 用什么标准？什么时间段？什么维度？ |
| 信息（Information） | "查一下 X"、"X 是什么？" | 直接处理，不澄清 |

只有前四类进入澄清。第五类（Information）直接处理。

## 三轮澄清的结构

**Round 1 · 复述核心问题**
用一句话把用户的问题重述回去，问："我这样理解对吗？"

作用：暴露 ROUTER 的理解和用户表达的差异。90% 的场景下这一轮就能纠正偏差。

**Round 2 · 针对最大缺口的犀利问题**
基于第 1 步分类，问**一个**关键问题（不是多个！）——命中该类型最常缺失的信息。

- 决策类：标准 / 约束 / 可逆性
- 规划类：资源 / 时间 / 目标可测性
- 迷茫类：情绪 vs 决策分离
- 复盘类：时间段 + 维度

**Round 3 · 确认约束（如果还需要）**
如果前两轮已经清楚，跳过第 3 轮。如果还有不确定，只在这里确认约束，不再开新话题。

## 好的升级 vs 仓促升级

### 仓促升级（反模式）

```
用户：我在考虑要不要换工作。
ROUTER: 🏛️ Petition | Subject: 换工作决策 | 建议领域：全六领域
[直接启动 PLANNER]
```

问题：
- 不知道是"具体某个 offer"还是"泛泛想换"
- 不知道薪水预期
- 不知道家庭情况
- 不知道情绪状态（是工作让人难受还是和老板闹翻了？）
- 不知道"换工作"对用户意味着什么——升职？转行？跳城市？

PLANNER 拿到 Subject 必须自己猜测，猜错了 REVIEWER 会封驳，重来。浪费时间和 token。

### 好的升级

```
用户：我在考虑要不要换工作。

ROUTER:（Round 1 · 复述）
听起来你现在不是在两个具体 offer 之间选，更像是"当前工作让我累，在想要不要动"——这样理解对吗？

用户：是的，但我手上也有一个朋友介绍的机会。

ROUTER:（Round 2 · 犀利问题）
明白。那我想知道一个关键点：你是在找一个"允许你留下"的理由，还是"允许你离开"的理由？直觉上更倾向哪边？

用户：其实……更想走。但怕薪水降太多。

ROUTER:（Round 3 · 确认约束）
好，那最后一个问题：朋友介绍的机会你了解到什么程度？薪水范围、工作内容、风险——这三样你手上有几个？

用户：薪水大概知道，会降 20%；内容差不多但换了个行业；风险不知道。

ROUTER: 🏛️ Petition
📋 Subject: 职业转型 · 减薪换行业
📝 背景：当前岗位疲劳、朋友推荐的机会在邻近行业、减薪 20%、风险未知
🔍 意图澄清：用户倾向"走"，但在找自我说服的证据。真正要分析的不是"走不走"而是"减薪 20% 换行业的风险边界在哪"
建议领域：finance + execution + people + governance
```

区别：PLANNER 拿到这个 Petition，维度会正确——聚焦"减薪 20% 换行业的风险边界"，而不是泛泛的"换工作利弊"。六领域给出的分析能用。奏折里的建议有针对性。

## 情绪分离的具体做法

当用户的话里同时有情绪词和决策词：

- 情绪词：焦虑、崩溃、撑不下去、烦透了、再也受不了、崩了
- 决策词：辞职、分手、卖掉、搬家、暴力中断

**ROUTER 的动作**：

1. 一句话承认情绪（不是长篇安抚，一句）：
   > "听起来这段时间很累。"

2. 一个分离问题：
   > "我们先把这种感受放一边。只看事实：你说 '撑不下去'，具体是哪些事情撑不下去？能举 3 个具体场景吗？"

3. **等用户回复具体场景后**，再进入 Round 1 的复述。

4. 如果用户在第 2 步没能给出具体场景（只是继续抒情）：
   - **不启动全朝议**
   - 建议启动 STRATEGIST（如果有价值观/迷茫信号）
   - 或者只做 Express 快车道（"我帮你把混乱先列清楚"）

这一步的目的不是"压抑情绪"，是**防止把情绪反应当偏好记录进 SOUL.md**。SOUL 的维度积累规则基于证据——如果情绪状态下的判断被 ADVISOR 写成"evidence_count +1"，会污染整个档案。

## 什么时候 2 轮就够

- 用户自己已经把背景铺陈得很清楚
- 问题本身是低风险、可逆、小额
- 前两轮已经问出所有关键信息

强迫第 3 轮反而是浪费。规则是**最多 3 轮**，不是"必须 3 轮"。

## 什么时候超出 3 轮

**不允许超过 3 轮。** `pro/agents/router.md` 的 anti-patterns 明确写了：

> Do not clarify for more than 3 rounds

如果 3 轮后还不清楚，说明：
- 用户自己也不清楚——这时应该问"要不要启动 STRATEGIST"
- 或者问题不适合决策引擎——ROUTER 可能应该直接处理或建议用户自己先想想
- 或者应该走快车道先把信息摊开，再决定是否升级

## 和 SOUL 的互动

如果 SOUL.md 已建立且有高置信度维度（≥0.6），ROUTER 在澄清时会主动检查：

- 当前话题是否触及高置信度 SOUL 维度？
- 如果触及但用户没提 → 主动问。

例子：
- SOUL 里 "家庭 > 事业"（置信度 0.8）
- 用户问："要不要接这个新项目？"
- ROUTER 的 Round 2 不止问工作相关，还会问："这个项目会占多少周末时间？家里怎么看？"

用户没提家庭，但 SOUL 说家庭是核心——ROUTER 替用户记得。这是 SOUL 存在的第一意义。

## 验证清单 · ROUTER 在升级前自问

- [ ] 用户问题已经分类了吗（5 类之一）？
- [ ] 第 1 轮复述做了吗？
- [ ] 第 2 轮问的是一个问题，不是三个并列？
- [ ] 情绪信号出现时做了分离吗？
- [ ] SOUL 高置信度维度检查过了吗？
- [ ] 现在写 Petition，PLANNER 能**立刻开始拆维度**而不需要再猜测？

最后一条是关键自检：如果写 Petition 时还要"PLANNER 应该会推断……"，澄清没做够。
