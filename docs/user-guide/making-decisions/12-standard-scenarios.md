# 12 个标准场景 · 预配置的领域路由

> 本文档是作者自用参考。写给懂系统内部的人看。内容源自 `references/scene-configs.md`。

## 为什么要标准场景

PLANNER 不是每次都从零拆维度——`references/scene-configs.md` 定义了 12 个高频场景的**默认领域路由**和 **REVIEWER 焦点问题**。PLANNER 先判断 Subject 落在哪个场景，再基于标准配置调整。

标准配置的价值：
- 不漏关键领域（过往经验总结）
- REVIEWER 有现成的情感/价值观追问点
- 节省 PLANNER 规划时间，token 使用更高效

下面逐个场景过一遍。每个场景包含：**领域路由**、**REVIEWER 焦点问题**、**适用信号**、**常见误判**。

---

## 1. 职业转型（辞职 / 换工作 / 创业）

**领域路由**：
- 财务 → FINANCE
- 技能 → GROWTH
- 人脉 → PEOPLE
- 执行 → EXECUTION
- 风险 → GOVERNANCE
- 生活影响 → INFRA

（六领域全到位，因为这是高权重多维度决策）

**REVIEWER 焦点问题**：
> 是在逃避还是在追求？家人怎么想？

**适用信号**：
- "要不要辞职"、"换工作"、"跳槽"
- "想创业"、"要不要下海"
- "接 offer 吗"（大方向改变的 offer）

**常见误判**：
- 被识别成场景 4（年度规划）—— 不，个人职业方向是场景 1
- 被识别成场景 5（产品/创业）—— 个人创业动因 → 场景 1；产品本身决策 → 场景 5

---

## 2. 投资决策

**领域路由**：
- 财务 → FINANCE
- 市场 → EXECUTION
- 风险 → GOVERNANCE
- 人 → PEOPLE

（4 领域，focus 在数字和风险）

**REVIEWER 焦点问题**：
> FOMO 还是理性？完全归零能否承受？

**适用信号**：
- "这只股票值得买吗"
- "要不要投这个基金"
- "这个朋友的项目值得入吗"
- "加密货币买不买"

**常见误判**：
- 被识别成场景 6（大额采购）—— 投资是期待回报的资产配置，采购是消费
- "朋友让我投他项目" —— 先问：是投资性质还是借款？纯投资走场景 2；借款带关系走场景 7

**特别注意**：
REVIEWER "完全归零能否承受"是关键追问。不能承受 = 不该投。
PEOPLE 领域的存在是为了评估：如果这是朋友项目，关系影响也要考虑。

---

## 3. 搬迁 / 跨城跨国

**领域路由**：
- 财务 → FINANCE
- 关系 → PEOPLE
- 基础设施 → INFRA
- 合规 → GOVERNANCE
- 文化 → GROWTH

（5 领域）

**REVIEWER 焦点问题**：
> 是否真的了解目的地？返回的成本？

**适用信号**：
- "要不要搬去 X"
- "跨国 / 跨省去 Y 工作生活"
- "回国 / 出国"

**常见误判**：
- 被识别成场景 1（职业转型）—— 如果换城市但公司不变 → 场景 3；如果换工作顺带换城市 → 两个场景叠加
- 只考虑财务—— 搬迁最大的成本是关系成本（配偶、孩子、父母、朋友）和基建成本（房子、医疗、学校），不能只看钱

**GROWTH 领域在场景 3 里意义特殊**：
文化适应 = growth 范畴，不是单纯学东西。包括语言、习俗、社交方式。

---

## 4. 年度目标规划

**领域路由**：
- 关系 → PEOPLE
- 财务 → FINANCE
- 学习 → GROWTH
- 执行 → EXECUTION
- 纠偏 → GOVERNANCE
- 健康 → INFRA

（六领域全到位）

**REVIEWER 焦点问题**：
> 目标太多？可测？和价值观对齐？

**适用信号**：
- "帮我做 2026 年度计划"
- "这个季度的目标"
- "年度复盘 + 新年规划"

**常见误判**：
- 被识别成场景 11（时间管理）—— 年度规划涉及**大方向设定**，时间管理涉及**日常节奏优化**
- 被误认为不是决策—— 年度规划里包含多处优先级取舍（事业 vs 家庭 vs 健康），是决策问题

**REVIEWER 在这个场景的三个关键问题**：
1. **目标太多？** — 一年超过 5 个大目标 = 必然完不成，需要缩减
2. **可测？** — 目标必须有可观察的成功信号，"努力学习" ≠ 目标
3. **和价值观对齐？** — 主动 ping SOUL.md，防止设了一堆和核心价值观冲突的目标

---

## 5. 产品 / 创业决策

**领域路由**：
- 市场 → EXECUTION
- 财务 → FINANCE
- 团队 → PEOPLE
- 合规 → GOVERNANCE
- 基础设施 → INFRA
- 品牌 → GROWTH

（六领域全到位）

**REVIEWER 焦点问题**：
> 真痛点？你是对的人吗？

**适用信号**：
- "这个产品要不要做"
- "要不要融资"
- "这个 pivot 值不值得"
- "要不要关掉这条业务线"

**常见误判**：
- 被识别成场景 1（职业转型）—— 已经创业的人在问下一步产品决策 → 场景 5；从上班→创业 → 场景 1
- 只考虑市场—— 产品决策必须涵盖团队能力 + 财务可持续 + 合规风险

**REVIEWER "你是对的人吗"的深意**：
这不是质疑创业者能力，是在问：**这个痛点是否需要你来解决？** 如果痛点真但你没有独特优势（领域知识 / 技术 / 资源 / 人脉），即使市场存在也不是你的机会。

---

## 6. 大额采购

**领域路由**：
- 财务 → FINANCE
- 实用性 → EXECUTION
- 风险 → GOVERNANCE

（3 领域，最精简配置之一）

**REVIEWER 焦点问题**：
> 需要还是想要？一个月后还想买吗？

**适用信号**：
- "要不要买 X"（X = 车、房、电器、奢侈品、设备）
- "这个订阅值不值得续"
- "升级设备吗"

**常见误判**：
- 被识别成场景 2（投资）—— 是否产生回报？会 → 投资；纯消费 → 采购
- 跳过澄清直接分析 —— 大额采购需要意图澄清（需要还是想要？使用频率？替代方案？）

**REVIEWER "一个月后还想买吗"是 Regret Test 变体**：
冲动购买典型模式：看到 → 想要 → 买 → 三周后失去兴趣。通过 1 个月延迟可以大幅过滤掉这类。

---

## 7. 关系决策

**领域路由**：
- 关系 → PEOPLE
- 情感 → INFRA（Fitness · 心理/情绪基建）
- 风险 → GOVERNANCE
- 社交 → GROWTH

（4 领域）

**REVIEWER 焦点问题**：
> 是否在偏见中评估对方？

**适用信号**：
- "要不要和 X 分手 / 离婚"
- "要不要和 X 在一起"
- "要不要把 X 从朋友圈删掉"
- "要不要修复和 X 的关系"

**常见误判**：
- 被识别成场景 4（年度规划）—— 关系虽然是人生大事但具体的关系决定 → 场景 7
- 被识别成场景 1（职业）—— 同事关系特殊冲突如果到分水岭级别 → 场景 7

**INFRA 在场景 7 里的角色**：
情感基建 = 心理健康、情绪承受力、恢复能力。关系决策伤身伤神，必须评估自身承受力。

**REVIEWER 问的"偏见评估"**：
- 确认偏误（只看对方的优点或缺点）
- 对比偏误（用他人/前任作对照）
- 投资偏误（"已经付出这么多不能放弃"）
- 人设偏误（对方某些言行不符合你对他的设定但仍想相信设定）

---

## 8. 复盘（Review）

**领域路由**：交给 RETROSPECTIVE 处理，不走全朝议

**不同时间粒度**：
- 日复盘
- 周复盘
- 月复盘
- 季度复盘
- 年度复盘

**REVIEWER 不介入**：复盘不是决策，是信息整理和模式识别。

**触发词**：
- "复盘一下"
- "看看怎么样"
- "振り返り"
- "morning court"

**不要**：走全朝议——复盘走 RETROSPECTIVE 的 Review Mode 快得多，也更合适。

---

## 9. 健康管理

**领域路由**：
- 身体 → INFRA
- 时间 → EXECUTION
- 成本 → FINANCE
- 纪律 → GOVERNANCE

（4 领域，"纪律"维度给 GOVERNANCE 是因为 GOVERNANCE 管规则执行）

**REVIEWER 焦点问题**：
（scene-configs.md 没给，这里补典型追问）
- 是被问题吓到才动还是主动预防？
- 能坚持多久？
- 家人支持吗？

**适用信号**：
- "要不要开始健身"
- "要不要改饮食"
- "要不要做 X 手术 / 治疗"
- "要不要看医生"

**常见误判**：
- 看医生 / 做检查类问题不必走全朝议—— 直接查信息和 ROUTER 判断就够了
- 生活方式级别改变 → 全朝议（因为涉及多维度+长期执行力）

---

## 10. 学习计划

**领域路由**：
- 内容 → GROWTH
- 执行 → EXECUTION
- 成本 → FINANCE
- 人脉 → PEOPLE

（4 领域）

**REVIEWER 焦点问题**：
（scene-configs.md 没给，这里补典型追问）
- 是为什么要学？
- 学完之后如何验证掌握了？
- 时间投入可持续吗？

**适用信号**：
- "帮我做学 X 的计划"
- "要不要报这个课"
- "学 X 还是学 Y"

**大多数情况走快车道而非全朝议**：
学习**计划**是规划类请求，不是决策——走快车道 1-2 个领域（growth + execution）就够。
只有涉及选择（学 X 还是 Y）、大额付费、换职业方向的学习才升级全朝议。

---

## 11. 时间管理与能量优化

**领域路由**（EXECUTION 拆 3 子领域 + FINANCE + GOVERNANCE + INFRA，共 6 维度）：
- 能量 → EXECUTION（Logistics · 后勤）
- 日程 → EXECUTION（Operations · 运营）
- 工具 → EXECUTION（Equipment · 装备）
- 成本 → FINANCE
- 习惯 → GOVERNANCE
- 节奏 → INFRA

**REVIEWER 焦点问题**：
> 真没时间还是在回避？砍掉的事情有人真的在意吗？上一次心安理得的休息是什么时候？

**适用信号**："时间不够"、"要不要放弃 X 活动"、"能量管理"、"注意力分配"、"换作息"

**独特处**：REVIEWER 的"真没时间还是在回避"非常刻薄但必要。80% 的"时间管理"请求本质是回避某件事的借口。

**INFRA 的"节奏"维度**：生理节奏（睡眠、运动、用餐）+ 生命节奏（季度、年度）。经常被忽略。

---

## 12. 重大家庭决策

**领域路由**（9 维度，最多的场景之一）：
- 财务 → FINANCE（储备 + 支出）
- 分工 / 动态 → PEOPLE（两次：分配 + 评估）
- 执行 / 情报 → EXECUTION
- 法律 / 应急 → GOVERNANCE
- 身心 / 住房 → INFRA

**REVIEWER 焦点问题**：
> 谁的声音没被听到？最坏情况能扛住吗？五年后会后悔做还是后悔没做？

**适用信号**：
- "要不要生孩子 / 二胎"、"接父母过来住"、"遗产分配"
- "结婚 / 离婚"（涉及家庭层 → 场景 12；纯夫妻 → 场景 7）
- "买学区房 / 换学区"、"父母手术 / 重大医疗决策"

**为什么 PEOPLE 要分两次**：
- 分配 — 决策执行时的角色分工
- 评估 — 家庭成员的态度和影响

家庭里不同角色的声音权重差别很大，不能用一个 PEOPLE 打包。

**REVIEWER 三问是最严格的 Regret Test 变体**——谁的声音被绕过了？能承受失败吗？5 年后哪个悔恨更重？

---

## 场景判别流程 · PLANNER 的自检

```
1. 复盘类？→ 场景 8，RETROSPECTIVE 处理
2. 大额金钱？有回报期望 → 场景 2；无 → 场景 6
3. 关于人？
   - 个人职业方向 → 1 | 产品创业 → 5 | 学习 → 10
   - 健康 → 9 | 关系 → 7 | 家庭 → 12
4. 空间/地理？→ 场景 3
5. 关于时间？长期 → 场景 4；日常 → 场景 11
6. 都不是？→ 自由拆维度
```

## 不在 12 个场景里的怎么办

PLANNER 遇到不匹配任何标准场景的 Subject：
- 自由拆维度，依然遵守不超过 6 个维度、每个维度有明确任务和质量标准的 HARD RULE
- **不要强行套模板**——如果某个标准场景部分匹配就把不匹配的部分删掉，反而容易漏
- 必要时用 ROUTER 澄清更多信息，再判断

## 验证清单 · PLANNER 用标准场景前自查

- [ ] Subject 明确落在某个标准场景吗？
- [ ] 标准路由适配用户当前情况吗？
- [ ] 有没有特殊情况需要加 / 减领域？
- [ ] SOUL 高置信度维度是否覆盖/补充标准路由？

符合则用标准场景，不符合自由拆。
