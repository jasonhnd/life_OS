# 主题系统概览 · 文化外壳可换，决策引擎不变

## 核心设计思想

Life OS 三层架构：

```
引擎（engine/*.md）     ← 多个功能 ID · 决策逻辑 · 永不变
    ↓
主题（themes/*.md）      ← 显示名 / 语气 / 触发词 · 可换
    ↓
语言环境（per-session）  ← 每次会话独立选择
```

引擎只认 `router`、`planner`、`reviewer` 这样的**功能 ID**。
主题把这些 ID 翻译成你文化里熟悉的名称：`丞相`、`中书省`、`门下省` / `Consul`、`Senate`、`Tribune` / `社長室`、`経営企画部`、`法務部`。

**换主题 ≠ 换系统**。你换的是外壳，不是引擎。

## 每个主题包含什么

每个主题文件约 60 行，结构固定：

| 字段 | 说明 | 例子 |
|------|------|------|
| **Language** | 硬规则——主题决定输出语言 | English / Chinese / Japanese |
| **Tone** | 语气定义 | "现代直白，不用古语" |
| **Role Mapping** | engine ID → 显示名 + emoji + 输出标题 | `router → 丞相 🏛️` |
| **Domain Mapping** | 6 个领域 → 显示名 | `FINANCE → 户部 · 财务` |
| **Summary Report** | 总结报告的名字和格式 | "奏折" / "Executive Brief" |
| **Session Commands** | 触发词（多语言） | "上朝"、"退朝"、"quick" |
| **Cultural Context** | 给 AI 的文化背景提示 | 帮 AI 理解为什么"御史台"要独立 |

## HARD RULE · 主题决定输出语言

**这是不可违反的规则：**

| 主题选定后 | 整个会话的所有输出必须是 |
|-----------|---------------------|
| `en-roman` / `en-usgov` / `en-csuite` | 英语 |
| `zh-classical` / `zh-gov` / `zh-corp` | 中文 |
| `ja-meiji` / `ja-kasumigaseki` / `ja-corp` | 日本語 |

**无混合。无例外。** 每个 agent、每份报告、每次响应。

这个规则的理由：
1. **沉浸感**。你选了"三省六部"却看到英文 "Summary Report"，立刻出戏
2. **一致性**。如果 ROUTER 说中文、户部说英文，读起来像机器翻译的官方文件
3. **思考节奏**。语言影响思考——选了一个语言，让你的脑子整场都在那里

**例外：代码**。代码文件按文件本身的语言（变量名通常英文）。但**对话和报告**必须遵守主题语言。

## 主题是 Per-Session 独立

每个会话窗口**独立**选主题，不会持久化。

这意味着：
- 你昨晚用"三省六部"聊了人生方向
- 今早新开一个会话想谈工作，选了"社長室"
- 两者互不影响

**为什么这样设计？**

- 不同场景适合不同外壳（生活 vs 工作 vs 创业）
- 跨语言使用者可以自由切换（日常中文 + 技术英文 + 商务日文）
- 避免"一次选错永远错"的心理负担

### 存储位置

- 当前会话的主题：在会话 context 里（不写盘）
- 你的"默认偏好"：如果 `_meta/config.md` 里有 `default_theme: xx-yyy`，新会话会用它；否则询问

## 如何选主题

### 自动推断（推荐）

你说的触发词如果**唯一指向**某个主题，系统自动加载：

| 你说 | 自动加载 |
|------|---------|
| "上朝" | `zh-classical`（唐朝专属词） |
| "閣議開始" | `ja-kasumigaseki`（現代日本政府专属词） |

系统会确认："🎨 主题：三省六部"然后就开始。

### 语言探测 + 二级选择

如果触发词只能确认**语言**但不确定具体主题：

| 你说 | 系统做什么 |
|------|----------|
| "开始" / "开会" | 中文探测到 → 列出 zh 三个主题让你选 |
| "はじめる" | 日語探测到 → 列出 ja 三个主题让你选 |
| "start" / "begin" | 英文探测到 → 列出 en 三个主题让你选 |

### 全量选择

如果没有任何触发词 / 你说的是其他话 → 系统展示全部 9 个选项让你选：

```
🎨 Choose your theme:

English:
  a) 🏛️ Roman Republic — Consul, Tribune, Senate
  b) 🇺🇸 US Government — Chief of Staff, Attorney General, Treasury
  c) 🏢 Corporate — CEO, General Counsel, CFO

中文：
  d) 🏛️ 三省六部 — 丞相、中书省、门下省
  e) 🇨🇳 中国政府 — 国务院、发改委、审计署
  f) 🏢 公司部门 — 总经理、法务部、财务部

日本語：
  g) 🏛️ 明治政府 — 内閣総理大臣、枢密院、大蔵省
  h) 🏛️ 霞が関 — 内閣官房長官、内閣法制局、財務省
  i) 🏢 企業 — 社長室、法務部、経理部

Type a-i
```

## 中途切换主题

你可以在会话任意时刻切换：

```
你："switch theme" / "切换主题" / "テーマ切り替え"

系统：
  🎨 Current theme: 三省六部. Choose new theme:
  [a-i 列表]
  
你：h

系统（用新语言）：
  🏛️ テーマを「霞が関」に切り替えました。
```

**切换生效立即**：下一条输出就是新语言、新显示名。之前的对话不会被翻译（留在原来的样子）。

## 主题不影响功能

无论你选哪个主题，**以下核心功能完全相同**：

1. ROUTER 接你的话
2. PLANNER 起草方案
3. REVIEWER 审查（有否决权）
4. DISPATCHER 派发
5. 六领域（PEOPLE / FINANCE / GROWTH / EXECUTION / GOVERNANCE / INFRA）执行
6. AUDITOR 审计
7. ADVISOR 行为分析
8. COUNCIL 跨领域辩论（当冲突时）
9. RETROSPECTIVE 会话开场
10. ARCHIVER 归档 + DREAM
11. STRATEGIST 召唤历史思想家

只是**显示名不同**。你在"三省六部"里叫"户部"的角色，和在"US Government"里叫"Dept. of Treasury"的角色，**是同一个 engine agent**，做同一件事，用同样的逻辑评估。

## 为什么要有这么多主题

三层原因：

### 1. 文化亲和力

一个日本用户看到"財務省"立刻知道"这是管钱的"。不需要解释。零学习成本。
一个中国用户看到"户部"或"财政部"同理。
一个英语用户看到"CFO"同理。

**用户的文化母语 = 最低认知负担**。

### 2. 适用场景

- 生活决策：很多人喜欢历史感（三省六部 / Roman Republic / 明治政府）
- 工作决策：企业外壳更契合（公司部门 / C-Suite / 企業）
- 严肃决策：政府外壳更贴合（中国政府 / US Government / 霞が関）

### 3. 心理锚点

外壳影响你**对待**决策的态度。"御史台"让人感到"要独立、要严肃"。"Internal Audit"让人感到"这是常规公司治理"。"元老"让人感到"长辈在看"。

选对外壳 = 调对心境。

## 什么时候该换主题

- **场景变了**：上午做生活决策（三省六部），下午做工作决策（霞が関）→ 切换
- **语言换了**：和日本同事聊完工作（日語主题），要给中国家人打电话讨论家事 → 切换
- **感觉不对**：选了 Roman Republic 觉得太严肃，聊不开 → 切到 C-Suite 或 Corporate
- **想要新鲜感**：同一个主题用久了审美疲劳 → 换一个

## 常见问题

**Q：我能只用一个主题永远不换吗？**
A：完全可以。在 `_meta/config.md` 里设 `default_theme: zh-classical` 就不会每次问。

**Q：我能改主题里的显示名吗？**
A：能。主题文件是开放的 markdown，你改了立刻生效。但建议先做拷贝 → 你自己的私人主题，别改原装的。

**Q：能在同一个会话里混语言吗？**
A：不能。HARD RULE。切到其他语言只能通过"switch theme"。

**Q：我想要一个不在列表的文化怎么办？**
A：你自己加。见 [添加新主题](adding-a-theme.md)。不需要改任何引擎代码。
