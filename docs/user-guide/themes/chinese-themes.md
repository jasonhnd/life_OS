## "上朝"的自动推断逻辑

"上朝"是**唐朝专属词**，只有三省六部主题用它。系统的逻辑：

```
用户说"上朝" 
  → 唯一指向 zh-classical
  → 自动加载，确认：🎨 主题：三省六部
  → 直接启动，不再问
```

相比之下：

```
用户说"开始" / "开会"
  → 探测到中文，但可能是 zh-classical / zh-gov / zh-corp 任一个
  → 询问：🎨 请选择主题（d/e/f）
```

**设计意图**：让熟悉唐朝制度的用户用最自然的词（"上朝"）零步启动；不熟悉的用户说通用词（"开始"）再选。

## 何时选哪个

### 🏛️ 三省六部 — 选这个如果...

- 你欣赏**唐朝制度的精妙**——这是**世界最早的三权分立**，早西方一千年
- 你希望决策氛围有**历史厚度**。"奏折"这两个字本身就让人严肃
- 你对中国文化有亲近感，愿意在"户部管钱、兵部管执行"这种对应关系里思考
- 你做**人生级**决策，希望配得上的氛围（不是"今天买哪件衬衫"）

**注意**：三省六部**不用古语**——是用现代汉语说的。不会有"臣启奏"这种台词，只是官职名用唐朝的。

### 🇨🇳 中国政府 — 选这个如果...

- 你对**当代中国政府架构**熟悉（新闻、政策、政府文件常见）
- 你看到"发改委"立刻知道是做什么的，看到"审计署"知道它独立向谁报告
- 你做**政策类 / 合规类 / 大方向类**决策，政府外壳最契合
- 你是**公务员 / 政策研究者 / 关心国家事务**的用户

**注意**：这不是"打官腔"。语气要求是"现代行政语言，简洁高效，不用官话套话"。"国务院决定"是你作为个人"国务院"对自己的决定。

### 🏢 公司部门 — 选这个如果...

- 你在**企业环境**工作，"总经理"、"财务部"、"法务部"是你每天的词
- 你做**商业 / 职业 / 管理**决策，企业外壳最契合语境
- 你想要**最低学习成本**——不需要学什么是"吏部"或"发改委"
- 你需要**工具感**，不是仪式感

## 文化背景详解

### 🏛️ 三省六部

**历史根**：唐朝（618-907）的三省六部制度是中国历史上最成熟的政治分权设计。核心机制：

- **中书省** 起草政策（"掌定制号令，为宰相之职"）
- **门下省** 审查政策，**有封驳权**——认为不合适可以打回去
- **尚书省** 执行，下辖吏、户、礼、兵、刑、工六部
- **御史台** 独立于三省六部，直接监察所有官员

**为什么重要**：这是**世界史上第一个制度化的**"起草-审查-执行"分离。中书省起草了也不能直接拍板，必须过门下省。门下省可以打回。这套机制比西方现代三权分立早 1000 年。

Life OS 把这套古代智慧直接拿过来用在你身上：
- 中书省（PLANNER）起草，**但不能拍板**
- 门下省（REVIEWER）审查，**真的有否决权**
- 户部、吏部、礼部、兵部、刑部、工部六部对应六个生活领域
- 御史台（AUDITOR）独立监察决策质量
- 谏官（ADVISOR）直言不讳——古代有"左拾遗、右补阙"专门对皇帝说"你错了"

### 🇨🇳 中国政府

**体制根**：中华人民共和国国务院体制。核心特点：

- **国务院** 作为最高行政机关
- **发改委** 是"小国务院"——负责重大战略规划（对应 PLANNER）
- **人大常委会** 有**立法审查权**——可以否决违宪的行政决定（对应 REVIEWER）
- **审计署** 独立于各部委，直接向国务院报告（对应 AUDITOR）
- **国务院参事室** 直接建言，不受部门利益约束（对应 ADVISOR）

**零学习成本**：对中国用户而言，"财政部"就是财政部，"司法部"就是司法部，"住建部"对应基建和住房。看一眼就懂。

### 🏢 公司部门

**架构根**：现代中国企业通用的部门划分。核心特点：

- **总经理** 统管日常运营
- **战略规划部** 制定中长期战略（对应 PLANNER）
- **法务合规部** 审查重大决策，**有否决权**（对应 REVIEWER）
- **内审部** 独立于各部门，直接向董事会报告（对应 AUDITOR）
- **外部顾问** 不受内部利益关系约束的独立建议（对应 ADVISOR）

**契合职场语境**：你日常就是在这套语言里工作，选这个主题不用切换思维模式。

## 实战例子

### 例子 1：用三省六部做重大决策

```
你：我想辞职去创业，要不要走一遍完整流程

丞相：🏛️ 是大事。我召开完整议政——中书省起草、门下省审查、六部评估。

中书省：📜 规划文书
    议题：辞职创业
    背景：...
    方案 A：立刻辞职全职创业
    方案 B：半年内 side project + 储蓄
    ...

门下省：🔍 我以封驳权审之。
    方案 A 的财务缓冲假设过于乐观——驳回。
    条件通过方案 B：要求 12 个月现金流，非 6 个月。

尚书省：📨 派遣令
    户部评估现金流方案
    吏部评估人脉网络支撑
    兵部排执行时间表
    ...

户部：💰 财务评估
    ...

（每部逐一显示）

丞相：📋 奏折：辞职创业
     综合评分 7/10 —— 可行，但需更严财务准备
     
     必须处理：
     - 户部：现金流低于 12 个月不启动
     - 门下省：需书面退出条件
     
     政事堂（COUNCIL）触发了：
     户部与兵部分歧。讨论结果：减半初始投入。
```

### 例子 2：用中国政府审查投资

```
你：我想把 50 万买某个 ETF

国务院总理：🏛️ 是财政决定还是全面决策？

你：全面。

[走完整流程]

国务院总理：📋 国务院决定：50 万 ETF 投资
     综合评分 4/10 —— 当前配置比例不建议
     
     必须处理：
     - 财政部：占总组合风险预算超过 5%
     - 司法部：无书面止损准则
     
     国务院常务会议（COUNCIL）讨论结果：
     财政部与国务院参事室分歧。决定降至 15 万以内。
```

```
你：组里要我带新人，我该不该接

总经理：是带人的决定，涉及精力分配和职业方向——全面分析一下？

你：好

[走完整流程，简化版]

总经理：📋 总经理办公会决议：接收新人带教任务
     综合评分 8/10 —— 建议接，但要谈条件
     
     必须处理：
     - 人力资源部：与直属上级明确带教占用工时
     - 培训发展部：制定阶段性交付目标，避免无限兜底
```

## 反模式 · 不要做的事

### ❌ 在三省六部里用官场黑话
"臣启奏…恳请圣上…"——错了。主题明确要求"现代直白，不用古语"。

### ❌ 在中国政府里打官腔
"本着…精神，做好…工作"——错了。要求是"简洁、高效、不用官话套话"。

### ❌ 在公司部门里说大道理
"从战略高度来看…"——错了。职场语言是"这事能不能做，成本几多，我的分析是…"。

### ❌ 中英混用
选了中文主题就不用 "CFO" / "roadmap" / "KPI" 这些词。用"财务部"、"路线图"、"关键指标"。
