# SOUL.md · 人格档案总览

> SOUL.md 是你的"人格档案"——一份活的文档，记录你是谁、你在乎什么、你怎么思考。

## SOUL 是什么

SOUL.md 位于你的 second-brain 根目录，和 `user-patterns.md`、`wiki/` 并列。它不记录你做过什么决策（那是 `decisions/` 的事），不记录你的行为模式（那是 `user-patterns.md` 的事），它只回答一个问题：

**你是谁？**

具体来说，SOUL 由一组"维度"组成，每个维度描述你性格、价值观、信念的一个侧面：

- 风险偏好：保守 ↔ 激进
- 决策风格：数据驱动 ↔ 直觉
- 优先级：家庭 ↔ 事业 ↔ 自由 ↔ 安全 ↔ 成长
- 沟通方式：直接 ↔ 委婉
- 时间取向：当下 ↔ 未来
- 红线：绝不会做的事
- 核心信念：世界观的基本假设

维度不是系统预设的。你可以自己定义，也可以由系统在观察到足够证据后自动写入。

---

## 两个字段：What IS vs What SHOULD BE

每个 SOUL 条目都包含两个核心字段：

### What IS（实然）

**系统观察到的你**。基于真实数据：你做过的决策、你的行为模式、你在对话中透露的信号。这部分由系统自动填写。

举例：
> 你最近 5 个决策中有 4 个把"控制权"置于"利润"之上。

### What SHOULD BE（应然）

**你希望的自己**。这是你对自己的愿望、期许、理想。这部分系统不会替你填——它是留白的，等你想清楚了自己写。

举例（你自己填）：
> 我希望在风险可控的前提下，愿意为了更大回报放弃部分控制权。

### Gap（差距）

What IS 和 What SHOULD BE 之间的差距，就是成长空间。

**差距即成长**。系统不会帮你消除这个差距——它只负责如实呈现。消除差距是你自己的事。

## 四个来源

SOUL 条目可以来自四个地方：

| 来源 | 怎么产生 | 举例 |
|------|---------|------|
| **DREAM** | 睡眠阶段从 3 天行为数据中发现模式 | "最近 5 个决策中有 4 个优先考虑控制" |
| **ADVISOR** | 工作流后观察重复出现的价值信号 | "你每次都先问家庭影响" |
| **STRATEGIST** | 深度对话中你主动透露的价值观 | 你告诉苏格拉底"稳定比冒险重要" |
| **你自己** | 任何时候直接输入 | "记住：我绝不妥协 X" |

## 核心原则

### 1. 从零成长

SOUL.md 初始是空的。没有初始化向导，不会预填任何假设性条目。系统会在观察到足够证据后（≥2 个决策支持）自动开始写入第一批维度。

**如果 SOUL.md 不存在**：所有角色都会跳过对它的引用，系统照常运行。

### 2. 证据驱动

每个条目都必须链接到支持它的决策或行为。空口无凭的维度不会存在。

系统通过 `evidence_count`（支持证据数）和 `challenges`（矛盾证据数）自动计算 `confidence`：

```
confidence = evidence_count / (evidence_count + challenges × 2)
```

你不管理 confidence——它是自动算出来的。

### 3. 矛盾有价值

人是复杂的。同一个人可能在家庭决策上保守、在事业决策上激进；可能嘴上说要"自由"、实际选择了"安全"。

**这些矛盾不应该被掩盖**。SOUL 系统不会试图"调和"矛盾——它会把矛盾浮现出来，让你看见真实的自己。

每个条目的 `Challenges` 部分专门记录与该维度冲突的行为。当一个维度连续 3 次决策都被挑战，REVIEWER 会在下一次决策时给你警告：**你正在和自己的核心价值观作战**。

## SOUL 如何影响系统

不同 confidence 级别的 SOUL 条目对系统的影响不同：

| Confidence | 影响范围 |
|------------|---------|
| < 0.3 | 仅 ADVISOR 看到（潜伏期） |
| 0.3 – 0.6 | ADVISOR + REVIEWER |
| 0.6 – 0.8 | + PLANNER |
| > 0.8 | 全系统引用（包括 ROUTER 清查意图时） |

一个 confidence 0.8 的"家庭优先"维度，会影响 ROUTER 如何帮你澄清意图、PLANNER 如何加维度、REVIEWER 如何否决、ADVISOR 如何提醒。

**这是一个反馈循环**：你的 SOUL 越稳固，系统越贴近你。

## SOUL 和其他文件的关系

| 文件 | 记录什么 | 举例 |
|------|---------|------|
| `decisions/` | 具体决策 | "2026-04-01 决定用信托结构" |
| `user-patterns.md` | 行为模式 | "倾向避开财务维度" |
| **`SOUL.md`** | **你是谁** | **"风险偏好：中偏高"** |

- `user-patterns.md` 记录"你做什么"——描述性的
- `SOUL.md` 记录"你是谁"——身份性的
- 两者互相喂养：模式揭示价值观，价值观解释模式

## 六个角色如何读 SOUL

每个角色用 SOUL 的方式不同：

| 角色 | 读什么 | 怎么用 |
|------|-------|-------|
| **ROUTER** | 偏好、红线（confidence > 0.8） | 更精准地澄清意图——即便你没提到，也会问你在乎的维度 |
| **PLANNER** | 价值优先级（confidence ≥ 0.6） | 自动加入你没提但重要的规划维度 |
| **REVIEWER** | What IS vs What SHOULD BE 差距（c ≥ 0.3） | 价值一致性检查——决策违背愿望时提醒 |
| **ADVISOR** | 全部条目、evidence/challenges 计数 | 行为审计——强化或挑战 SOUL 条目 |
| **STRATEGIST** | Worldview、未解决的矛盾 | 根据你的张力推荐思想家 |
| **DREAM** | 全部（读写候选） | 发现新候选、更新计数、提议演化 |

**所有角色在引用 SOUL 前都会先检查它是否存在**。如果不存在或为空，跳过 SOUL 相关逻辑，系统照常运行。

## 如何开始

**不用做任何事**。

- 第一次 session 时 SOUL.md 不存在 → 系统照常工作
- 几次决策后 ADVISOR 开始积累观察
- 当某个维度积累到 ≥2 个证据，系统会自动写入第一个条目（confidence 0.3）
- 每次 Start Session 的健康报告会告诉你新增了什么维度
- 你随时可以编辑 What SHOULD BE、删除维度、或直接添加新维度

**SOUL 是活的**。它会随着你的每个决策慢慢长大。你不需要一次性把自己写明白。
