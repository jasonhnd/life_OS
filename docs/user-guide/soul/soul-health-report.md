# SOUL 健康报告

> 每次 Start Session，简报顶部会出现一块固定位置的"SOUL Health Report"。这是你读懂自己的窗口。

## 报告位置

**固定在 Start Session 简报的靠前位置**，在项目状态和待办事项之前。

不管 SOUL 多大、多小，甚至为空，这块都会出现。它是 Mode 0（Start Session）简报的必选部分，不是可选的。

如果 SOUL 为空：
> 🔮 SOUL 还在积累初始观察。再做几次决策，你的第一个维度就会浮现。

---

## 报告结构

```
🔮 SOUL Health Report
━━━━━━━━━━━━━━━━━━━━━━━━━━━

📊 当前画像：
   活跃维度（confidence > 0.5）：N 个
   · [维度 A] 0.8 🟢 ↗ (+2 evidence since last session)
   · [维度 B] 0.6 🟢 → (no change)
   · [维度 C] 0.5 🟡 ↘ (+1 challenge — 上次决策矛盾)

🌱 新维度（自上次 session 以来自动检测）：
   · [维度 D] 0.3（基于 2 个决策）
     What IS：[系统观察]
     What SHOULD BE：[等你填写——当你想清楚]

⚠️ 冲突提醒：
   · [维度 X] 最近 3 个决策全部被挑战 → 需要反思或修订

💤 休眠维度（30+ 天无激活）：
   · [维度 Y] — 最近没有相关决策

📈 本期轨迹：
   净 evidence +N、净 challenges +M、新维度 +K
```

---

## 趋势箭头怎么读

每个活跃维度后面有一个箭头，表示 confidence 相对上次 session 的变化：

| 箭头 | 含义 | 触发条件 |
|------|------|---------|
| ↗ | 上升 | `confidence_Δ > +0.05` |
| → | 持平 | `|confidence_Δ| ≤ 0.05` |
| ↘ | 下降 | `confidence_Δ < -0.05` |

这是通过 snapshot 机制计算的——archiver 在每次 session 结束时 dump 一份 SOUL 快照到 `_meta/snapshots/soul/`，下次 Start Session 时 RETROSPECTIVE 读快照和当前 SOUL.md 对比。

**没有快照的维度**：新添加的维度标 🌱 NEW，被删除的维度标 🗑️ REMOVED。

## 特殊状态标记

### 🌟 新晋核心

当某个维度的 confidence 从 secondary 跨入 core（> 0.7），标为"新晋核心"。

含义：这个维度现在要开始影响 PLANNER 和 ROUTER 了，不再只是 REVIEWER 看看。你的人格中某个侧面**固化**了。

### ⚠️ 从核心降级

当某个核心维度的 confidence 从 core 掉回 secondary（< 0.7），标为"从核心降级"。

含义：最近累积的矛盾让这个维度不再稳固。可能你在变化，也可能当时的观察不够准确。需要你关注。

### 💤 休眠

当某个维度连续 30 天没有相关决策激活（没被 support 也没被 challenge），标为"休眠"。

含义：要么这个维度已经不再是你生活的焦点，要么你最近的决策类型没有涉及它。如果长期休眠（90+ 天），考虑删除或归档。

### ❗ 冲突区

当某个维度最近 3 个决策**全部**被标记为 challenge，进入冲突区。

含义：你正在和自己的某个价值观作战。有两种解读：

1. 这个价值观过时了，你应该修订它（编辑条目或删除）
2. 你最近在做违背核心价值观的事，需要反思

下次决策时 REVIEWER 会把这个维度摆在最前面，用 ⚠️ SOUL CONFLICT 警告让你意识到。

## 本期轨迹

报告末尾的"本期轨迹"是一个粗粒度的整体画像：

```
📈 本期轨迹：
   净 evidence +12、净 challenges +3、新维度 +2
```

- **净 evidence / net challenges**：所有维度总的支持 / 矛盾变化
- **新维度**：本期系统自动检测到的维度数

这个数据跨多个 session 有意义——如果你连续几期都是"净 challenges 多于 evidence"，意味着你整体处于一个"价值观动荡"期，值得在 STRATEGIST 对话中深挖。

## 如何用这份报告

### 快速扫一遍箭头

30 秒就能看完：

- 所有活跃维度是 ↗ 还是 ↘？
- 有没有 ❗ 冲突？
- 有没有 🌱 新维度出现？

### 如果有新维度

打开 SOUL.md，看看 What IS 写得对不对。

- 对 → 填 What SHOULD BE（或先留着）
- 不对 → 删除维度，或在本次 session 开头说"undo recent SOUL"

### 如果有冲突警告

这是系统在**对你说实话**。

不要忽略它。要么花时间想清楚这个维度是不是要修订，要么在本次 session 里自觉避开会和它矛盾的选择。

### 如果有休眠维度

思考一下：这个维度是否还反映真实的你？

- 如果**是**，不用动——它只是最近没被激活
- 如果**不是**，手动删除它（系统不会自动删）

## 举个真实例子

一个连续 5 周用 Life OS 的用户，第 5 周的 SOUL Health Report 可能长这样：

```
🔮 SOUL Health Report
━━━━━━━━━━━━━━━━━━━━━━━━━━━

📊 当前画像：
   活跃维度（confidence > 0.5）：6 个
   · 家庭优先 0.85 🟢 → (no change)
   · 控制权偏好 0.78 🟢 ↗ (+2 evidence since last session)   🌟 新晋核心
   · 数据驱动决策 0.72 🟢 → (no change)
   · 风险保守 0.65 🟡 ↘ (+2 challenges — 最近两次选择激进方案)
   · 长期视角 0.58 🟡 ↗ (+1 evidence)
   · 过程透明 0.52 🟡 → (no change)

🌱 新维度（自上次 session 以来自动检测）：
   · 独立决策偏好 0.3（基于 2 个决策）
     What IS：你在 2 个决策中都拒绝了加入投委会的建议
     What SHOULD BE：[等你填写]

⚠️ 冲突提醒：
   · 风险保守 最近 3 个决策全部被挑战 → 可能你正在进入新阶段，或该维度需要修订

💤 休眠维度（30+ 天无激活）：
   · 社交活跃度 — 最近没有相关决策

📈 本期轨迹：
   净 evidence +5、净 challenges +3、新维度 +1
```

从这份报告他能读出：

- **控制权偏好**刚跨过 0.7，从今天开始会影响 PLANNER 和 ROUTER
- **风险保守**正在动摇——最近他做了两次激进选择，这个维度是真的在改变，还是只是一时冲动？需要反思
- 系统发现他可能有"独立决策偏好"——他可以打开看看观察是否准确
- **社交活跃度**休眠了，30 天没有社交相关决策——是最近工作太忙？还是这个维度不再反映他？

**SOUL Health Report 不是一个"数据面板"**。它是一面镜子。镜子不漂亮，但它不骗你。
