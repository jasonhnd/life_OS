# 年度规划 · 跑一次完整的年度议程

年度规划是 Life OS 里最重的一种议程——它组合了六部议政、RETROSPECTIVE 的年度模式、Strategic Map 年终复盘、SOUL / Wiki 演变回顾。

这一次跑完，你会得到：来年的方向、战略线的梳理、明确废弃的项、需要挽救的项、SOUL 成熟度评估。

本文按真实节奏走：场景配置 #4（年度目标规划），**一次 2-4 小时**。

---

## 什么时候跑

- **年末**（12 月末到 1 月第 2 周）最常见
- **财年结束**（如果你按日本财年 4 月初）
- **生日前后**（按年龄维度）
- **重大人生转折后**（搬家、结婚、换工作后 1 个月内）

**不适合**：刚经历重大事件还没沉淀的 2 周内——情绪干扰太大。

## 前置准备（议程开始前）

这步不在议程里，但决定了议程质量。建议议程前 2-3 天完成。

### 1. 全量 review

```
review
```

触发 Review Mode。看系统给的 metrics：
- DTR（决策吞吐率）
- ACR（动作完成率）
- OFR（计划遵守率）
- DQT（决策质量趋势）—— 月度以上才显示
- MRI（决策修正指数）
- DCE（决策一致性评价）
- PIS（项目影响评分）

把这些数据记下来。议程里会用到。

### 2. SOUL 审视

打开 SOUL.md，看每个维度：
- confidence > 0.7 的：是否还准确？今年还是这样吗？
- confidence 0.3-0.6 的：是成长中还是伪命题？
- 维度之间有冲突吗？哪个 Gap 最大？

### 3. Wiki 审视

打开 INDEX.md，按 domain 扫一遍：
- 哪些 domain 增长快？这是关注焦点？
- 哪些条目 confidence 下降？知识在过时？
- 哪些 domain 是空的？你是否忽视了某个领域？

### 4. Strategic Map 审视

打开 `_meta/STRATEGIC-MAP.md`（最新编译）：
- 每条战略线本年进展
- 哪条线 archetype 是 🔴（stall / drift）？
- 哪条线已完成 time_window？
- 哪些项目不在任何线上？

## 意图澄清（更深）

普通决策澄清 2-3 轮。**年度规划要 3-5 轮**。

### 第一轮：定义"年度"

```
🏛️ 丞相

开始前先对齐。你说"2027 年度规划"是：
a) 日历年：2027 年 1 月到 12 月
b) 你的个人年：比如生日到下一个生日
c) 财年：2027 年 4 月到 2028 年 3 月

另外这次规划你想要的是：
- 全面重置（从零开始想要什么）
- 增量调整（在现有战略线上优化）
- 方向校验（不大改只确认）
```

你：`日历年，方向校验 + 某些战略线的重置`

### 第二轮：今年的主基调

```
🏛️ 丞相

2026 年（去年）的基调，我读到的：
- 从 SOUL 演变看："长期主义" 从 0.5 涨到 0.82
- 从 Strategic Map：3 条战略线，1 条完成（personal-finance-restructure）
  2 条活跃（product-expansion / health-baseline）
- 从 Decisions Dashboard：17 次决策，3 次封驳修订，2 次议政

2027 年你希望的基调是？关键词 3 个以内。
```

你：`深耕、对家庭更多在场、不开新线`

### 第三轮：约束

```
🏛️ 丞相

"不开新线"是重要的约束——这会排除很多新想法。
你确认吗？因为去年你也说过"不开新线"最后开了 Telegram 这条。

另外"对家庭更多在场" 是时间上的（陪伴时长）还是 
在场质量上的（专注度）？这两个完全不一样。
```

你：`我确认不开新线。家庭主要是质量——专注时间少了。`

### 第四轮：成功标准

```
🏛️ 丞相

最后：2027 年末回头看，你怎么判断"成功了"？

不要说 "我会知道的" —— 请具体：
- 哪个战略线上的什么里程碑
- 家庭关系上的什么信号
- 健康/财务上的什么数字
- SOUL 上的什么演变
```

你：`product-expansion 线的 critical-path 到达 XX；家庭每周至少 2 晚无手机时段；每周 3 次运动；SOUL "完美主义门槛" 维度 What SHOULD BE 填完`

### 第五轮：排除项

```
🏛️ 丞相

那什么是你明确 **不** 要的？（这能让六部聚焦）
```

你：`不投入新的开源项目，不接受任何短期外包，不参加超过 1 次行业大会`

## 一些经验

**最常见失败**："规划完美，3 月后全忘"。原因是没有降级机制——规划了目标但没规划**如何知道在失败**。解：刑部的季度 review + Q2 降级规则。

**诚实 > 漂亮**：如果澄清时说假话，议程基于假话推理，规划漂亮但无用。

**不是商业 OKR**：不是 stretch goal 文化，是**刑部 + 工部 + SOUL** 的联合产出——能真做到 + 你真想要 + 失败早期可发现。**目标不是达到所有，是年末清楚为什么达到或没达到**。
