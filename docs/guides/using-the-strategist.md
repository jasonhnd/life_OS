# 翰林院 · 和历史上的思想家对话

翰林院（STRATEGIST）是 Life OS 里唯一**不做决策**的部分。它不打分、不出奏折、不走议程。它只做一件事：**让你和人类最伟大的思想家对话**。

本文告诉你：什么问题适合来翰林院、怎么挑思想家、三种对话模式怎么用、对话结束后东西存在哪。

---

## 什么时候来翰林院

翰林院不是研究工具（那是六部的事）。它处理的是**你自己想不明白**的问题。

**适合来翰林院的问题**：
- "我到底想要什么"
- "我的人生是不是偏了"
- "这两个价值观哪个更重要"
- "我最近感觉不对但说不出哪里不对"
- 经过深度对话你可能得到新视角而非数据

**不适合**：
- "我该接这个 offer 吗" → 升朝议事
- "这合同有没有风险" → 快车道刑部
- "帮我做个计划" → 六部议程

**简单判断**：问题是"选 A 还是 B"→六部；问题是"我是谁/我要什么"→翰林院。

## 丞相怎么主动问你

你不用记口令。只要你说的话里有**抽象思考信号**，丞相会主动问：

```
你：最近心里有点乱，说不上来是什么

🏛️ 丞相

这听起来更像是需要一次深度对话而不是一次议程。
要启动翰林院吗？

(翰林院不做决策，不打分——它是和历史上的思想家对话
的空间。适合你现在这种"说不上来"的状态)
```

你：`好`

或你也可以主动说：

```
我想找个思想家聊聊
我想去翰林院
让我和贾伯斯对话
```

都可以。

## Step 1 · 翰林院先问你要探索什么

翰林院接管后不会立刻给你列思想家清单。先问：

```
🎋 翰林院

今天你想探索什么问题？
```

**这里不要一句话**。翰林院需要你展开说——越具体越好。

**差的输入**：
- "完美主义问题"
- "我最近焦虑"
- "人生意义"

**好的输入**：
- "我发现自己 3 次推迟发布 Life OS 新版本，每次都说'再打磨一轮'。
  但我也知道这样发布窗口在溜走。我不确定这是我追求质量的本能还是
  在逃避评价"
- "我母亲最近病了，我觉得应该回去陪她。但我又在创业关键期。
  我不知道我是在逃避还是在尽责"
- "我最近总想买一个明明用不上的东西。不是缺钱。我不明白"

翰林院听到这种具体的输入会给你好的推荐。

```

"完美主义门槛" 0.3）+ 这个具体问题，我推荐三个方向：

一对一模式（你和一个人深谈）:

     偏执，两面都有

  ⚔️ 达·芬奇 vs 贾伯斯 — 完成度 vs 发布节奏

  🎭 贾伯斯 + 达·芬奇 + 宫本武藏（武道里的"炼到极致" vs

我建议从一对一贾伯斯开始——他会直接戳你。然后你觉得
被刺到了再加达·芬奇中和。
```

## Step 3 · 选择 + 开始对话

你：`先一对一贾伯斯`

翰林院**把贾伯斯作为独立 subagent 启动**（这是硬规矩——不是模拟）。贾伯斯收到的信息：
- 他的身份和角色扮演指令
- 你的原始问题
- 模式（一对一）
- 不传递翰林院的推理过程、不传递其他思想家的意见

贾伯斯开始：

```
💼 Steve Jobs

🔎 你说你 3 次推迟发布，每次都说"再打磨一轮"——
这个模式我熟悉。1997 年我回苹果时，有一堆没发布的产品。
工程师说"再一个月就好"。我砍掉 70%，留下的 30% 直接发布。
不是因为它们完美，是因为 they were ready to teach us
something we couldn't learn otherwise.

💭 我想问你——你"再打磨一轮"的时候，是在让产品更好，
还是在避免听到用户说"这不够好"？

🎯 不要回答我。诚实回答你自己。
```

你回答。贾伯斯追问。这是**真实的对话**，不是讲课。

他**不会**：
- 给你"完美主义的 5 个解决方案"
- 假装他是 AI
- 切出角色说"综合来看..."

他**会**：
- 引用他真实的经历（砍 NeXT 产品线、iPod 发布、1997 年苹果）
- 用他的方法论（focus by saying no）应用到你的具体问题
- 质问你、挑衅你、戳你的舒适区

## Step 4 · 三种模式的具体玩法

### 一对一（1 对 1）

**最密集**。你和一个人深谈。翰林院保持沉默除非你叫它。

用时：15-45 分钟。

**适合**：你需要被一种特定风格的质问刺到。比如你需要被苏格拉底问到哑口无言，或者需要被尼采戳破自欺。

### 圆桌（2-4 人）

多角度但不对抗。翰林院做主持，让各位轮流说，然后总结异同。

**真实例子**：

```
你：我 40 岁了，是应该继续技术深耕还是转管理

🎋 翰林院

圆桌推荐:
- 稻盛和夫（从技术到经营的转变）
- 诸葛亮（管理人的 high-agency 视角）
- 庄子（"无用之用"，不是非此即彼）

开始？
```

圆桌跑 2-3 轮。每轮翰林院传递的不是发言全文，是**要点摘要**（这是信息隔离规矩）。你看每个人的完整发言，但他们互相看的是摘要。

每轮后翰林院说："稻盛说 X，诸葛说 Y，庄子说 Z。他们都同意 A，但在 B 点分歧。"

你可以随时：
- "让诸葛多说说"
- "庄子展开刚才那句"
- "我想再加一个人，牛顿 40 岁时怎么想这事的？"（翰林院启动牛顿 subagent，加入圆桌）

### 辩论（正面对抗 2 人）

针对真实冲突。两个立场鲜明的人交锋。

格式：
- 第 1 轮：A 立场 → 翰林院传 A 核心 → B 反驳
- 第 2 轮：A 再驳 → B 再驳
- 第 3 轮：双方 final statement
- 翰林院总结：核心论证、根本分歧、你可以从每一方学到什么

**真实例子**：

```
辩论：马斯克 vs 巴菲特
议题：40 岁时投入毕生积蓄做火箭公司，对还是错？

Round 1:
  马斯克: "人类文明的延续需要多行星。40 岁不是风险上限，
  是最后的机会。大部分创业者的问题是太害怕失败。"
  
  (翰林院传 "马斯克: 存在性目标 + 时机论"给巴菲特)
  
  巴菲特: "我从来不投我不懂的。火箭是你的使命，
  但把全部身家压上不是投资，是宗教。区别很重要。"

Round 2:
  ...

翰林院总结:
  根本分歧: 马斯克把风险看作"代价"，巴菲特把风险
  看作"消除目标"。这不是方法论差异，是对"什么是
  重要"的根本分歧。
  
  你可以从马斯克学: 不要让安全感决定人生方向
  你可以从巴菲特学: 承担风险 ≠ 毁掉基础
  
  你自己的判断?
```

辩论最适合当**你发现两个价值观在你心里打架**——让它们找两个代言人去打，你旁观。

## Step 5 · 中途切换

你在一对一贾伯斯聊了 10 分钟，发现他太激进了，想要更柔和的视角：

```
你：加个达·芬奇
```

翰林院启动达·芬奇 subagent，从一对一变成圆桌。贾伯斯和达·芬奇都收到你的话，但**看不到对方的完整发言**。

或者你完全受不了贾伯斯了：

```
你：切换到老子
```

翰林院结束贾伯斯 subagent（给他一句 parting word 的机会），启动老子。继续一对一。

---

## Step 6 · 结束仪式

你说"够了"/"谢谢"/"我想好了"：

翰林院执行**结束仪式**（必须，不能跳）：

### 1. Parting words

每个参与过的思想家说一句 final line（in character）：

```
💼 Jobs: "Now go ship something. You'll learn more
in a day of user feedback than a month of polish."

🎨 达·芬奇: "E non v'è cosa perfetta sotto il sole——
世间无完美之物。但未完成也可以是艺术。
你要分清哪些作品等待打磨，哪些等待世界。"
```

### 2. 翰林院自己的总结（不是任何角色）

```
🎋 翰林院 · 本次对话总结

📝 Journey:
你进来时说"不知道是追求质量还是在逃避评价"。
→ 贾伯斯戳你第一层：你在避免听用户说"不够好"
→ 你回击："但完美主义是真实的"
→ 达·芬奇加入后提出：有些作品是"等待世界"，
  有些是"等待打磨"——你要分类
→ 最后你说："我的 v1.6.3 其实是等待世界那类"

💡 Key insights (最有价值的是你说的):
- "我混淆了'未完成'和'不够好'"（这是全程最亮的一句）
- "把发布改成 ship-to-learn 可能能解锁我"

❓ Unresolved:
- "等待打磨"那一类作品你怎么标识？（没问完）
- 如果用户反馈很残酷你怎么办？（回避了）
```

## 翰林院 vs 六部议政 的区别

很多人混：

| 场景 | 去哪里 |
|------|--------|
| "我该接 offer 吗" | 六部议程（有具体选项、需评分、结果是决策） |
| "我要什么样的职业" | 翰林院（抽象、价值观、无选项对比） |
| "数据显示冲突" | 政事堂（COUNCIL）自动触发 |
| "我两个价值观打架" | 翰林院辩论模式 |
| "我想改善完美主义" | 翰林院（先搞清问题本质） |
| "做个 30 天戒完美主义计划" | 六部议程（执行计划） |

**思考 vs 决定** 的区别。翰林院处理前者。

## 让翰林院更有效的技巧

1. **不要隐藏真实状态** — 空话换空话。开场像写邮件"我想探讨工作生活平衡"就废了。说真话："老婆昨天哭了说我是室友"。
2. **诚实对抗** — 不同意贾伯斯的话就说不同意，别 performative 同意。对话才有价值。
3. **用正确的模式** — 想被肯定 → 不要来；困在自己框架 → 一对一尼采/苏格拉底；多元视角 → 圆桌；两价值观打架 → 辩论。
4. **用不在列表上的人** — 直接点名"Brené Brown 一对一"，翰林院会启动 subagent。

---

## 常见错误

1. **把翰林院当 Google** — "老子怎么看 AI" 不是对话。从你的困惑开始，不从思想家开始。
2. **期待清晰答案** — 几乎从不是"你该做 X"。如果走出来**问题变形了**（"我该不该发布" → "我害怕被评价"），这是成功。
3. **走完不看总结** — 翰林院总结是**你自己说过的最亮的话**。读一遍贴 journal 里。
4. **聊太久** — 45 分钟基本上限。超过会 perform，重复自己。见好就收。

## 一个完整对话的样子

```
(12 分钟聊，贾伯斯)
(8 分钟，加入达·芬奇变成圆桌)
(5 分钟，你说够了)
(2 分钟，结束仪式)
```

总共 27 分钟。journal 里写了 1500 字。SOUL 的"完美主义门槛"维度 evidence +1。

第二天上朝，简报里：

```
📝 昨晚翰林院对话总结:
  "我混淆了'未完成'和'不够好'"
  
  今日是否要把这个洞察应用到 v1.6.3 的发布决策里？
```

对话的**影响延续到下次**。这就是归档存在的意义。
