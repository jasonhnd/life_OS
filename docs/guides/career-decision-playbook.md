# 职业决策手册 · 离职 / 跳槽 / 创业

职业决策是 Life OS 里最高频的"重度议程"类型。`scene-configs.md` 第 1 个场景专门为它设计。

本文告诉你：场景 #1 的六部配置、门下省典型的担忧（尤其"逃避 vs 追寻"）、怎么算 finance runway、SOUL 一致性如何检查、最后怎么读六部报告做决定。

不同于 `your-first-decision.md`（端到端流程的技术展示），本篇是**专精**——你真面对职业决定时该怎么走。

---

## 场景 #1 的标准配置

六部全上，各自 focus:

```
财务 → 户部 (FINANCE)
技能 → 礼部 (GROWTH) 
人脉 → 吏部 (PEOPLE)
执行 → 兵部 (EXECUTION)
风控 → 刑部 (GOVERNANCE)
生活影响 → 工部 (INFRA)

门下省 focus: "你在逃避什么, 还是在追求什么? 家人怎么想?"
```

六部全上是因为职业决定**涉及生活所有维度**。省任何一部都会漏掉关键评估。

## 第一步 · 意图澄清（独特于职业决定）

丞相对职业决定有专门的深度澄清。**至少 3 轮**。

**第 1 轮 · 分类**：

```
你说"创业"是:
a) 有具体想法已验证
b) 有方向但未验证
c) 没想法，离职换空间
d) 有合伙人/机会等

"离职"是明天 / 几周 / 几月后?
```

你选 `c + 几月后`，丞相标红：

```
⚠️ "无想法 + 换空间" 高风险:
风险不是"创业失败"，是"空间填不满"
这类决定 50% 变成"闲置 6 月后找新工作"

第 2 轮: 工作的痛具体是什么? (不要"没意思")
```

**第 2 轮 · 痛点**：你说"周会全是虚无讨论"。丞相：

```
你提到不想做什么，但没说想做什么 — 这是逃避信号。

第 3 轮最后一问:
如果现职不变但周会消失 + 30% 时间自由 — 还要离职吗?
```

你：`不要` → 丞相识别核心不是离职是缺自主；`还要` → 得到真实追寻 pull，才升堂。

## 门下省典型担忧：逃避 vs 追寻

门下省的 emotional dimension 对职业决定最严。**明确的 10/10/10**：

```
🔍 门下省 · 情感维度审查

🔎 情绪信号: "浪费" 5 次 + 无具体离职后生活 + deadline 假急
🎯 10/10/10:
- 10 分钟: 是昨天糟糕会导致的, 还是持续 2 周以上?
- 10 月: 离职后 3 月无想法, 刷招聘网站 — 画面让你怕还是平?
- 10 年: 叙事里你要"勇敢跳出去"还是"没被逼急冲动"?

🧠 逃避 vs 追寻: 3:1
如升堂, 这比例会 propagate 到所有 domain 分析中
```

**这是门下省在保护你。** 6 月后回看你会感谢这一刀。

### 议程变形

门下省判"逃避 > 追寻" → 中书省调整 Subject：

```
Original: 离职创业决定
Revised: 现职 vs 离职 vs 第三条路

1. 三方案财务 → 户部 (A 留 B 离 C 改)
2. 现职改造可能性 → 兵部 (转 IC / 缩职责)
3. 创业准备度 → 礼部 + 兵部
4. 风控 → 刑部
5. 关系 → 吏部
6. 基建 → 工部
```

**不是"要不要离职"，是"找到最小 regret 的路径"**。

## 户部 · Finance Runway 计算

职业决定里户部最硬核的输出是 **runway**——你能支撑多久。

```
💰 户部 · Runway 分析

输入:
- 流动资产 L / 半流动 H
- 离职后月 burn: 必要支出(剥离办公相关) + 自由 burn(工具/咖啡店/应急 buffer) = B

3 种 Runway:
- Bare bones: L / B
- Extended:   (L + 0.5H) / B
- All-in:     (L + H) / B

决定基线: 用 Bare bones — 不 count on "能回本"

解读:
- < 6 月   不建议离职
- 6-12 月  有条件 (必须有具体计划)
- 12-24 月 可以离职探索
- > 24 月  真正不担心

示例: Bare bones 9.2 月 → 评分 7/10
```

**注意**：配偶收入是加分项，不作为基础假设。单人 runway 是基线。

## SOUL 一致性检查

门下省会引用你的 SOUL 来做**价值观对齐审查**。职业决定经常暴露 SOUL 内部矛盾：

```
🔮 SOUL Reference

core · Core (must consider):
  · 长期主义 (0.82) — 离职创业是长期投资吗?
    → 支持 (3-5 年跨度) 
  · 风险偏好 中等偏高 (0.72) — 这个风险你能承受吗?
    → 支持 (边界内)
    
secondary · Relevant:
  · 决策时对家庭意见的权重 (0.58) [strong match]
    → 挑战 — 用户未谈配偶, 可能在忽视此维度
  · 避免低信号活动 (0.41) [weak match]
    → 支持 — 会议多是低信号

Integrated:
  SOUL 兼容性: 🟡 mixed
  Focus callout: 3/4 维度支持但"家庭意见权重"未兑现, 
  门下省要求先完成家庭对话

⚠️ SOUL CONFLICT:
  "决策时对家庭意见的权重" 是 secondary 但 strong match,
  用户未与配偶谈过 — 这与 SOUL 表达的价值观矛盾.
  建议: 议程加入 "家庭对话" 前置任务.
```

**SOUL 帮你看见你没看见的**——职业决定里你总容易聚焦"offer 好不好"而忘了"我这样选择对不对"。

## 六部报告（典型输出）

假设议程跑完（用调整版 Subject "三条路抉择"）：

### 户部

```
💰 户部

方案 A (留职): runway ∞, 收入持续增长 5-8%/年
方案 B (离职创业): runway 9.2 月, 0 收入可能 
方案 C (现职改造): runway ∞ + 可能减薪 10-20%

推荐: C (风险最低, 自由度 get)
```

### 吏部

```
👥 吏部
A: 关系成本 0 但已有"不满"需处理 
B: 配偶未谈，关系风险高
C: 上司关系的 leverage test
关键缺失: 配偶对话 (B 和 C 都依赖)
推荐: 任何方案前必须谈配偶
```

### 礼部

```
📖 礼部
技能盘点: PM 7 年强，销售弱，BP 弱
B (创业) 缺: 销售(致命) + BP(可学) + 技术(合伙)
A/C 不需新技能
推荐: 先留职 + 业余验证 (pre-B)
```

### 兵部

```
⚔️ 兵部
C: 公司有 3 位同级转 IC, 成功 2。3 月季度评估是谈判窗口
B: 无 MVP 无客户访谈，需 3+ 月准备
A: 接受 + 优化工作内外比
推荐: C 先试 → 失败考虑 B
```

### 刑部

```
⚖️ 刑部
最坏情况:
A: 2 年后后悔"没试过" → 可挽回
B: 18 月失败 → 简历 gap 需解释
C: 谈判失败 → 被动 → B
合同: 无竞业，60 天 notice，未付 bonus 需放弃
推荐: C → B → A
```

### 工部

```
🏗️ 工部
A: 节奏不变，情绪压力持续
B: 节奏彻底变，需自律（500字完成率 12% 是警示）
C: 最少 disruption
推荐: C 风险最小
```

## 门下省最终审议

```
🔍 门下省 · 最终审议

部门评分:
户部 C > A > B (7/6/3)
吏部 必须先谈 (all 4 前置)
礼部 B 无准备度, C 最低门槛 (5/4/3)
兵部 C 有路径, B 无路径 (7/6/3)
刑部 C → B → A (6/5/3)
工部 C 风险小 (7/5/3)

共识: C (现职改造) 是当前最优路径
分歧: B 的可能性 (礼部和兵部都指出 B 需要前置准备)

得分差: 7 - 3 = 4 → 触发议政 但 domain 实际共识度高
→ 议政会显示 "low-intensity", 快速结束

🔮 SOUL Re-check:
- "长期主义" ↔ C (短中期路径) → 支持
- "风险偏好" ↔ C → 支持 (比 B 低风险)
- "家庭意见权重" ↔ 未谈 → ⚠️ 仍然挑战

10/10/10 二次:
- 10 月后 C 失败 → 你会回到 B, 届时有更多准备
- 10 年后 C 路径可能是最"优雅"的—— 渐进非突变

✅ Approved (conditional)

条件:
1. 先与配偶 90 分钟对话 (吏部框架)
2. 谈判前准备 specific proposal (兵部 brief)
3. 若 C 谈判失败, 进入 6 月 cooling period 再议 B
```

## 谏官（特别关键）

职业决定后，谏官会做**深度行为审查**：

```
💬 谏官 · 行为审查

📊 Data: 历史决策 + user-patterns.md + SOUL.md

观察:

1. 模式识别:
   你这是第 2 次讨论 "离职" 话题 (上次 8 月). 
   两次都没有真实的"追寻"目标, 都是痛点驱动.
   
   这不是批评 — 这是"逃避驱动"的典型模式. 
   它的风险是: 如果离职, 你找到的新工作大概率
   也会在 6-12 月出现同样的痛点, 因为痛点源自你
   和工作的 framework 关系, 不是特定公司.

2. 🔮 SOUL 候选 (新维度):
   - Dimension: "应对不满的默认方式"
   - What IS: 倾向于"离开不满"而非"改造不满"
   - Evidence: 2 次离职想法 + 其他 3 次类似 pattern

3. 📈 Progress: 
   上次你直接升堂没问"改造现职", 这次议程变形加入了 C —
   你允许系统质疑你的 framing, 这是重要进步.

4. 建议:
   如果 C 谈判成功但 3 月后仍不满 — 那可能真正的问题是
   你对"不满"的关系需要重新理解. 届时考虑翰林院对话,
   推荐: 爱比克泰德 (dichotomy of control) 或 Jobs 
   (focus by saying no).
```

## 如何读六部报告

六份报告的正确读法：

1. **先看奏折 Total Score** — 7+ 高概率、5-6 有条件、<5 更可能不行
2. **Must Address 不可跳** — 跳了议程白做
3. **找评分最低的部** — 低分是系统在提醒你忽视的维度
4. **议政是共识 vs 分歧的显化** — 看看自己的价值观
5. **谏官可能比奏折更重要** — 奏折讲"这次"，谏官讲"你"
6. **Action 要 24 小时消化再执行** — 不要当场做

## 决策后的节奏

- **周 1**：前置任务（配偶对话 + proposal 准备）
- **周 2-4**：执行主路径
- **月 3**：Q1 review — 看 OFR + 和议程时预期对比
- **月 6**：深度复盘 — 实际 vs 预期 gap 写到 user-patterns

## 三种常见模式

**模式 A 逃避但装追寻**：开场"我想创业"，无具体想法。系统议程变形，探索现职改造。改造路径 70% 1 年满意度提升；硬离职 50% 后悔。

**模式 B 追寻但无准备**：有想法但无 MVP / 客户 / 销售能力。系统建议保留现职 + weekend 验证，6-12 月后再决。

**模式 C 充分准备**：有 offer / 联创 + 财务 + 配偶对齐。议程快速过，focus 在合同 / 时间表（logistic 而非 "该不该"）。

---

## 反模式

1. **跳过配偶对话** — SOUL "家庭意见权重"最常违反。对话是前置不是后置。
2. **用 offer 做 leverage** — 如果公司不接你被迫离开，新公司可能也不完美。leverage 应基于"你值得"而非"你要走"。
3. **乐观预期 runway** — 假设"6 月有收入"。大部分创业 12-18 月才稳定。用 bare bones + 0 收入假设。
4. **忽略机会成本** — "创业不要钱"忽略了 2X 的年薪成本。runway 包含机会成本。
