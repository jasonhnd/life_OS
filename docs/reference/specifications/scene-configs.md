# 标准场景推荐配置（Scene Configs）

本文件列出 12 类常见决策场景的推荐领域激活配置和 REVIEWER 重点关注问题。PLANNER 和 ROUTER 在派遣时参考这些配置。

---

## 1. 职业转换（辞职/跳槽/创业）

**领域激活**：
- 财务 → FINANCE
- 技能 → GROWTH
- 人脉 → PEOPLE
- 执行 → EXECUTION
- 风险 → GOVERNANCE
- 生活影响 → INFRA

**REVIEWER 关注**：
- 是在逃避还是在追求？
- 家人怎么想？

---

## 2. 投资决策

**领域激活**：
- 财务 → FINANCE
- 市场 → EXECUTION
- 风险 → GOVERNANCE
- 人（创始人/团队）→ PEOPLE

**REVIEWER 关注**：
- FOMO 还是理性？
- 全损能不能扛得住？

---

## 3. 搬迁/搬家

**领域激活**：
- 财务 → FINANCE
- 关系 → PEOPLE
- 基础设施 → INFRA
- 合规 → GOVERNANCE
- 文化 → GROWTH

**REVIEWER 关注**：
- 你真的了解目的地吗？
- 回头的成本？

---

## 4. 年度目标规划

**领域激活**：
- 关系 → PEOPLE
- 财务 → FINANCE
- 学习 → GROWTH
- 执行 → EXECUTION
- 纠偏 → GOVERNANCE
- 健康 → INFRA

**REVIEWER 关注**：
- 目标太多吗？
- 可衡量吗？
- 与价值观对齐吗？

---

## 5. 产品/创业决策

**领域激活**：
- 市场 → EXECUTION
- 财务 → FINANCE
- 团队 → PEOPLE
- 合规 → GOVERNANCE
- 基础设施 → INFRA
- 品牌 → GROWTH

**REVIEWER 关注**：
- 解决的是真实痛点吗？
- 你是合适的人吗？

---

## 6. 大额购买

**领域激活**：
- 财务 → FINANCE
- 实用性 → EXECUTION
- 风险 → GOVERNANCE

**REVIEWER 关注**：
- 需要还是想要？
- 一个月后你还想买吗？

---

## 7. 关系决策

**领域激活**：
- 关系 → PEOPLE
- 情绪 → INFRA（Fitness）
- 风险 → GOVERNANCE
- 社交 → GROWTH

**REVIEWER 关注**：
- 是不是在带偏见看对方？

---

## 8. 复盘（Review）

**交给 RETROSPECTIVE**。日/周/月/季/年。

---

## 9. 健康管理

**领域激活**：
- 身体 → INFRA
- 时间 → EXECUTION
- 成本 → FINANCE
- 纪律 → GOVERNANCE

---

## 10. 学习计划

**领域激活**：
- 内容 → GROWTH
- 执行 → EXECUTION
- 成本 → FINANCE
- 人脉 → PEOPLE

---

## 11. 时间管理与精力优化

**领域激活**：
- 精力 → EXECUTION（Logistics）
- 排期 → EXECUTION（Operations）
- 工具 → EXECUTION（Equipment）
- 成本 → FINANCE
- 习惯 → GOVERNANCE
- 节奏 → INFRA

**REVIEWER 关注**：
- 真的没时间，还是在回避什么？
- 被砍掉的事情，真的有人在乎吗？
- 上一次无愧疚感的休息是什么时候？

---

## 12. 家庭重大决策

**领域激活**：
- 财务 → FINANCE（Reserves + Spending）
- 分工 → PEOPLE（Allocation）
- 动态 → PEOPLE（Evaluation）
- 执行 → EXECUTION
- 情报 → EXECUTION
- 法律 → GOVERNANCE
- 应急 → GOVERNANCE
- 身心 → INFRA
- 住房 → INFRA

**REVIEWER 关注**：
- 谁的声音没有被听到？
- 最坏情况下你能活下来吗？
- 五年后你会后悔做了，还是后悔没做？

---

## 使用说明

- 这 12 个配置是 **推荐起点**，不是强制清单
- PLANNER 可以根据用户的实际需求增减领域
- ROUTER 在把用户请求归类到某个场景后，把该场景的 REVIEWER 关注问题传给 REVIEWER，作为审查时的"专属守则"
- 场景归类不唯一——"辞职去创业" 同时涉及场景 1（职业转换）和场景 5（产品/创业），可以合并激活领域
- 若用户的请求不匹配任何场景，PLANNER 自由判断，无需强行套用
