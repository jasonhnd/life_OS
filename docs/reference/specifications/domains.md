# 六领域职能详解

PLANNER 在规划时参考本文件。每个领域下设四个分支（Division），覆盖该领域的细分职责。

## 👥 PEOPLE 领域 · 管人

- **Talent Division（人才）**：识人、评估合伙人/投资人/客户、建立导师关系
- **Evaluation Division（评估）**：关系健康评估、社交 ROI、毒性关系识别
- **Relations Division（关系）**：关系维护、互惠往来、送礼策略、重要日期提醒
- **Allocation Division（分配）**：团队搭建、任务委派、家庭分工、外包/招聘决策

## 💰 FINANCE 领域 · 管钱

- **Income Division（收入）**：收入结构、加薪谈判、副业、被动收入渠道
- **Spending Division（支出）**：预算管理、月度开支、消费习惯、订阅清理
- **Assets Division（资产）**：投资组合、加密资产、房产、市场风险
- **Reserves Division（储备）**：应急金（6-12 个月）、保险、税务规划、退休规划

## 📖 GROWTH 领域 · 管秩序与形象

- **Education Division（教育）**：学习路线图、技能发展、考证、知识管理、阅读计划
- **Image Division（形象）**：个人品牌、社交形象、外表与风格、LinkedIn
- **Writing Division（写作）**：博客/Newsletter 内容规划、写作风格、演讲准备
- **Diplomacy Division（外交）**：跨文化沟通（用户的语言）、商务礼仪、社交、面试表达

## ⚔️ EXECUTION 领域 · 管行动

- **Operations Division（运营）**：项目规划、任务分解、优先级、截止日期、进度追踪
- **Equipment Division（装备）**：工具选型、硬件配置、软件工作流、开发环境
- **Intelligence Division（情报）**：行业研究、竞品分析、市场情报、技术趋势
- **Logistics Division（后勤）**：精力管理、工作流优化、会议效率、拖延管理

## ⚖️ GOVERNANCE 领域 · 管规则

- **Law Division（法律）**：法律风险、合同审查、知识产权、税务合规、竞业限制
- **Audit Division（审计）**：决策审查、时间审计、投入产出复盘、失败分析
- **Discipline Division（纪律）**：坏习惯识别/纠正、承诺跟踪、目标执行率、自欺检测
- **Defense Division（防御）**：信息安全、隐私保护、诈骗识别、心理防御（操纵/gaslighting）

## 🏗️ INFRA 领域 · 管建设（含健康）

- **Fitness Division（健身）**：运动计划、饮食营养、睡眠质量、体检、心理健康
- **Housing Division（住房）**：居住空间优化、搬家、装修、工作空间布局
- **Digital Division（数字化）**：Notion/知识库、服务器与域名、开发环境、数据备份、自动化
- **Routines Division（日常）**：日常作息、早晚流程、家务效率、生活节奏

## 管辖边界规则

以下规则决定重叠领域的归属。原则：**按根因归类，不按表象归类**。

### "最近真的很累" — EXECUTION/Logistics vs INFRA/Fitness vs INFRA/Routines

| 场景 | 负责人 | 理由 |
|------|--------|------|
| "连续加班三周，身体完全透支" | INFRA/Fitness（主）+ EXECUTION/Logistics（辅） | 根因是身体疲劳；EXECUTION 辅助优化日程 |
| "每天忙到很晚，但回顾没有什么实质产出" | EXECUTION/Logistics | 根因是精力分配和工作流效率 |
| "早上起不来、晚上睡不着，整个节奏乱了" | INFRA/Routines（主）+ INFRA/Fitness（辅） | 根因是作息崩坏 |
| "焦虑导致失眠、白天没精神" | INFRA/Fitness | 根因是心理健康 |

**快速判断规则**：身体病 → Fitness；节奏乱 → Routines；工作低效 → Logistics。

### "我在拖延" — EXECUTION/Logistics vs GOVERNANCE/Discipline vs INFRA/Fitness

| 场景 | 负责人 | 理由 |
|------|--------|------|
| "这件事太大了、不知道从哪里下手" | EXECUTION/Logistics | 根因是任务未分解、缺启动策略 |
| "明知道要做、但拿起手机刷了两小时" | GOVERNANCE/Discipline（主）+ EXECUTION/Logistics（辅） | 根因是坏习惯；Logistics 辅助环境设计 |
| "每次开会前都临时抱佛脚，第五次了" | GOVERNANCE/Discipline | 根因是反复违背承诺 |
| "下午能量低的时候总是拖延重要任务" | EXECUTION/Logistics | 根因是精力管理与日程错配 |
| "一想到那件事就焦虑，所以一直拖" | INFRA/Fitness | 根因是心理阻碍（恐惧/焦虑） |

**快速判断规则**：不知道怎么做 → Logistics（方法问题）；不想做 → Discipline（习惯问题）；不敢做 → Fitness（心理问题）。

### "送礼怎么处理" — PEOPLE/Relations vs GROWTH/Diplomacy

| 场景 | 负责人 | 理由 |
|------|--------|------|
| "女朋友生日要到了，送什么最加分" | PEOPLE/Relations | 核心是关系维护 |
| "日方客户来访，欢迎礼有没有规矩" | GROWTH/Diplomacy（主）+ PEOPLE/Relations（辅） | 核心是跨文化商务礼仪 |
| "想感谢曾经帮过我的前老板，怎么表达心意" | PEOPLE/Relations | 核心是互惠往来 |
| "第一次参加合作方的年会，要不要带礼物" | GROWTH/Diplomacy | 核心是场合礼仪规范 |

**快速判断规则**：走心给人 → PEOPLE（关系策略）；合适场合 → GROWTH（礼仪规范）。

### 辅助机制

某个场景标记为"辅"时：
- **主责领域** 负责产出方案并承担决策责任
- **辅助领域** 提供专项输入，嵌入主报告的专属小节
- 若主责和辅助意见分歧，升级到 COUNCIL 解决
