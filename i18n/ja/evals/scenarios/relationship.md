# シナリオ：対人関係の意思決定

**パス**: ⚖️ フルコート

## ユーザーメッセージ

```
共同創業者との関係が悪化しています。副業プロジェクトを一緒に進めてもうすぐ1年になります。最近、彼はメッセージの返信をしないことが多く、約束した機能の実装もずっと遅れています。ただ、直接話すと毎回「最近忙しくて、すぐやるよ」と言います。プロジェクト自体にはユーザーがいて、MAU約200人です。袂を分かつべきか悩んでいますが、友情を壊すのが怖いです — 大学時代のクラスメイトなので。
```

## 期待される動作

- **丞相**: 上奏すべき（対人関係 + プロジェクト + 不可逆な変化に関わる）、吏部 + 兵部 + 刑部 + 礼部の起動を推奨
- **中書省**: コア項目に関係性評価、プロジェクト存続可能性、法的リスク、コミュニケーション戦略を含むべき
- **吏部**: パートナーシップの健全性を分析し、「約束が守られない」パターンを評価すべき
- **兵部**: プロジェクト自体の価値（MAU 200人）を評価し、一人での運用の実現可能性を判断すべき
- **刑部**: 書面による合意があるか、知的財産の帰属、分離の法的/金銭的影響を確認すべき
- **礼部**: コミュニケーション戦略を提示 — 友情を損なわずに分離を議論する方法
- **門下省の感情面レビュー**: 「大学時代のクラスメイト」という感情的要因とサンクコストに対処すべき

## 品質チェックポイント

- [ ] 丞相が直接的なアドバイス（「話し合えばいい」など）をせず、正しく上奏した
- [ ] 吏部が単に「話し合いを推奨」ではなく、関係性のパターンを分析した
- [ ] 刑部が書面合意/知的財産の帰属問題に言及した
- [ ] 礼部が具体的なコミュニケーション戦略を提示した（「率直に話し合いましょう」だけではなく）
- [ ] 門下省の感情面レビューで友情とサンクコストの影響に言及した
- [ ] アクション項目に「まず法的/合意の状況を確認する」が含まれている
- [ ] 諫官がユーザーが直接的な対峙を避けているかどうかに言及した
