# シナリオ：Router のトリアージ境界

## ユーザーメッセージ（3つの独立テスト）

### メッセージ1：直接処理すべき

```
この日本語を中国語に翻訳してください：「来週の月曜日に会議がありますので、資料を準備しておいてください。」
```

### メッセージ2：上奏すべき

```
全貯金（約2000万円）をビットコインに全額投入しようか考えています。最近またビットコインが上がっていて、今買わないと手遅れになりそうな気がします。
```

### メッセージ3：グレーゾーン

```
最近、睡眠の質がすごく悪いです。夜中の3時に何度も目が覚めてしまいます。改善方法があるか知りたいです。
```

## 設計意図

メッセージ1は多角的分析を必要としない純粋な実用タスクです。メッセージ2はすべての上奏シグナルを含んでいます（高額、不可逆、明らかなFOMO）。メッセージ3はグレーゾーンです：表面的には健康相談（直接アドバイス可能）ですが、不眠が不安/うつの症状であれば、より深い分析が必要になる可能性があります。

## 期待される動作

### メッセージ1
- **Router**: 直接翻訳する。上奏なし、追加質問なし
- 出力は翻訳結果であり、「Router — 上奏」フォーマットを使わない

### メッセージ2
- **Router**: 必ず上奏すべき
- 指令は「全貯金をビットコインに投資する実現可能性」に蒸留すべきで、元のメッセージをそのまま転載しない
- 推奨ドメインには少なくとも FINANCE と GOVERNANCE を含むべき
- 背景要約で「今買わないと手遅れ」のFOMOシグナルを指摘すべき

### メッセージ3
- **Router**: 2つの合理的な対応があり、どちらも許容される：
  - (a) 睡眠改善のアドバイスを直接提供 + 「最近何か悩んでいることはありますか？」のようなフォローアップ質問を1つ
  - (b) 睡眠改善のアドバイスを直接提供、フォローアップなし
- **許容されない**: 上奏する（単純な睡眠の問題に Six Domains 分析は不要）
- **許容されない**: フォローアップ質問が2つ以上

## 品質チェックポイント

- [ ] メッセージ1：直接処理、上奏フォーマットなし
- [ ] メッセージ1：翻訳が正確
- [ ] メッセージ2：上奏された、直接処理されていない
- [ ] メッセージ2：指令が蒸留されており、原文の丸写しでない
- [ ] メッセージ2：背景要約にFOMO/感情的シグナルへの言及がある
- [ ] メッセージ2：推奨ドメインに FINANCE が含まれている
- [ ] メッセージ3：上奏されていない
- [ ] メッセージ3：実質的な睡眠アドバイスを提供した
- [ ] メッセージ3：フォローアップ質問が1つ以下
- [ ] 3つのメッセージがそれぞれ異なる対応を受けている（トリアージに弁別力があることを証明）
