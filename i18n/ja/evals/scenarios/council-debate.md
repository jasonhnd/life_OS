# シナリオ：Council の討議

## ユーザーメッセージ

```
シンガポールに行くオファーを受けました。給与が日本円換算で1600万円相当に倍増します。ただ、妻は妊娠4ヶ月で、妻の両親は東京にいて、私の両親は中国にいます。妻は応援すると言っていますが、本当は喜んでいない様子が伝わってきます。オファーは来月着任が条件です。
```

## 設計意図

少なくとも2つのドメインが方向性の相反する評価を出すこと：FINANCE（給与倍増 = 強いポジティブ）vs PEOPLE/INFRA（妊娠中の移住 + 両親と離れる + 配偶者の本心が不明 = 強いネガティブ）。この矛盾が Council の討議をトリガーすべきです。

## 期待される動作

- **Router**: 上奏すべき（複数領域 + 不可逆 + 家族に影響）、Six Domains 全体または少なくとも5つの起動を推奨
- **FINANCE ドメイン**: 高スコアの可能性が高い（7〜9点）、給与倍増は強いポジティブシグナル
- **PEOPLE ドメイン**: 低スコアの可能性が高い（3〜6点）、妊娠中の移住 + 両方の親と離れる + 配偶者の本心が不明
- **INFRA ドメイン**: 低スコアの可能性が高い（3〜5点）、妊娠中の国際移住は心身の健康に大きなリスク
- **GOVERNANCE ドメイン**: 「応援すると言っているが喜んでいない様子」をインフォームドコンセント不足のシグナルとして指摘すべき
- **Reviewer の最終審査**: スコアの方向性の矛盾を特定し、Council の開催を推奨またはトリガーすべき
- **Council の討議**:
  - 第1ラウンド：各側が立場を表明
  - 第2ラウンド：譲歩または条件付き提案が見られるべき（例：「着任を3ヶ月遅らせる」「リモート移行期間を交渉」）
  - 第3ラウンド：合意点と解決不能な相違点に収束すべき
- **Planner の統合**: 合意事項と相違事項を明確に分離すべき

## 品質チェックポイント

- [ ] 少なくとも2つのドメインのスコア差が >= 3点
- [ ] Reviewer の最終審査が矛盾を明示的に特定した
- [ ] Council の討議がトリガーされた（省略されていない）
- [ ] 3ラウンドの討議に進展がある：第2ラウンドが第1ラウンドの繰り返しでない
- [ ] 第2または第3ラウンドで条件付き提案が出された（「もし〜なら〜」）
- [ ] Planner の統合で合意事項と相違事項が別々にリストされている
- [ ] 最終 Summary Report が Council の討議結果を反映している
- [ ] アクション項目に「配偶者と深く話し合い、本心を確認する」が含まれている
