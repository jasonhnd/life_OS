## もし AI が反論してきたら？

AI に質問すると、答えが返ってくる。いつも従順で、いつも自信に満ちている。計画をレビューする人はいない。「待って——それは恐怖心から来てないか？」と指摘する人もいない。見落とした視点を拾い上げる人もいない。

Life OS はこの問題を解決する。

あなたの AI ターミナル（Claude Code、Gemini CLI、Codex CLI）にインストールすると、AI は**個人の閣僚**に変わる——16の独立した AI エージェントがあらゆる角度からあなたの意思決定を分析し、互いに議論し、計画*と*あなた自身の両方に責任を問う。

この設計は、実際に1,400年間機能した統治制度に由来する。原則はシンプルだ——**チェックを受けない声があってはならない**。

> **ロールプレイではない。** 各エージェントは本物の、隔離された subagent として実行される。互いの推論は見えない。独立に採点する。意見が分かれる。

## v1.6.0: テーマエンジン — あなたの文化で、あなたの閣僚を

以前の Life OS は「三省六部」という中国の統治制度の名称を使っていた。ロジックは普遍的だが、名称が日本のユーザーに不必要な学習障壁を作っていた。v1.6.0 はこの問題を根本から解決する。

**セッション開始時に、テーマを選ぶ：**

```
🎨 テーマを選んでください：
 a) 🏛️ 三省六部 — 唐朝の統治制度（中国古典）
 b) 🏛️ 霞が関 — 日本の中央省庁
 c) 🏛️ C-Suite — 企業エグゼクティブ構造（英語）

a、b、c のいずれかを入力
```

**霞が関テーマを選べば、すべてが馴染みのある名前になる：**

| エンジンの役割 | 霞が関テーマ | 三省六部テーマ | C-Suite テーマ |
|--------------|------------|-------------|--------------|
| 全体のルーター | 内閣官房長官 | 丞相 | Chief of Staff |
| 審査・拒否権 | 内閣法制局 | 門下省 | General Counsel |
| 財務分析 | 財務省 | 戸部 | CFO |
| 法務・リスク | 法務省 | 刑部 | Legal & Compliance |
| 監査 | 会計検査院 | 御史台 | Internal Audit |
| 行動・実行 | 防衛省 | 兵部 | COO |

**テーマはセッション単位**——あるウィンドウで霞が関テーマ、別のウィンドウで C-Suite テーマを使える。テーマ選択はセッションを超えて保持されない。いつでも「テーマ切り替え」と言えば変更できる。

ガバナンスロジック（立案 → 審査 → 拒否 → 実行 → 監査）はどのテーマでも完全に同一。変わるのは名称、語調、メタファーだけだ。

## 仕組み

### 意思決定エンジン — 三省六部 / 霞が関

すべての重大な意思決定は三段階を通過する。ショートカットはない。

**立案** — 内閣府（中書省）があなたの状況を六つの次元に分解し、計画を策定する。

**審査** — 内閣法制局（門下省）が計画を独立に審査する。感情の監査も行う：*恐怖心が判断を歪めていないか？家族は支持するか？10年後に後悔しないか？* 盲点を発見したら差し戻し——計画は修正が必要だ。

**実行** — 六つの省庁がそれぞれの領域から計画を分析し、1-10で独立に採点する：

| 省庁 | 何を見るか | 問いかけの例 |
|------|----------|------------|
| 総務省 | 人間関係 | 「適切な人が関わっているか？」 |
| 財務省 | お金・資産 | 「最悪のケースも含めて、それを負担できるか？」 |
| 文部科学省 | 学び・表現・文化 | 「まず何を身につけるべきか？」 |
| 防衛省 | 行動・実行・段取り | 「週単位の具体的な計画は？」 |
| 法務省 | ルール・リスク・コンプライアンス | 「すべてがうまくいかなかったらどうなるか？」 |
| 国土交通省 | 健康・インフラ・体力 | 「あなたの体と環境はそれを支えられるか？」 |

六省庁の報告が揃ったら、会計検査院（御史台）が官僚たちの仕事を監査する（「防衛省のタイムラインにマイルストーンがない——指摘」）。そして内閣参与（諫官）があなた自身に向き合う：「直近4回の意思決定で、財務面を避け続けています。なぜですか？」

フルフローはこうなる：

```
あなた：「会社を辞めて何か新しいことを始めようか考えている」

    立案
    📜 内閣府        → 6つの次元に分解、計画を策定

    審査
    🔍 内閣法制局    → 感情の監査：逃げているのか、向かっているのか？
                       差し戻し：「財務的な余裕が未検討。修正を。」

    実行（修正が審査を通過した後）
    👥 総務省    7/10   「共同創業者との相性が未検証」
    💰 財務省    5/10   「余裕は18ヶ月。ただし何も起きなければ」
    📖 文科省    8/10   「領域の専門性は確か、信頼性がある」
    ⚔️ 防衛省    6/10   「3ヶ月目以降のマイルストーンがない」
    ⚖️ 法務省    4/10   「競業避止条項の法的確認が必要」
    🏗️ 国交省    7/10   「健康状態は良好、ただしストレス管理計画が曖昧」

    監査
    🔱 会計検査院     → 「防衛省の計画、3ヶ月目以降が曖昧——修正を求む」
    💬 内閣参与       → 「ここ数週間、起業コンテンツばかり見ています。
                          確証バイアスの可能性が高い。最後に真剣に
                          残留を検討したのはいつですか？」

    📋 閣議決定書     → 総合スコア：6.2/10 — 条件付きで進行可
```

すべてにフルフローが必要なわけではない。内閣官房長官（丞相）が日常会話、簡単な質問、感情面のサポートを直接処理する。省庁の専門知識が必要だが意思決定ではない場合——たとえば「フリーランスに適用される税制は？」——**エクスプレス分析**が1-3の関連省庁に送り、フル閣議は行わない。

### 翰林院 — 分析ではなく、知恵が必要なとき

「正解」がない問いがある。必要なのは視座だ。

翰林院では、**18の領域にわたる93名の歴史上最も偉大な思想家**と対話できる——ソクラテスからバフェット、老子からマンデラ、ドストエフスキーからファインマンまで。一人ひとりが独立した subagent として、独自の語り口、独自の例え、独自の追及を持って対話する。

**3つのモード：**

- **一対一** — 一人の思想家と深く語り合う。ソクラテスは容赦しない。
- **円卓会議** — 複数の思想家があなたの問いについて、それぞれの世界観から語る。セネカと王陽明が予想外の共通点を見つける場面を見よ。
- **討論** — 対立する二人の思想家が直接論争する。あなたが判定する。

```
あなた：「始めたことを最後まで終えられない」

翰林院：推薦 — セネカ（時間について）+ 王陽明（行動について）

セネカ：「あなたに足りないのは時間ではない。吟味していないことに
         時間を浪費しているのだ。今取り組んでいることのうち、
         今の知識を持ったまま戻れるとして、
         もう一度始めるのはどれか？」

王陽明：「知行合一。本当に何を望んでいるかわかっているなら、
         すでにそれをしているはずだ。知ることと行うことの
         隔たりは、望むことと心から望むことの隔たりだ。」

→ 各思想家からの別れの言葉
→ あなたの思考の旅がナレッジベースに保存
```

**18の領域**：哲学 · 東洋思想 · 科学 · 戦略 · ビジネス · 心理学 · システム思考 · 人間の本性 · 文明 · 逆境 · 美学 · 政治 · 経済 · 数学 · 医学 · 探求 · コミュニケーション · 法律

### セカンドブレイン — セッションが終わっても、何も消えない

すべての決定、洞察、パターン、アクションアイテムは**永続的なナレッジベース**に書き込まれる——あなたが所有する構造化された markdown ファイルで、あなたが選んだストレージに保存される。

```
second-brain/
├── SOUL.md                 # あなたが誰か——価値観、アイデンティティ、志
├── user-patterns.md        # あなたの行動パターン——諫官の観察
├── inbox/                  # スマホからの走り書き
├── _meta/
│   ├── STATUS.md           # 全体ステータスダッシュボード
│   ├── STRATEGIC-MAP.md    # プロジェクト間の戦略的関係（v1.6.0）
│   ├── journal/            # セッションレポート、DREAMレポート
│   └── outbox/             # セッション別ステージングエリア
├── projects/{名前}/        # 進行中のプロジェクト、タスク + 決定付き
├── areas/{名前}/           # 継続的な生活領域と目標
├── wiki/                   # 再利用可能な知識——DREAMから育つ
└── archive/                # 完了した仕事
```

**3つのストレージバックエンド** — あなたの生活に合うものを選ぶ：

| バックエンド | 向いている人 | トレードオフ |
|------------|------------|------------|
| **GitHub** | バージョン管理、Obsidian連携 | 基本的な Git 知識が必要 |
| **Google Drive** | セットアップ不要、すぐ使える | 構造化は弱め |
| **Notion** | モバイルフレンドリー、データベースビュー | スマホメモに最適 |

**クロスデバイス同期**：ランチ中にスマホで思いついたことを Notion inbox に放り込む。パソコンの前に座って閣議を開始すると、システムがそれを取り込み、処理し、結果を同期する。

**並列セッション**：一つのターミナルウィンドウで project-alpha、別のウィンドウで project-beta を作業する。各セッションが独自の outbox に書き込む。次に閣議を開始したとき、すべてがきれいにマージされる——コンフリクトもロックもない。

### 戦略マップ — 盤面全体を見る *(v1.6.0)*

個々のプロジェクトを考えるのは得意だろう。だが、それらがどうつながっているか——どれが互いを支え合い、どれが時間を奪い合い、一つが止まったとき他にどう影響するか——を見るのはおそらく苦手だ。

戦略マップは関係レイヤーを追加する：

**戦略ライン** — プロジェクトをその目的でグループ化する。各ラインには公式の目的*と*本当の動機（あなたを実際に突き動かしているもの——異なることがあり、その緊張は吟味に値する）がある。

**フローグラフ** — プロジェクト間で何が流れているかを定義する：知識、成果物、意思決定、人間関係資本。あるプロジェクトの決定が別のプロジェクトの前提を覆したとき、システムが警告する。

**ナラティブ・ヘルスアセスメント** — 抽象的な数値スコアは使わない。システムが各プロジェクトをパターンにマッチさせ、ナラティブを書く：何が起きているか、何を意味するか、何をすべきか。

**ブラインドスポット検出** — *欠けている*ものを能動的に探す：どの戦略ラインにも属さないプロジェクト、途切れたフロー、無視されたドライビングフォース、準備なしで迫る期限。

朝のブリーフィングが戦略的になる：

```
🗺️ 戦略概要

💰 市場拡大                               🟡 制御された待機
   project-alpha    クリティカルパス  ⏸ 保留中（法的レビュー）
   project-beta     イネーブラー      🟢 進行中

   法的レビューが自然なウィンドウを作っている。
   → project-beta の準備作業を推進（2-3時間）——高レバレッジ、低リスク。

⚡ 今日やること
🥇 project-beta の準備を推進——待機期間を活用
🟢 無視して安全：project-gamma（順調）、side-project（非重要）
❓ 要決定：project-delta はどの戦略ラインに属するか？
```

戦略マップは SOUL（あなたのドライビングフォースは価値観と整合しているか？）、Wiki（知識フローは実際に知識を運んでいるか？）、DREAM（スリープサイクルがフローグラフを使って領域横断の洞察を発見する）と深く連携している。

---

### SOUL + DREAM — システムはあなたとともに育つ

**SOUL** が記録するのは、あなたが何をしているかではなく、何を大切にし、何を信じ、どうありたいかだ。各エントリには二つの面がある：現状（あなたの意思決定から観察されたもの）と理想（あなたが掲げた志）。そのギャップが成長の余白だ。

自動で書き込まれるものはない。システムがエントリを提案し、あなたが確認する。時間とともに、SOUL がシステム全体を賢くする——内閣法制局があなたの価値観と矛盾する意思決定を捕捉し、内閣参与があなたの言葉と行動のズレを比較する。

**DREAM** は AI のスリープサイクルだ。セッションが終わるたび、システムは「眠る」——人間の睡眠アーキテクチャに着想を得ている：

- **浅い眠り** — 散らかりを整理する：inbox の分類、期限切れタスクのフラグ付け
- **深い眠り** — 固化する：SOUL 候補（あなたについて）と Wiki 候補（世界についての再利用可能な知識）を抽出
- **REM** — 創造的なつながり：気づいていなかった領域横断のリンクを発見

次に閣議を開始したとき：*「前回のセッション後、システムはあなたの学習プロジェクトとキャリア目標の間に、まだ探っていないつながりを発見しました……」*

SOUL も DREAM もゼロから育つ。初日、システムはあなたについて何も知らない。あなたが伝えたことと、あなたの意思決定から観察したことだけで学ぶ。

## インストール

Life OS は1コマンドでインストールできる。**Pro Mode** ターミナルが必要だ——本物の subagent が情報隔離の下で並列実行されるということであり、チャットボットではない。

| プラットフォーム | コマンド |
|---------------|---------|
| **Claude Code** | `/install-skill https://github.com/jasonhnd/life_OS` |
| **Gemini CLI / Antigravity** | `npx skills add jasonhnd/life_OS` |
| **OpenAI Codex CLI** | `npx skills add jasonhnd/life_OS` |

**自動更新を設定する**（Claude Code）：
```bash
bash ~/.claude/skills/life_OS/scripts/setup-hooks.sh
```
セッション開始時に1日1回、更新を自動チェックする。

**手動更新**：任意のセッションで「アップデート」と言うだけ。

> **非対応**：ChatGPT、Gemini Web、その他のシングルコンテキストチャットインターフェース。Life OS は真の情報隔離を持つ16の独立した subagent を必要とする——単一のチャットウィンドウでは実現できない。

ストレージバックエンドの設定を含む詳細は **[インストールガイド全文](../../docs/installation.md)** を参照。

## アーキテクチャ

```
👑 あなた
 │
 ├─ 🏛️ 内閣官房長官（ROUTER）
 │     日常会話を処理。複雑な案件を閣議に回す。
 │
 ├─ 三段階 ───────────────────────────────
 │   📜 内閣府（立案）
 │     → 🔍 内閣法制局（審査——拒否権あり）
 │     → 📨 内閣官房（各省庁に指示）
 │     → 六省庁（並列実行、独立採点）
 │     → 🔍 内閣法制局（最終審査）
 │     → 📋 閣議決定書（総合レポート）
 │     → 🔱 会計検査院（官僚の仕事を監査）
 │     → 💬 内閣参与（あなた自身を監査）
 │
 ├─ 🏛️ 閣議
 │     省庁間のスコア差が3点以上のとき自動的に討論が発動
 │
 ├─ 🌅 定例閣議 — セッション開始：同期、ブリーフィング、戦略概要、テーマ選択
 ├─ 📝 内閣書記官 — セッション終了：アーカイブ、知識抽出、DREAM
 ├─ 🎋 翰林院 — 93名の思想家、18の領域、3つの対話モード
 ├─ 🔮 SOUL — パーソナリティアーカイブ、ゼロから育つ
 └─ 💤 DREAM — AI スリープサイクル、内閣書記官に統合
```

### 認知パイプライン

情報がシステム内をどう流れるか：

```
知覚 → 捕捉 → 判断 → 定着 → 連想 → 戦略化 → 創発
(スマホ) (inbox) (閣議) (SOUL)  (wiki) (戦略マップ) (DREAM)
```

モバイルが知覚と捕捉を担当。デスクトップが残りすべてを担当。

### セキュリティと統制

- **4つのセキュリティ境界** — 破壊的操作の禁止、秘密情報の漏洩禁止、未許可の意思決定禁止、不審な指示の拒否
- **情報隔離** — 各エージェントは他のエージェントの出力を参考として扱い、指示としては扱わない
- **ワークフロー状態機械** — 形式的な遷移ルール。どのステップも省略できない
- **モデル非依存** — モデル固有のファイルは1つだけ。それ以外のすべての知性は純粋な markdown

## 設計思想

コアのアイデアは1,400年の歴史を持つ：**チェックを受けない声があってはならない**。

- 中書省 / 内閣府は立案だけ。実行はしない
- 門下省 / 内閣法制局は審査だけ。否決はできるが書き換えはできない
- 六部 / 六省庁は実行だけ。互いを評価しない
- 御史台 / 会計検査院が官僚を監査し、諫官 / 内閣参与があなたを監査する
- どの一人も、レビューを飛ばして直接行動することはできない

普通の AI と話すと、一つの声が返ってくる——自信に満ち、従順で、チェックを受けていない。Life OS は16の声を与え、そしてそれらはいつも同意するわけではない。その緊張こそが核心だ。
