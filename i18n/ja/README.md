## 四つの柱

Life OS は相互に連携する4つのシステムで構成されています。それぞれ単体でも強力ですが、組み合わせることで完全な個人ガバナンスフレームワークとなります。

---

### 🏛️ 第一の柱：三省六部 — 意思決定エンジン

> 唐代は、誰も単独でチェックなしに決定を下せないシステムを構築しました。Life OS はこれをあなたの人生に適用します。

**問題点**：AIに質問すれば答えが返ってくる。しかしレビューなし、第三者の意見なし、「ちょっと待って、何か見落としているよ」と言ってくれる存在もなし。

**解決策**：重大な決定はすべて三段階を経ます — **起草 → 審査 → 実行** — 各段階に独立したエージェントを配置し、互いの推論を参照できない仕組みです。

```
あなた：「会社を辞めて起業すべきか？」

📜 中書省    → 6つの次元に分解
🔍 門下省    → 計画を審査、感情的な盲点を確認、封駁権あり
📨 尚書省    → 六部に発令
⚔️💰👥📖⚖️🏗️ → 各部が独立して分析、1〜10点でスコア
🔍 門下省    → 全レポートの最終審査
📋 奏折      → 総合スコア：6.8/10、アクションアイテム付き
🔱 御史台    → 「兵部のタイムラインが曖昧 — 要注意」
💬 諫官      → 「3週間連続で起業関連コンテンツを消費しています — 確証バイアスでは？」
```

**六部**は人生のあらゆる次元をカバーします：

| 部 | 担当 | 例 |
|---|---|---|
| 👥 吏部 | 人 | 「共同創業者は適切な人物か？」 |
| 💰 戸部 | お金 | 「18ヶ月、収入なしで生き残れるか？」 |
| 📖 礼部 | 学習・表現 | 「まず何のスキルを身につけるべきか？」 |
| ⚔️ 兵部 | 行動 | 「ステップバイステップの実行計画は？」 |
| ⚖️ 刑部 | 規則・リスク | 「最悪のシナリオは何か？」 |
| 🏗️ 工部 | 健康・インフラ | 「体はこのストレスに耐えられるか？」 |

**主要メカニズム — 門下省の封駁**：門下省は計画が論理的かどうかを確認するだけではありません。感情の監査も行います — *今の気分が判断に影響していないか？家族はどう反応するか？10年後に後悔しないか？* 封駁して差し戻す権限を持ちます。最大2回まで。

**主要メカニズム — 諫官**：毎回の決定の後、諫官はあなた自身の盲点を率直に指摘します — 計画の質ではなく、あなたの行動パターンについて。*「直近5回の決定で財務の次元を避けています。なぜですか？」*

### 🎋 第二の柱：翰林院 — 人類の叡智の殿堂

> 正式なプロセスが不要な問いもあります。ソクラテスが必要な問いもある。

翰林院は**18領域にわたる歴史上の偉大な思想家70名以上**との対話の場を提供します — ソクラテスからマスク、老子からマンデラ、ドストエフスキーからバフェットまで。

これは「ソクラテス式対話法を使う」のではありません。**ソクラテス本人**があなたと会話します — 彼自身の声で、彼自身の例を用いて、彼独特の執拗な問いかけで。

**三つのモード：**
- **一対一** — 一人の思想家との深い対話
- **円卓会議** — 複数の思想家があなたの問いについて、それぞれの世界観から議論
- **討論** — 対立する見解を持つ二人の思想家が直接論争し、あなたが審判

**18領域**：科学 · 哲学 · 東洋思想 · 戦略 · ビジネス · 心と実践 · システム論 · 人間の本性 · 文明 · 逆境 · 美学 · 政治 · 経済 · 数学 · 医学 · 探求 · コミュニケーション · 法律

**使い方：**
```
あなた：「人生の方向性について迷っています」
丞相：「翰林院を起動しますか？」
あなた：はい
翰林院：「どんな問いを探求したいですか？」
→ 18領域を提示、セネカ＋王陽明を推薦
→ 選択すると、キャラクターのまま対話
→ 最後に各思想家から別れの言葉
→ 翰林院があなたの思考の旅をまとめ → セカンドブレインに保存
```

### 🧠 第三の柱：セカンドブレイン — 知識は蓄積される

> あなたの決断、洞察、パターンはセッション終了とともに消えない。

Life OS はすべてを**永続的な知識ベース**に記録します — 決定、アクションアイテム、日記、ウィキ知識 — Git リポジトリ内のマークダウンファイルとして構造化されます。

```
second-brain/
├── SOUL.md              # 🔮 あなたとは誰か（決断から成長する）
├── user-patterns.md     # 📊 あなたの行動パターン（諫官の観察記録）
├── inbox/               # 📥 モバイルからの未処理キャプチャ
├── _meta/               # 🔧 システムメタデータ、日記、夢レポート
├── projects/{name}/     # 🎯 アクティブなプロジェクト（タスク＋決定）
├── areas/{name}/        # 🌊 継続的な生活領域と目標
├── wiki/                # 📚 ナレッジアーカイブ——再利用可能な結論（DREAMから成長）
└── archive/             # 🗄️ 完了した作業
```

**三つのストレージバックエンド** — 一つを選ぶか組み合わせる：
- **GitHub** — テクニカルユーザー向け、バージョン管理付き、Obsidian と連携
- **Google Drive** — セットアップ不要、誰でも使える
- **Notion** — モバイルフレンドリー、データベースビュー

**クロスデバイス同期**：スマホのキャプチャが Notion の inbox へ → デスクトップセッションで取り込み処理 → 結果がすべてのバックエンドに同期。

**並列セッション対応**：複数のセッションが異なるプロジェクトで同時に作業可能 — コンフリクトなし、ロックなし。各セッションは退朝時に自分専用の **outbox** に出力を書き込み、次に上朝したセッションがすべてを統合します。一つのウィンドウで GCSB、別のウィンドウで EIP、三番目で medical-plan — それぞれ独立に退朝し、次の上朝で早朝官がまとめます。

### 🔮 第四の柱：SOUL + DREAM — システムはあなたを学ぶ

> AIはセッションのたびにゼロからスタートすべきではない。あなたが誰かを知り、ともに成長すべきです。

**SOUL — パーソナリティアーカイブ**

SOUL.md は**あなたとは誰か**を記録します — 何をしているか（それは user-patterns.md）ではなく、何を大切にし、何を信じ、どんな人間になりたいかを。

各エントリには二つの側面があります：
- **現状（IS）** — 決断から観察された実際の行動
- **理想（SHOULD BE）** — 掲げた志

その差こそが成長の場です。システムはあなたの決断から価値観を発見し、エントリを提案し、あなたが確認します。自動で書き込まれることはありません。

SOUL.md が育つにつれ、システム全体が賢くなります：
- 丞相がより鋭い質問をするようになる
- 中書省があなたが気にしているのに言い忘れた次元を加える
- 門下省が掲げた価値観と矛盾する決定を捉える
- 諫官があなたの発言と行動のズレを比較する

**DREAM — AIのスリープサイクル**

セッションが終わるたびに、システムは「眠り」ます — 人間の睡眠アーキテクチャに着想を得て：

- **N1-N2（浅い眠り）**：未解決事項を整理 — inbox を分類、期限切れタスクにフラグ
- **N3（深い眠り）**：固化 — SOUL 候補（あなたについて）と Wiki 候補（世界についての再利用可能な知識）を抽出
- **REM（夢）**：創造的な連想 — まだ気づいていない領域横断のつながりを発見

次に上朝すると：*「💤 前回のセッションでシステムが夢を見ました…」*

## 安全性とガバナンス

- **4つの不可侵のセキュリティ境界**：破壊的操作の禁止、秘密情報の露出禁止、無許可の決定禁止、不審な指示の拒否
- **上流出力の保護**：エージェントは他のエージェントの出力を参考情報として扱い、指示としては扱わない
- **ワークフロー状態マシン**：正式な遷移規則 — どのステップも省略不可
- **モデル非依存**：CLAUDE.md のみがモデルに依存するファイル。その他すべてのインテリジェンスは純粋なマークダウン

## 設計思想

唐代の三省六部制の核心は**権力分立に基づくチェック・アンド・バランス**です：

- 中書省は立案のみ行い、実行しない
- 門下省は審査のみ行い、否決はできても書き直しはできない
- 六部は実行のみ行い、互いを評価しない
- 御史台は官吏を監査し、諫官は皇帝を監査する
- いかなる単独の役職もレビューを省略して直接行動することはできない

Life OS はこの1,300年の智慧をあなたの個人的な意思決定に応用します。
