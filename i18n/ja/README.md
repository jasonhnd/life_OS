## 財務省、法務省、会計検査院——あなた専用の閣僚が揃っている

Life OS は AI ターミナル（Claude Code、Gemini CLI、Codex CLI）にインストールして使う**個人の内閣**だ。16の独立した AI エージェントがあなたの意思決定をあらゆる角度から分析し、互いに議論し、計画*と*あなた自身の両方に責任を問う。

セッションを始めると、まずテーマを選ぶ：

```
🎨 テーマを選んでください：
 a) 🏛️ 三省六部 — 唐朝の統治制度（中国古典）
 b) 🏛️ 霞が関 — 日本の中央省庁
 c) 🏛️ C-Suite — 企業エグゼクティブ構造（英語）

a、b、c のいずれかを入力
```

**霞が関テーマ**を選べば、エージェントはすべて馴染みのある省庁名で動く。「財務省」と言われれば金の話だとわかる。「法務省」と言われればリスクの話だとわかる。学習コストはゼロだ。

同じ意思決定——「会社を辞めるべきか？」——を三つのテーマで見るとこうなる：

```
  三省六部                  霞が関                    C-Suite
  ─────────                ─────────                ─────────
  📜 中書省 drafts plan     📜 内閣府 drafts plan      📜 VP Strategy drafts plan
  🔍 门下省 vetoes:         🔍 内閣法制局 vetoes:       🔍 General Counsel vetoes:
     "Financial runway        "財務的余裕が不明"          "Runway not addressed"
      not addressed"

  💰 户部  5/10            💰 財務省  5/10            💰 CFO  5/10
  ⚔️ 兵部  6/10            ⚔️ 防衛省  6/10            ⚔️ VP Ops  6/10
  ⚖️ 刑部  4/10            ⚖️ 法務省  4/10            ⚖️ CCO  4/10

  🔱 御史台 audits          🔱 会計検査院 audits       🔱 Internal Audit audits
  💬 谏官 challenges you    💬 内閣参与 challenges you  💬 Exec Coach challenges you

  📋 奏折: 5.8/10          📋 閣議決定書: 5.8/10       📋 Executive Brief: 5.8/10
```

三つの世界。裏側のロジックは完全に同じ。テーマはセッション単位で適用され、いつでも「テーマ切り替え」と言えば変更できる。テーマが出力言語を決定する——霞が関なら日本語、三省六部なら中国語、C-Suite なら英語。これは絶対のルールだ。

**トリガーワード自動推論**：「閣議開始」と入力すれば霞が関テーマが自動選択される。「上朝」なら三省六部。英語の "start" やその他の場合は選択プロンプトが表示される。

> **ロールプレイではない。** 各エージェントは本物の、隔離された subagent として実行される。互いの推論は見えない。独立に採点する。意見が分かれる。

## 仕組み

Life OS は五つの柱で構成される。**意思決定エンジン**が中核で、残りはすべてそこから派生する。

---

### I. 意思決定エンジン — 立案、審査、差し戻し、実行、監査

エンジンは16のエージェントを動かす。設計原理は1,400年の歴史を持つ——**チェックを受けない声があってはならない**。テーマはこれらのエージェントにあなたの文化の名前を与える。ロジック自体はつねに同一だ。

すべての重大な意思決定は三段階を通過する。ショートカットはない。

**立案** — 内閣府（PLANNER）があなたの状況を六つの次元に分解し、計画を策定する。

**審査** — 内閣法制局（REVIEWER）が計画を独立に審査する。感情の監査も行う：*恐怖心が判断を歪めていないか？ 家族は支持するか？ 10年後に後悔しないか？* 盲点を発見したら差し戻し——計画は修正が必要だ。

**実行** — 六省庁がそれぞれの領域から計画を分析し、1-10で独立に採点する：

| 省庁 | 守備範囲 | 問いかけの例 |
|------|---------|------------|
| 総務省（PEOPLE） | 人間関係、ステークホルダー | 「適切な人が関わっているか？」 |
| 財務省（FINANCE） | 金、資産、リソース | 「最悪のケースも含めて、負担できるか？」 |
| 文部科学省（GROWTH） | 学び、表現、文化 | 「まず何を身につけるべきか？」 |
| 防衛省（EXECUTION） | 行動、段取り、タイムライン | 「週単位の具体的な計画は？」 |
| 法務省（GOVERNANCE） | ルール、リスク、コンプライアンス | 「すべてが裏目に出たらどうなるか？」 |
| 国土交通省（INFRA） | 健康、体力、環境 | 「あなたの体と環境はそれを支えられるか？」 |

各省庁には4つの部門がある。六省庁の報告が揃うと、会計検査院（AUDITOR）が官僚たちの仕事を監査する（「防衛省のタイムラインにマイルストーンがない——指摘」）。そして内閣参与（ADVISOR）があなた自身に向き合う：「直近4回の意思決定で、財務面を避け続けています。なぜですか？」

省庁間のスコアに3点以上の差があると、閣議（COUNCIL）が自動的に発動し、省庁間の討論が行われる。

フルフローの全体像：

```
あなた：「会社を辞めて何か新しいことを始めようか考えている」

    立案
    📜 内閣府        → 6つの次元に分解、計画を策定

    審査
    🔍 内閣法制局    → 感情の監査：逃げているのか、向かっているのか？
                       差し戻し：「財務的な余裕が未検討。修正を。」

    実行（修正が審査を通過した後）
    👥 総務省    7/10   「共同創業者との相性が未検証」
    💰 財務省    5/10   「余裕は18ヶ月。ただし何も起きなければ」
    📖 文科省    8/10   「領域の専門性は確か、信頼性がある」
    ⚔️ 防衛省    6/10   「3ヶ月目以降のマイルストーンがない」
    ⚖️ 法務省    4/10   「競業避止条項の法的確認が必要」
    🏗️ 国交省    7/10   「健康状態は良好、ただしストレス管理計画が曖昧」

    監査
    🔱 会計検査院     → 「防衛省の計画、3ヶ月目以降が曖昧——修正を求む」
    💬 内閣参与       → 「ここ数週間、起業コンテンツばかり見ています。
                          確証バイアスの可能性が高い。最後に真剣に
                          残留を検討したのはいつですか？」

    📋 閣議決定書     → 総合スコア：6.2/10 — 条件付きで進行可
```

すべてにフルフローが必要なわけではない。内閣官房長官（ROUTER）が日常会話、簡単な質問、感情面のサポートを直接処理する。省庁の専門知識は必要だが意思決定ではない場合——たとえば「フリーランスに適用される税制は？」——**エクスプレスパス**が1-3の関連省庁に送り、フル閣議は行わない。

### II. 内閣特別顧問 — 分析ではなく、知恵が必要なとき

「正解」がない問いがある。必要なのは視座だ。

内閣特別顧問（STRATEGIST）では、**18の領域にわたる93名の歴史上最も偉大な思想家**と対話できる——ソクラテスからバフェット、老子からマンデラ、ドストエフスキーからファインマンまで。一人ひとりが独立した subagent として、独自の語り口、独自の例え、独自の追及を持って対話する。

**3つのモード：**

- **一対一** — 一人の思想家と深く語り合う。ソクラテスは容赦しない。
- **円卓会議** — 複数の思想家があなたの問いについて、それぞれの世界観から語る。セネカと王陽明が予想外の共通点を見つける場面を見よ。
- **討論** — 対立する二人の思想家が直接論争する。あなたが判定する。

```
あなた：「始めたことを最後まで終えられない」

推薦 — セネカ（時間について）+ 王陽明（行動について）

セネカ：「あなたに足りないのは時間ではない。吟味していないことに
         時間を浪費しているのだ。今取り組んでいることのうち、
         今の知識を持ったまま戻れるとして、
         もう一度始めるのはどれか？」

王陽明：「知行合一。本当に何を望んでいるかわかっているなら、
         すでにそれをしているはずだ。知ることと行うことの
         隔たりは、望むことと心から望むことの隔たりだ。」

→ 各思想家からの別れの言葉
→ あなたの思考の旅がナレッジベースに保存
```

**18の領域**：哲学 · 東洋思想 · 科学 · 戦略 · ビジネス · 心理学 · システム思考 · 人間の本性 · 文明 · 逆境 · 美学 · 政治 · 経済 · 数学 · 医学 · 探求 · コミュニケーション · 法律

### III. セカンドブレイン — セッションが終わっても、何も消えない

すべての決定、洞察、パターン、アクションアイテムは**永続的なナレッジベース**に書き込まれる——あなたが所有する構造化された Markdown ファイルで、あなたが選んだストレージに保存される。

```
second-brain/
├── SOUL.md                 # あなたが誰か——価値観、アイデンティティ、志
├── user-patterns.md        # あなたの行動パターン——内閣参与の観察
├── inbox/                  # スマホからの走り書き
├── _meta/
│   ├── STATUS.md           # 全体ステータスダッシュボード
│   ├── STRATEGIC-MAP.md    # プロジェクト間の戦略的関係
│   ├── journal/            # セッションレポート、DREAM レポート
│   └── outbox/             # セッション別ステージングエリア
├── projects/{名前}/        # 進行中のプロジェクト、タスク + 決定付き
├── areas/{名前}/           # 継続的な生活領域と目標
├── wiki/                   # 再利用可能な知識——DREAM から育つ
└── archive/                # 完了した仕事
```

**3つのストレージバックエンド** — あなたの生活に合うものを選ぶ：

| バックエンド | 向いている人 | トレードオフ |
|------------|------------|------------|
| **GitHub** | バージョン管理、Obsidian 連携 | 基本的な Git 知識が必要 |
| **Google Drive** | セットアップ不要、すぐ使える | 構造化は弱め |
| **Notion** | モバイルフレンドリー、データベースビュー | スマホメモに最適 |

**クロスデバイス同期**：ランチ中にスマホで思いついたことを Notion inbox に放り込む。パソコンの前に座って閣議を開始すると、システムがそれを取り込み、処理し、結果を同期する。

**並列セッション**：一つのターミナルウィンドウで project-alpha、別のウィンドウで project-beta を作業する。各セッションが独自の outbox に書き込む。次に閣議を開始したとき、すべてがきれいにマージされる——コンフリクトもロックもない。

初回起動時、セカンドブレインのディレクトリ構造は自動的に作成される。

### IV. 戦略マップ — 盤面全体を見る

個々のプロジェクトを考えるのは得意だろう。だが、それらがどうつながっているか——どれが互いを支え合い、どれが時間を奪い合い、一つが止まったとき他にどう影響するか——を見るのはおそらく苦手だ。

戦略マップは関係レイヤーを追加する：

**戦略ライン** — プロジェクトをその目的（purpose）でグループ化する。各ラインには公式の目的*と*本当の動機（driving force）がある。この二つが食い違っていることがあり、その緊張は吟味に値する。ラインごとに健康シグナル（health signals）も定義する。

**プロジェクト役割** — 各プロジェクトが戦略ラインの中で果たす役割を定義する：critical-path（不可欠）、enabler（支援）、accelerator（加速）、insurance（保険）。

**フローグラフ** — プロジェクト間で何が流れているかを定義する。フロータイプは4種類：cognition（知識）、resource（リソース）、decision（意思決定）、trust（信頼関係資本）。あるプロジェクトの決定が別のプロジェクトの前提を覆したとき、システムが警告する。

**ナラティブ・ヘルスアセスメント** — 抽象的な数値スコアは使わない。6つのアーキタイプで各プロジェクトの健康状態をパターンマッチし、ナラティブ（物語）で語る：何が起きているか、何を意味するか、何をすべきか。

**ブラインドスポット検出** — *欠けている*ものを能動的に5種類の視点で探す：どの戦略ラインにも属さないプロジェクト、途切れたフロー、無視されたドライビングフォース、準備なしで迫る期限、バランスの崩れた注力配分。

**クロスレイヤー統合** — 戦略マップは SOUL（あなたのドライビングフォースは価値観と整合しているか？）、Wiki（知識フローは実際に知識を運んでいるか？）、DREAM（スリープサイクルがフローグラフを使って領域横断の洞察を発見する）と深く連携し、アクション推奨まで行う。

朝のブリーフィングが戦略的になる：

```
🗺️ 戦略概要

💰 市場拡大                               🟡 制御された待機
   project-alpha    クリティカルパス  ⏸ 保留中（法的レビュー）
   project-beta     イネーブラー      🟢 進行中

   法的レビューが自然なウィンドウを作っている。
   → project-beta の準備作業を推進（2-3時間）——高レバレッジ、低リスク。

⚡ 今日やること
🥇 project-beta の準備を推進——待機期間を活用
🟢 無視して安全：project-gamma（順調）、side-project（非重要）
❓ 要決定：project-delta はどの戦略ラインに属するか？
```

### V. SOUL + DREAM — システムはあなたとともに育つ

**SOUL** が記録するのは、あなたが何をしているかではなく、何を大切にし、何を信じ、どうありたいかだ。各エントリには二つの面がある：現状（IS）——あなたの意思決定から観察されたもの、理想（SHOULD BE）——あなたが掲げた志。そのギャップが成長の余白だ。

自動で書き込まれるものはない。システムがエントリを提案し、あなたが確認する。時間とともに、SOUL がシステム全体を賢くする——内閣法制局があなたの価値観と矛盾する意思決定を捕捉し、内閣参与があなたの言葉と行動のズレを比較する。

**DREAM** は AI のスリープサイクルだ。セッションが終わるたび、システムは「眠る」——人間の睡眠アーキテクチャに着想を得ている：

- **N1-N2（浅い眠り）** — 散らかりを整理する：inbox の分類、期限切れタスクのフラグ付け
- **N3（深い眠り）** — 固化する：SOUL 候補（あなたについて）と Wiki 候補（世界についての再利用可能な知識）を抽出
- **REM** — 創造的なつながり：気づいていなかった領域横断のリンクを発見

次に閣議を開始したとき：*「前回のセッション後、システムはあなたの学習プロジェクトとキャリア目標の間に、まだ探っていないつながりを発見しました……」*

SOUL も DREAM もゼロから育つ。初日、システムはあなたについて何も知らない。あなたが伝えたことと、あなたの意思決定から観察したことだけで学ぶ。

## 使ってみる

### 一日の始め方

```
あなた：閣議開始。

🌅 定例閣議：
   テーマ自動検出 → 霞が関
   セカンドブレインを同期中……Notion から3件取り込みました。
   📥 「資格プログラムを調べる」——昨日スマホでメモ
   📥 「project-alpha：取引先から返信あり」——メールから転送
   📥 クイックメモ：「予算の前提を見直す」

   🗺️ 戦略概要：[上記の戦略マップ参照]

   💤 DREAM レポート：前回のセッションで、交渉テクニックに関する
      wiki エントリが project-alpha の取引先との会話に
      適用できることを発見しました。

   📋 推奨：取引先への返信を先に処理（時間的制約あり）。
```

セッション開始時には18ステップのプロセスが走る：同期、DREAM レポート、戦略概要、inbox 処理、ハウスキーピング。

### 意思決定をする

```
あなた：正社員からフリーランスへの転向を考えている。
→ 2-3ラウンドの意図確認
→ フルフロー：立案 → 審査 → 実行 → 監査
→ 閣議決定書：5.8/10——「実行可能だが時期尚早。収入の余裕は
   想定の18ヶ月ではなく11ヶ月。推奨：あと3ヶ月分の
   貯蓄を積み、アンカークライアントを1社獲得してから
   トランジションすること。」
```

### 深く考える

```
あなた：自分が気にしていないことにいつもイエスと言ってしまう。
→ 内閣特別顧問が起動
→ マルクス・アウレリウスと一対一で、優先順位と断ることについて
→ 洞察がセカンドブレインに保存
```

### 一日の終え方

```
あなた：閣議終了。

📝 内閣書記官：
   セッションをアーカイブ中……知識を抽出中……
   💤 DREAM サイクル実行中……
   - SOUL 候補：「安定より自律を重視する」——確認しますか？[Y/n]
   - Wiki 候補：「フリーランスの余裕計算式」→ wiki/career/
   - REM の洞察：健康目標とキャリアのタイムラインに、
     まだ対処していない共通の制約があります——エネルギー管理。
   Notion に同期中……完了。
```

閣議終了は4フェーズで構成される：アーカイブ、知識抽出、DREAM サイクル、同期。完了チェックリストで漏れを防ぐ。

### レビュー

セッション中、3種類の定期レビューが利用できる：

- **DTR（日次レビュー）** — その日の振り返り
- **ACR（累積レビュー）** — 週次・月次の累積評価
- **OFR（公式レビュー）** — 四半期・年次の戦略的レビュー

## 12の標準シナリオ

Life OS は人が実際に直面する意思決定にあらかじめ対応している：

| # | シナリオ | 関与する省庁 | 内閣法制局が問うこと |
|---|---------|------------|-------------------|
| 1 | キャリア転換 | 六省庁全体 | 「逃げているのか、何かを追いかけているのか？」 |
| 2 | 投資判断 | 財務省、防衛省、法務省、総務省 | 「FOMO か合理的判断か？全損に耐えられるか？」 |

## インストール

Life OS は1コマンドでインストールできる。**Pro Mode** ターミナルが必要だ——本物の subagent が情報隔離の下で並列実行されるということであり、チャットボットではない。

| プラットフォーム | コマンド |
|---------------|---------|
| **Claude Code** | `/install-skill https://github.com/jasonhnd/life_OS` |
| **Gemini CLI / Antigravity** | `npx skills add jasonhnd/life_OS` |
| **OpenAI Codex CLI** | `npx skills add jasonhnd/life_OS` |

初回起動時にテーマを選ぶ。システムが言語を自動検出して最適なテーマを提案するが、選択はつねにあなた次第だ。いつでも「テーマ切り替え」と言えば変更できる。

**自動更新を設定する**（Claude Code）：
```bash
bash ~/.claude/skills/life_OS/scripts/setup-hooks.sh
```
セッション開始時に1日1回、更新を自動チェックする。

**手動更新**：任意のセッションで「アップデート」と言うだけ。

> **非対応**：ChatGPT、Gemini Web、その他のシングルコンテキストチャットインターフェース。Life OS は真の情報隔離を持つ16の独立した subagent を必要とする——単一のチャットウィンドウでは実現できない。

ストレージバックエンドの設定を含む詳細は **[インストールガイド全文](../../docs/installation.md)** を参照。

### 認知パイプライン

情報がシステム内をどう流れるか：

```
知覚 → 捕捉 → 判断 → 定着 → 連想 → 戦略化 → 創発
(スマホ) (inbox) (閣議) (SOUL)  (wiki) (戦略マップ) (DREAM)
```

モバイルが知覚と捕捉を担当。デスクトップが残りすべてを担当。

### テーマシステム

```
themes/
├── zh-classical.md      # 三省六部 — 唐朝の統治制度（中国語）
├── ja-kasumigaseki.md   # 霞が関 — 日本の中央省庁（日本語）
└── en-csuite.md         # C-Suite — 企業エグゼクティブ（英語）
```

各テーマは約60行のファイルひとつ。16の機能 ID を表示名にマッピングし、語調を定義し、トリガーワードを設定し、出力フォーマットを命名する。エンジンはセッション開始時にテーマファイルを一度読み込み、以降はその名前をあらゆる箇所で使用する。

新しいテーマの追加（韓国政府、EU 議会、幕府、スタートアップ取締役会など）にはファイルを1つ追加するだけ。エンジンの変更も、エージェントの追加も不要だ。

### セキュリティと統制

- **4つのセキュリティ境界** — 破壊的操作の禁止、秘密情報の漏洩禁止、未許可の意思決定禁止、不審な指示の拒否
- **情報隔離** — 各エージェントは他のエージェントの出力を参考として扱い、指示としては扱わない
- **ワークフロー状態機械** — 形式的な遷移ルール。どのステップも省略できない
- **モデル非依存** — モデル固有のファイルは1つだけ。それ以外のすべての知性は純粋な Markdown

## 設計思想

コアのアイデアは1,400年の歴史を持つ：**チェックを受けない声があってはならない**。

- 内閣府は立案だけ。実行はしない
- 内閣法制局は審査だけ。否決はできるが書き換えはできない
- 六省庁は実行だけ。互いを評価しない
- 会計検査院が官僚を監査し、内閣参与があなたを監査する
- どの一人も、レビューを飛ばして直接行動することはできない

普通の AI と話すと、一つの声が返ってくる——自信に満ち、従順で、チェックを受けていない。Life OS は16の声を与え、そしてそれらはいつも同意するわけではない。その緊張こそが核心だ。

テーマエンジンは第二の原理を加える：**ガバナンスは普遍的だが、文化は個人的なものだ**。良い意思決定を生むロジックはどこでも同じ。それを「自分のもの」と感じさせる言葉は、同じではない。Life OS はこの二つを分離することで、両方を手に入れる。
