## ふつうの AI と何が違うのか

ChatGPT に「転職すべきか？」と聞けば、もっともらしい答えが返ってくる。でも——

- 誰もその答えをレビューしていない
- 「ちょっと待て、感情に流されてないか？」と指摘する人がいない
- あなたの過去の判断パターンを知らない
- 次のセッションではすべて忘れている

Life OS は違う。

あなたの AI に**16人の独立した閣僚**を配置する。立案する省、それを審査して拒否権を持つ省、実行する6つの部、官僚の仕事ぶりを監視する御史台、そして**あなた自身の行動パターンを率直に指摘する諫官**。互いの推論は見えない。チェック・アンド・バランス。1,300年前の唐王朝が発明した統治制度を、あなたの人生の意思決定に適用する。

## Life OS の五本柱

---

### 🏛️ 柱 I：三省六部——意思決定エンジン

**あなたが抱えている問題：**

重大な決断を前にして、AI に相談する。答えが返ってくる。でもその答えを誰がチェックしたのか？財務面は？家族の反応は？そもそも今の気分に引っ張られた判断ではないか？

**Life OS の答え：**

すべての重大な決定は三段階を通過する——**立案 → 審査 → 実行**。各段階には独立したエージェントがいて、互いの出力を参照できない。

```
あなた：「会社を辞めて起業すべきか？」

📜 中書省      → 問いを6つの次元に分解する
🔍 門下省      → 計画を審査、感情を監査、拒否権あり
📨 尚書省      → 六部に実行を指示
⚔️💰👥📖⚖️🏗️  → 各部が独立分析、1〜10点で評価
🔍 門下省      → 全レポートの最終審査
📋 奏折        → 総合スコア 6.8/10、具体的アクション付き
🔱 御史台      → 「兵部のスケジュール、根拠が曖昧」
💬 諫官        → 「3週間、起業コンテンツしか見ていません——確証バイアスでは？」
```

**六部が人生のあらゆる面をカバーする：**

| 部 | 何を見るか | たとえばこう聞かれる |
|---|---|---|
| 👥 吏部 | 人間関係 | 「その共同創業者、本当に信頼できる？」 |
| 💰 戸部 | お金 | 「収入ゼロで18か月、生き残れるか？」 |
| 📖 礼部 | 学び・表現 | 「まず何を身につけるべきか？」 |
| ⚔️ 兵部 | 行動計画 | 「来週月曜からの具体ステップは？」 |
| ⚖️ 刑部 | ルール・リスク | 「最悪のシナリオは何？取り返しがつくか？」 |
| 🏗️ 工部 | 健康・インフラ | 「この負荷に体が耐えられるか？」 |

#### 門下省の封駁——「ちょっと待て」と言える唯一の存在

門下省は計画のロジックだけをチェックするわけではない。**感情の監査**も行う。

- 今の気分が判断を歪めていないか？
- 家族はどう反応するか？
- 10年後、これを後悔しないか？

問題があれば**封駁**（差し戻し）する。最大2回。

#### 諫官——あなた自身の盲点を突く

計画の質ではなく、**あなたの行動パターン**を見ている。

> *「直近5回の決定で、財務面を避け続けています。なぜですか？」*

これは丁寧なアドバイスではない。率直な指摘だ。

### 🎋 柱 II：翰林院——人類の叡智の殿堂

**あなたが抱えている問題：**

人生の方向性がわからない。キャリアと家庭のバランスが見えない。こういう問いに必要なのは、分析ではなく**知恵**だ。

**Life OS の答え：**

翰林院には**18の領域にわたる70名以上の歴史上の思想家**がいる。ソクラテス、老子、セネカ、マンデラ、ドストエフスキー、バフェット、イーロン・マスク——。

これは「ソクラテス式対話法を使う AI」ではない。**ソクラテス自身**が、あなたに問いかける。彼の言葉で、彼の例で、彼特有の執拗な追及で。

**3つのモード：**

| モード | どんなとき | 例 |
|---|---|---|
| 一対一 | 一人の思想家とじっくり | セネカと「不安との付き合い方」を語る |
| 円卓会議 | 複数の視点が欲しい | 孔子とニーチェが「義務と自由」について議論 |
| 討論 | 対立する意見を見たい | マルクスとハイエクが「格差」について論争 |

**18領域**：科学 · 哲学 · 東洋思想 · 戦略 · ビジネス · 心と実践 · システム論 · 人間の本性 · 文明 · 逆境 · 美学 · 政治 · 経済 · 数学 · 医学 · 探求 · コミュニケーション · 法律

**使い方の流れ：**
```
あなた：「最近、自分の人生の方向性がわからなくなってきた」
丞相：「翰林院を起動しますか？」
あなた：はい
翰林院：18の領域を提示 → セネカと王陽明を推薦
→ 選択すると、思想家がキャラクターのまま対話
→ 最後にそれぞれから別れの一言
→ 思考の旅をまとめ → セカンドブレインに保存
```

### 🧠 柱 III：セカンドブレイン——知識が蓄積される

**あなたが抱えている問題：**

AI との会話はセッションが終わると消える。先月の重要な決定も、去年の年間計画も、すべてゼロに戻る。

**Life OS の答え：**

決定、アクション、日記、再利用可能な知識——すべてを**永続的なナレッジベース**に markdown で保存する。セッションを重ねるほど、システムはあなたを深く理解する。

```
second-brain/
├── SOUL.md                # あなたが「誰であるか」の記録
├── user-patterns.md       # 諫官が観察するあなたの行動パターン
├── inbox/                 # スマホから放り込んだ未処理のメモ
├── _meta/
│   ├── STATUS.md          # 全体のステータス
│   ├── STRATEGIC-MAP.md   # 戦略マップ（v1.5.0 新機能）
│   ├── strategic-lines.md # 戦略ラインの定義
│   ├── journal/           # レポート・DREAMレポート
│   └── outbox/            # セッションごとの一時出力
├── projects/{名前}/       # 進行中のプロジェクト
├── areas/{名前}/          # 継続的な生活領域と目標
├── wiki/                  # 再利用できる結論のアーカイブ
└── archive/               # 完了済み
```

**ストレージは3つから選べる：**

| バックエンド | 向いている人 |
|---|---|
| GitHub | Git に慣れている人。Obsidian と連携可能 |
| Google Drive | セットアップ不要。誰でも使える |
| Notion | スマホでも見やすい。データベースビュー対応 |

組み合わせて使うこともできる。

**スマホとの連携：** スマホで思いついたことを Notion の inbox に放り込む → 次にデスクトップで上朝すると自動で取り込み → 全バックエンドに同期。

**複数セッションの同時作業：** ウィンドウ A で仕事のプロジェクト、ウィンドウ B で副業、ウィンドウ C で引越しの計画。それぞれが独立に退朝し、次に誰かが上朝したとき、早朝官がすべてを一つにまとめる。競合もロックもない。

### 🗺️ 柱 IV：戦略マップ——森を見る（v1.5.0）

**あなたが抱えている問題：**

プロジェクトが5つある。そのうち2つは互いに知識を供給し合っている。1つが止まると、気づかないうちに他の3つもブロックされる。でも毎朝のブリーフィングはフラットなリストで、関係性も優先順位も見えない。

**Life OS の答え：**

戦略マップはプロジェクトの**上に**関係レイヤーを敷く。

- **戦略ライン** ── プロジェクトを目的別にグループ化。各ラインには `purpose`（建前の目的）と `driving_force`（本当の動機）がある。両者が異なることは珍しくない
- **フローグラフ** ── プロジェクト間の流れを可視化：知識、成果物、意思決定、人間関係
- **ヘルスアーキタイプ** ── 数字のスコアは使わない。パターンマッチング＋ナラティブ：「project-alpha は法的レビュー待ち。この待ち時間を活用して enabler を前に進めよ」
- **アクション推奨** ── 最もレバレッジの高いアクション。工数見積もりと、やらなかった場合のコスト付き

**朝のブリーフィングがこう変わる：**
```
🗺️ 戦略概要：

💰 市場拡大ライン                     🟡 制御された待機
   ウィンドウ: 2026-09-30（24週間）
   本当の動機: 市場プレゼンスの構築

   project-alpha    クリティカルパス   ⏸ 保留中
   project-beta     イネーブラー       🟢 進行中（3日目）

   要約: 法的レビュー期間。待ちを活かして project-beta を推進。
   → 推奨: project-beta の準備作業を進める（2-3時間、高レバレッジ）

⚡ 今日やるべきこと：
🥇 project-beta の準備——待機期間を活用
🟢 今日は無視して良い: project-gamma（順調）、side-project-x（非重要）
❓ 要決定: project-delta はどの戦略ラインに属するか？
```

**クロスレイヤー連携：** 戦略マップは SOUL（価値観との整合）、Wiki（知識フローの検証）、DREAM（領域横断の洞察）と接続されている。4つの知識レイヤーで1つの認知システムを構成する。

### 🔮 柱 V：SOUL + DREAM——システムはあなたとともに育つ

**あなたが抱えている問題：**

AI は毎回まっさらな状態で始まる。先月「家族を最優先にする」と決めたことも、半年前から財務の問題を避け続けていることも、知らない。

**Life OS の答え：**

#### SOUL ── あなたが「誰であるか」の記録

SOUL.md は**あなたが何をしているか**ではなく（それは user-patterns.md の仕事）、**あなたが何を大切にしているか**を記録する。

各エントリには二つの面がある：
- **現状（IS）** ── 実際の決定から観察された行動
- **理想（SHOULD BE）** ── 自分が掲げた志

そのギャップが、成長の余白だ。

システムはあなたの決定から価値観を見つけ、エントリを提案する。あなたが確認するまで、何も書き込まれない。

SOUL が育つと、システム全体が賢くなる：
- 丞相の質問がより鋭くなる
- 中書省が「言い忘れたけど大切な次元」を補う
- 門下省が、あなたの価値観と矛盾する決定を捉える
- 諫官が「あなたが言っていること」と「あなたがやっていること」のズレを比較する

#### DREAM ── AI の睡眠サイクル

セッションが終わるたびに、Life OS は「眠る」。人間の睡眠アーキテクチャに着想を得ている：

| フェーズ | やること |
|---|---|
| N1-N2（浅い眠り） | inbox の分類、期限切れタスクの検出 |
| N3（深い眠り） | SOUL 候補（あなたについて）と Wiki 候補（世界についての知識）を抽出 |
| REM（夢） | 領域をまたいだ創造的なつながりを発見 |

次の上朝でこう表示される：

> 💤 *前回のセッション後、システムは夢を見ました...*

## エクスプレス分析 🏃

すべての決定に三省六部のフル朝議が必要なわけではない。

「この資格、取るべきか？」「この本と、あの本、どちらを先に読むべきか？」——こういう場面では**エクスプレス分析**を使う。

- 1〜3部だけが参加する迅速分析
- 中書省・門下省・御史台・諫官はスキップ
- 奏折ではなく簡潔なレポート
- 終了後「フル朝議に切り替えますか？」と確認

「ちょっと聞きたいだけ」の時に、16人が集まる必要はない。

## セッションの始め方と終わり方

Life OS では、セッションを**上朝（じょうちょう）**と**退朝（たいちょう）**で管理する。

| 操作 | 何が起きるか |
|---|---|
| 「上朝」と言う | 全バックエンドから同期 → ステータス確認 → 朝のブリーフィング → DREAM レポート |
| 「退朝」と言う | 作業をアーカイブ → 全バックエンドに同期 → DREAM サイクルを実行 |

「アップデート」と言えば、Life OS が最新版に更新される。

## 使ってみる

### 日常会話——丞相が直接対応

```
あなた：最近ずっと疲れが取れない
丞相：（傾聴し、必要なら工部に照会するか判断する）

あなた：この段落を英語にしてほしい
丞相：（そのまま翻訳する）
```

丞相は雑談でも翻訳でも愚痴でも、まず受け止める。重大な判断が必要なときだけ、三省六部にエスカレーションする。

### 重大な意思決定——三省六部が動く

```
あなた：会社を辞めて起業しようか迷っている
→ 丞相が意図を2〜3回確認
→ 三省六部フルフロー
→ 奏折：6.8/10、具体的アクション付き
```

### 深い思考——翰林院を開く

```
あなた：最近、何のために頑張っているのかわからなくなった
丞相：「翰林院を起動しますか？」
→ セネカと王陽明があなたと対話
→ 思考の旅を記録 → セカンドブレインに保存
```

### クイック分析——全力は不要なとき

```
あなた：この2つのオファー、どっちがいい？（クイック）
→ 戸部＋兵部が迅速に分析 → 簡潔なレポート
```

## セキュリティと統制

- **4つの不可侵の境界** ── 破壊的操作の禁止、秘密情報の漏洩禁止、無許可の決定禁止、不審な指示の拒否
- **上流出力の保護** ── 各エージェントは他のエージェントの出力を「参考」として扱い、「指示」としては扱わない
- **ワークフロー状態機械** ── 厳密な状態遷移ルール。どのステップも省略できない
- **モデル非依存** ── CLAUDE.md だけがモデルに紐づくファイル。それ以外のすべての知性は純粋な markdown

## 設計思想

唐代の三省六部制の核心は、**権力分立に基づくチェック・アンド・バランス**にある：

- 中書省は立案だけ。実行はしない
- 門下省は審査だけ。否決はできるが、書き直しはできない
- 六部は実行だけ。互いを評価しない
- 御史台が官僚を監査し、諫官が皇帝を監査する
- どの一人も、レビューを飛ばして直接行動することはできない

1,300年間機能し続けたこの知恵を、あなたの日々の意思決定に。
