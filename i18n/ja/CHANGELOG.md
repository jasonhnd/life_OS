# 変更履歴

## バージョニングルール

このプロジェクトは **Strict SemVer** に従います：

- **MAJOR** (X.0.0) — ユーザーの移行作業が必要な破壊的変更（ディレクトリ構成の変更、設定フォーマットの変更など）
- **MINOR** (0.X.0) — 後方互換性のある新機能追加。ユーザー側の対応不要
- **PATCH** (0.0.X) — バグ修正、ドキュメント更新、i18n 同期、文言変更

追加ルール：
- 同日の変更は単一リリースにまとめる
- `a`/`b`/`rc` サフィックスは使用しない
- 全リリースに git タグを付与する

## [1.3.0] - 2026-04-10

### 追加 — ストレージ抽象化レイヤー + クロスプラットフォーム Pro Mode

**ストレージ抽象化レイヤー**
- **`references/data-model.md`** — 標準データモデル：6種類（Decision/Task/JournalEntry/WikiNote/Project/Area）のフィールド定義、7つの標準オペレーション（Save/Update/Archive/Read/List/Search/ReadProjectContext）
- **`references/adapter-github.md`** — GitHub バックエンド：.md + front matter 形式、ディレクトリパス、grep/glob によるクエリ、git log による変更検出
- **`references/adapter-gdrive.md`** — Google Drive バックエンド：同じ .md 形式、Google Drive MCP 呼び出し、modifiedTime による変更検出
- **`references/adapter-notion.md`** — Notion バックエンド：標準フィールドから Notion プロパティへのマッピング、Notion MCP 呼び出し、last_edited_time による変更検出
- **マルチバックエンドサポート** — ユーザーは1つ、2つ、または3つのバックエンド（GitHub/GDrive/Notion）を選択可能。マルチバックエンド時：全バックエンドに書き込み、プライマリバックエンドから読み取り（自動優先順位：GitHub > GDrive > Notion）
- **クロスデバイス同期** — セッション開始時に毎回フル同期：全バックエンドに最終同期以降の変更をクエリ、タイムスタンプ比較、競合解決（最終書き込み優先、1分未満の差異はユーザーに確認）
- **トリガーワード表** — 5つのアクション（上朝 / レビュー / 退朝 / クイック分析 / 朝議）について英語・中国語・日本語のトリガーワードを定義
- **上朝 / 退朝** — 「上朝」= フル同期PULL + 準備 + ブリーフィング。「退朝」= フル同期PUSH + アーカイブ

**クロスプラットフォーム Pro Mode**
- **`pro/GEMINI.md`** — Gemini CLI / Antigravity 用オーケストレーションファイル。14 subagent と完全なワークフロー状態マシン、情報隔離、六部並列実行を実装
- **`pro/AGENTS.md`** — AGENTS.md オープンスタンダードに準拠した OpenAI Codex CLI 用オーケストレーションファイル
- **プラットフォーム自動検出** — SKILL.md が実行環境のプラットフォームを検出し、適切なオーケストレーションファイルを自動的に読み込む
- **モデル非依存マッピング** — 各プラットフォームが利用可能な最強モデルを自動使用。モデル名はハードコードされておらず、将来のモデルアップグレードに際してコード変更不要
- **トラブルシューティングガイド** — `docs/installation.md` に追加。よくある問題（worktree コンテキストオーバーフロー、Pro Mode が有効にならないなど）をカバー

### 変更
- **SKILL.md** — 「データレイヤー」→「ストレージ設定」にマルチバックエンドテーブルを追加；Pro Mode セクションを3プラットフォーム対応に全面改訂；行動規範 #14 を「Pro 環境は Pro mode を強制する」に更新
