# トークン消費量の内訳

Life OSのトークン消費量は2つの変数に依存します：**シナリオの複雑さ**（何部が起動されるか）と**プロセスイベント**（封駁/政事堂が発動するか）。このドキュメントでは各ケースの消費量を分解します。

> Claude Max/Proサブスクリプションの場合、トークン単位の課金はありません。以下の分析はAPIユーザーおよびコスト意識の高いユーザー向けです。

---

## 基本概念

### 役職ごとのトークン構成

各役職の呼出 = **入力トークン**（システム命令 + 受信コンテキスト）+ **出力トークン**（役職の出力）

| 役職 | システム命令 | 典型的な入力コンテキスト | 典型的な出力 | 1回あたりの合計 |
|------|--------------------|-----------------------|----------------|---------------|
| 丞相 | 約500 | ユーザーメッセージ 約200 | 上奏 約150 / 直接対応 約300 | 約700-1,000 |
| 中書省 | 約400 | 題目 + 背景 + ユーザーメッセージ 約500 | 立案 約800 | 約1,700 |
| 門下省 | 約450 | 立案/報告 約800-2,000 | 審査結果 約600 | 約1,850-3,050 |
| 尚書省 | 約350 | 立案 約800 | 発令 約600 | 約1,750 |
| 六部（各部）| 約350 | 指示 + 背景 約500 | 評価報告 約800 | 約1,650 |
| 御史台 | 約350 | 完全なプロセス記録 約3,000 | 業績審査 約300 | 約3,650 |
| 諫官 | 約300 | 奏折 + ユーザーメッセージ 約1,500 | 諫言 約300 | 約2,100 |
| 早朝官 | 約350 | Notionデータ 約1,000 | ブリーフィング 約600 | 約1,950（セッション開始時のみ） |
| 起居郎 | 約300 | セッションコンテキスト 約1,500 | ジャーナル + wiki抽出 約500 | 約2,300 |
| 翰林院 | 約250 | ユーザーメッセージ 約300 | 会話返答 約500 | 約1,050 |

> 上記は推定値です。実際の消費量はユーザー入力の長さとモデル出力の詳細度によって変動します。

## 利用シナリオ別の内訳

### シナリオA：丞相が直接対応

```
ユーザー → 丞相 → 返答
```

最も軽い利用パターン。丞相が簡単と判断し直接対応。

| | トークン |
|--|---------|
| 呼出回数 | 1（routerエージェント） |
| トークン | 約700-1,000 |
| コスト | 約$0.01-0.02 |

適用：雑談、翻訳、調べもの、記録、単一ステップのタスク。

---

### シナリオA2：快車道分析（🏃）

```
丞相 → 1〜2部 → 奏折総括
```

リクエストが意思決定を伴わない場合（分析、調査、計画のみ）、丞相は完全な三省プロセスをスキップし、1〜2の関連部を直接起動します。

| | トークン |
|--|---------|
| トークン | 約15,000-20,000 |
| コスト | 約$0.35-0.50 |

適用：調査リクエスト、計画タスク、市場分析、情報収集。

---

### シナリオB：簡略プロセス（3部）

```
丞相 → 中書省 → 門下省 → 尚書省 → 3部 → 門下省最終審査 → 奏折 → 御史台 → 諫官
```

典型的なシナリオ：大きな買い物（戸部 + 兵部 + 刑部）

| ステップ | トークン |
|------|---------|
| 丞相 | 約1,000 |
| 中書省 | 約1,700 |
| 門下省立案審査 | 約1,850 |
| 尚書省 | 約1,750 |
| 3部 | 約4,950（3x1,650、並列） |
| 門下省最終審査 | 約2,450 |
| 丞相奏折総括 | 約2,000 |
| 御史台 | 約3,650 |
| 諫官 | 約2,100 |
| **合計** | **約21,000-23,000** |
| **コスト** | **約$0.50-0.60** |

---

### シナリオC：標準プロセス（4部）

```
丞相 → 中書省 → 門下省 → 尚書省 → 4部 → 門下省最終審査 → 奏折 → 御史台 → 諫官
```

典型的なシナリオ：投資判断（戸部 + 兵部 + 刑部 + 吏部）、健康管理（工部 + 兵部 + 戸部 + 刑部）

| | トークン |
|--|---------|
| 合計 | 約25,000-28,000 |
| コスト | 約$0.60-0.75 |

---

### シナリオD：完全プロセス（6部）

```
丞相 → 中書省 → 門下省 → 尚書省 → 6部 → 門下省最終審査 → 奏折 → 御史台 → 諫官
```

典型的なシナリオ：キャリア転換、転居、年間目標、起業判断

| | トークン |
|--|---------|
| 合計 | 約35,000-40,000 |
| コスト | 約$0.80-1.20 |

---

### シナリオE：完全プロセス + 封駁

```
丞相 → 中書省 → 門下省（封駁）→ 中書省（修正）→ 門下省（准奏）→ 尚書省 → 6部 → ...
```

封駁により中書省 + 門下省のサイクルが1回追加される。

| | トークン |
|--|---------|
| 封駁1回あたりの追加消費 | +5,000-6,000 |
| 合計（全6部 + 封駁1回） | 約40,000-46,000 |
| コスト | 約$1.00-1.40 |

封駁は最大2回。封駁2回の場合：

| | トークン |
|--|---------|
| 合計 | 約45,000-52,000 |
| コスト | 約$1.20-1.60 |

---

### シナリオF：完全プロセス + 封駁 + 政事堂

```
... → 6部 → 門下省最終審査（矛盾を発見）→ 政事堂（3ラウンドの議論）→ 奏折 → 御史台 → 諫官
```

政事堂は部の結論が矛盾する場合に発動される。関連する部が3ラウンドの議論を行う。3部が参加すると仮定。

| | トークン |
|--|---------|
| 政事堂の追加消費 | +8,000-10,000 |
| 合計（全6部 + 封駁 + 政事堂） | 約48,000-62,000 |
| コスト | 約$1.50-2.00 |

これは1回のプロセスの理論上の最大消費量です。

---

### シナリオG：早朝レビュー

```
早朝官（独立プロセス、三省六部を経由しない）
```

| | トークン |
|--|---------|
| 合計 | 約1,500-2,500 |
| コスト | 約$0.03-0.05 |

早朝官はセッション開始時の1回の呼出。Notionに接続している場合、データの読み取りにより入力トークンが増加。

---

### シナリオH：翰林院の深い対話

```
丞相（判断）→ 翰林院（マルチターン対話）
```

翰林院はマルチターン対話で、1ターンあたり約1,000-1,500トークン。

| | 3ターン | 5ターン | 10ターン |
|--|---------|---------|----------|
| 合計 | 約4,000-5,000 | 約7,000-9,000 | 約15,000-20,000 |
| コスト | 約$0.08-0.12 | 約$0.15-0.22 | 約$0.35-0.50 |

## トークン節約戦略

| 戦略 | 効果 | 備考 |
|----------|--------|-------|
| **簡単な案件で朝議を開かない** | 90%以上節約 | 丞相が直接対応、約1kトークン |
| **快車道を使用** | 60-75%節約 | 三省をスキップし、1〜2部を直接起動 |
| **クイックモード** | 約1k節約 | 丞相トリアージをスキップし、直接中書省へ |
| **必要に応じて部を起動** | 20-50%節約 | 3部での大きな買い物 vs 全6部は2倍の差 |
| **日常レビューは早朝官を使用** | 80%以上節約 | 1回の呼出 約2k、完全プロセスなし |
| **翰林院のターン数を制限** | 場合による | 追加ターンごとに+1-2k |

## API料金参考（例示、may drift）

> **注記**：以下の料金は 2026-04 時点のスナップショットで、プロバイダの価格改定や新モデルティアのリリースで変動します。最新の料金は必ず各社の公式ページを確認してください。Life OS 自体はモデル非依存です — 具体的にどのモデルを使うかは、お使いのプラットフォームの orchestrator バインディングファイル（例：`CLAUDE.md`）で決まり、本ドキュメントで指定するものではありません。

Claude の sonnet ティアモデル API 料金を例に（2026-04 時点）：

| | 料金 |
|--|-------|
| 入力 | $3 / 100万トークン |
| 出力 | $15 / 100万トークン |

Pro モードは Claude の opus ティアモデルを使用（2026-04 時点）：

| | 料金 |
|--|-------|
| 入力 | $15 / 100万トークン |
| 出力 | $75 / 100万トークン |

> 実際のコストは入力/出力の比率に依存します。Life OSは比較的出力が多く（部の報告 + 奏折）、出力トークンが総コストの60-70%を占めます。
