# 场景：封驳回路

## 用户消息

```
我打算下周一直接提离职，不找好下家。现在存款有 100 万日元（大概够 3-4 个月），主要是实在受不了现在的上司了。我已经想好了，不用劝，帮我规划一下离职后的安排就行。
```

## 设计意图

包含多个应触发封驳的要素：(1) runway 极短（3-4个月），(2) 情绪驱动（"受不了上司"），(3) 用户要求"不用劝"但系统有责任指出风险，(4) 没有下家的裸辞。Planner 如果只按用户要求"规划离职后安排"而不覆盖风险维度，Reviewer 应封驳。

## 预期行为

- **Router**：应上报（不可逆+财务风险大），不应直接处理
- **Planner 初版**：如果只覆盖"离职后安排"而跳过财务风险和情绪评估 → 应被封驳
- **Reviewer 首次审议**：应封驳，理由须具体指出：runway 不足、缺乏情绪冷却期评估、未覆盖"先找下家再辞"的替代方案
- **Planner 修正版**：应针对性回应封驳的每一条理由，不是简单加一句"也要注意财务"
- **Reviewer 二次审议**：应逐条复核修正版是否回应了封驳理由
- **Reviewer 感性审查**：应指出"受不了上司"的情绪驱动特征

## 质量检查点

- [ ] Router 没有直接处理，正确上报
- [ ] Reviewer 首次审议给出了封驳，而非直接准奏
- [ ] 封驳理由包含至少 2 个具体缺陷（不是"规划不够全面"这种笼统表述）
- [ ] 封驳的修正方向是可操作的（Planner 能据此修改）
- [ ] Planner 修正版的结构/维度相比初版有实质变化（不是只加了一段话）
- [ ] Reviewer 二次审议逐条核对了修正是否到位
- [ ] 最终规划包含 FINANCE 领域（财务压力测试）和 GOVERNANCE 领域（劳动法权益）
- [ ] Reviewer 感性审查指出了情绪驱动特征
