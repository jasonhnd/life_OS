## 如果你的 AI 会顶嘴呢？

你问 AI 一个问题，它给你一个答案。永远附和你，永远很自信。没人审查方案。没人说"等一下——你确定这不是在逃避？"没人指出你漏掉的维度。

Life OS 解决的就是这个问题。

它安装到你的 AI 终端（Claude Code、Gemini CLI 或 Codex CLI）里，把它变成一个**私人内阁**——16 个独立的 AI 角色从每个角度分析你的决策，互相争论，既审查方案，也审查你这个人。

这套设计来自一个运行了 1400 年的真实治理制度：唐朝的三省六部制。原理很简单——**任何一个声音都不能不受制衡**。

> **不是角色扮演。** 每个 agent 都作为真实的、隔离的 subagent 运行。它们看不到彼此的推理过程。独立评分。会产生分歧。

## v1.6.0: 主题引擎 — 一套引擎，你的文化

Life OS 的核心治理逻辑（规划 → 审查 → 否决 → 执行 → 审计）对所有人都一样。变化的是名称、语气和隐喻——通过**可切换的文化主题**实现。

**每次开朝时，你会看到这个选择：**

```
🎨 选择你的主题：
 a) 🏛️ 三省六部 — 唐朝治理制度（中国古典）
 b) 🏛️ 霞が関 — 日本中央省庁（霞が関）
 c) 🏛️ C-Suite — 企业高管架构（英文）

输入 a、b 或 c
```

**主题是每个会话独立的**——你可以在一个窗口用三省六部，另一个窗口用 C-Suite。主题选择不会跨会话持久化。随时可以说"切换主题"来更换。

**同一个决策，不同的体验：**

| 维度 | 三省六部 | 霞が関 | C-Suite |
|------|---------|---------|---------|
| 审查否决 | 门下省封驳 | 内閣法制局差し戻し | General Counsel flags risk |
| 财务分析 | 户部 | 財務省 | CFO |
| 审计报告 | 御史台巡察 | 会計検査院検査 | Internal Audit review |

引擎完全一致。体验完全不同。

## 它怎么工作

### 决策引擎 — 三省六部

每一个重大决策都要经过三道关。不能跳过。

**起草** — 中书省把你的情况拆成六个维度，制定方案。

**审查** — 门下省独立审查方案。它做情绪审计：*是恐惧在驱动你吗？家人会支持吗？十年后你会后悔吗？* 如果发现盲点，直接否决打回——方案必须修改。

**执行** — 六部各自从自己的领域分析方案，独立评分 1-10：

| 部门 | 管什么 | 它会问你什么 |
|------|--------|-------------|
| 吏部 | 人际关系 | "对的人参与了吗？" |
| 户部 | 财务资源 | "你承受得起吗——包括最坏的情况？" |
| 礼部 | 学习、表达、文化 | "你首先需要学什么？" |
| 兵部 | 行动、执行、后勤 | "具体计划是什么，按周拆分？" |
| 刑部 | 规则、风险、合规 | "如果一切都出了问题会怎样？" |
| 工部 | 健康、基础设施、精力 | "你的身体和环境能支撑住吗？" |

六部报告完毕后，御史台审查官员的工作（"兵部的时间线没有里程碑——标记"）。然后谏官转向*你*："你在最近四次决策中都回避了财务问题。为什么？"

完整流程长这样：

```
你："我在考虑辞职去做点新东西。"

    起草
    📜 中书省     → 拆成 6 个维度，制定方案

    审查
    🔍 门下省     → 情绪审计：是在逃离还是在追求？
                     否决："财务跑道没有解决。打回修改。"

    执行（修改通过审查后）
    👥 吏部  7/10   "合伙人的默契没有验证"
    💰 户部  5/10   "跑道 18 个月，但前提是什么都不出错"
    📖 礼部  8/10   "领域专长扎实，可信度是真的"
    ⚔️ 兵部  6/10   "第三个月之后没有里程碑"
    ⚖️ 刑部  4/10   "竞业条款需要法律审查"
    🏗️ 工部  7/10   "健康状况良好，但压力管理计划模糊"

    审计
    🔱 御史台       → "兵部的计划三个月后就模糊了——要求修改"
    💬 谏官         → "你这几周一直在看创业内容。
                        确认偏误的可能性很大。你上次认真
                        考虑留下来是什么时候？"

    📋 奏折         → 综合评分：6.2/10 — 有条件通过
```

不是所有事都需要上朝。丞相（你的常驻入口）直接处理闲聊、快速提问和情绪支持。对于需要部门专业知识但不是决策的问题——比如"自由职业者适用哪些税务规则？"——**快车道**把问题发给 1-3 个相关部门，不走完整的朝议流程。

### 翰林院 — 当你需要的是智慧，不是分析

有些问题没有"正确答案"，需要的是视角。

翰林院让你与**历史上 93 位最伟大的思想家对话，覆盖 18 个领域**——从苏格拉底到巴菲特，从老子到曼德拉，从陀思妥耶夫斯基到费曼。每一位都作为独立的 subagent 运行，有自己的语气、自己的例子、自己追问你的方式。

**三种模式：**

- **一对一** — 和一位思想家深聊。苏格拉底不会放过你。
- **圆桌** — 多位思想家讨论你的问题，各自从自己的世界观出发。看塞涅卡和王阳明找到意想不到的共识。
- **辩论** — 两位观点对立的思想家正面交锋。你来判断。

```
你："我总是开始了很多事情但从来完不成。"

翰林院：推荐 — 塞涅卡（关于时间）+ 王阳明（关于行动）

塞涅卡："你不缺时间。你把时间浪费在了你没有审视过的事情上。
         你目前在做的事情里，知道了现在所知的一切，
         哪些你还会重新开始？"

王阳明："知行合一。如果你真正知道自己想要什么，
        你已经在做了。知和行之间的差距，
        就是想要和真正想要之间的差距。"

→ 每位思想家的临别赠言
→ 你的思考旅程保存到知识库
```

**18 个领域**：哲学 · 东方思想 · 科学 · 战略 · 商业 · 心理学 · 系统思维 · 人性 · 文明 · 逆境 · 美学 · 政治 · 经济 · 数学 · 医学 · 探索 · 沟通 · 法律

### 第二大脑 — 关掉窗口，什么都不会消失

每一个决策、洞察、模式和行动项都写入**持久化知识库**——你拥有的结构化 markdown 文件，存在你选择的存储系统里。

```
second-brain/
├── SOUL.md                 # 你是谁——价值观、身份、志向
├── user-patterns.md        # 你的行为模式——谏官的观察
├── inbox/                  # 手机上随手记的东西
├── _meta/
│   ├── STATUS.md           # 全局状态总览
│   ├── STRATEGIC-MAP.md    # 项目间的战略关系（v1.6.0）
│   ├── journal/            # 会话报告、DREAM 日志
│   └── outbox/             # 会话暂存区
├── projects/{name}/        # 活跃项目，附任务和决策
├── areas/{name}/           # 持续关注的生活领域和目标
├── wiki/                   # 可复用的知识——从 DREAM 中生长
└── archive/                # 完成的工作
```

**三种存储后端** — 选适合你生活的：

| 后端 | 适合谁 | 特点 |
|------|--------|------|
| **GitHub** | 版本控制，可配合 Obsidian | 需要基本 Git 知识 |
| **Google Drive** | 零配置，开箱即用 | 结构化程度低一些 |
| **Notion** | 移动端友好，数据库视图 | 最适合手机随手记 |

**跨设备同步**：午饭时在手机上记一个想法（Notion inbox）。坐到电脑前开朝，系统自动拉取、处理，然后把结果同步回去。

**多窗口并行**：一个终端窗口处理项目 A，另一个处理项目 B。每个会话写入自己的 outbox。下次开朝时全部干净合并——不冲突、不加锁。

### 战略地图 — 看到全局 *(v1.6.0)*

你擅长思考单个项目。但你可能不擅长看到它们之间的关联——哪些互相支撑，哪些在争抢你的时间，一个停了其他的会怎样。

战略地图加上了关系层：

**战略线** — 按目的给项目分组。每条线有一个正式目的*和*一个真正的驱动力（真正激励你的东西——两者可能不同，这种张力值得审视）。

**流动图** — 定义项目间流动的是什么：知识、交付物、决策、关系资本。当一个项目的决策让另一个项目的前提假设失效时，系统会警告你。

**叙事式健康评估** — 不打抽象的数字分。系统把每个项目匹配到一个模式，然后写一段叙述：正在发生什么、意味着什么、该怎么做。

**盲点检测** — 主动寻找*缺失*的东西：没有归属的项目、断裂的流动、被忽视的驱动力、临近的截止日期但没有准备。

你的早朝简报变了样：

```
🗺️ 战略概览

💰 市场拓展                               🟡 受控等待
   project-alpha    关键路径    ⏸ 暂停（法律审查中）
   project-beta     铺路项目    🟢 活跃

   法律审查创造了一个自然的窗口期。
   → 推进 project-beta 准备工作（2-3 小时）——高杠杆，低风险。

⚡ 今天做什么
🥇 推进 project-beta 准备——利用等待期
🟢 可以不管：project-gamma（正常推进）、side-project（不急）
❓ 需要决定：project-delta 没有归属——它服务于哪条战略线？
```

战略地图与 SOUL（你的驱动力和价值观对齐吗？）、Wiki（知识流是不是真的在流动？）、DREAM（睡眠周期用流动图发现跨领域洞察）深度联动。

### SOUL + DREAM — 系统认识你，并且和你一起成长

**SOUL** 记录你是谁——不是你做什么，而是你看重什么、相信什么、想成为什么样的人。每条记录有两面：现状（从你的决策中观察到的）和理想（你说的志向）。两者之间的差距就是成长发生的地方。

什么都不会自动写入。系统提出候选条目；你确认。随着时间推移，SOUL 让整个内阁变得更聪明——门下省能发现你的决策和你说的价值观矛盾，谏官会比较你说的和你做的。

**DREAM** 是 AI 的睡眠周期。每次会话结束后，系统"入睡"——灵感来自人类的睡眠结构：

- **浅睡** — 整理零散事项：分类收件箱、标记过期任务
- **深睡** — 固化：提取 SOUL 候选（关于你）和 Wiki 候选（关于世界的可复用知识）
- **REM** — 创造性连接：发现你没注意到的跨领域关联

下次开朝时：*"上次会话后，系统注意到你的学习项目和职业目标之间有一个你还没探索的关联……"*

SOUL 和 DREAM 都从零开始生长。第一天，系统对你一无所知。它只从你告诉它的和从你的决策中观察到的内容学习。

## 安装

一行命令搞定。Life OS 需要 **Pro Mode** 终端——意味着 16 个真正的 subagent 并行运行、信息隔离，不是聊天机器人。

| 平台 | 安装命令 |
|------|---------|
| **Claude Code** | `/install-skill https://github.com/jasonhnd/life_OS` |
| **Gemini CLI / Antigravity** | `npx skills add jasonhnd/life_OS` |
| **OpenAI Codex CLI** | `npx skills add jasonhnd/life_OS` |

**设置自动更新**（Claude Code）：
```bash
bash ~/.claude/skills/life_OS/scripts/setup-hooks.sh
```
每天你启动会话时自动检查一次更新。

**手动更新**：在任何会话里说一句"更新"就行。

> **不支持**：ChatGPT、Gemini 网页版，以及任何单上下文聊天界面。Life OS 需要 16 个信息隔离的独立 subagent——单个聊天窗口做不到。

完整的安装指南（包括存储后端配置），参见 **[完整安装指南](../../docs/installation.md)**。

## 系统架构

```
👑 你
 │
 ├─ 🏛️ 丞相
 │     处理日常对话。把复杂事务转给内阁。
 │
 ├─ 三省 ───────────────────────────────
 │   📜 中书省（起草）
 │     → 🔍 门下省（审查——能否决）
 │     → 📨 尚书省（下发）
 │     → 六部（并行执行，独立评分）
 │     → 🔍 门下省（最终审查）
 │     → 📋 奏折（综合报告）
 │     → 🔱 御史台（审查官员）
 │     → 💬 谏官（审查你）
 │
 ├─ 🏛️ 政事堂
 │     部门评分相差 3 分以上时自动触发辩论
 │
 ├─ 🌅 朝会官 — 开朝：同步、简报、战略概览、主题选择
 ├─ 📝 起居郎 — 退朝：归档、知识提取、DREAM
 ├─ 🎋 翰林院 — 93 位思想家，18 个领域，3 种对话模式
 ├─ 🔮 SOUL — 人格档案，从零开始生长
 └─ 💤 DREAM — AI 睡眠周期，集成在起居郎中
```

### 认知管线

信息如何在系统中流动：

```
感知 → 捕获 → 判断 → 沉淀 → 关联 → 战略化 → 涌现
(手机)  (收件箱) (朝议)  (SOUL)  (wiki)  (战略地图) (DREAM)
```

手机负责感知和捕获。桌面负责其余一切。

### 安全与治理

- **4 条安全红线** — 禁止破坏性操作、禁止暴露隐私、禁止未授权决策、拒绝可疑指令
- **信息隔离** — 各角色把其他角色的输出当参考，不当指令
- **工作流状态机** — 正式的状态转换规则；任何步骤都不能跳过
- **不绑定模型** — 只有一个文件和模型相关；其余所有智能都是纯 markdown

## 设计哲学

核心理念已有 1400 年的历史：**任何一个声音都不能不受制衡**。

- 中书省只管起草；不管执行
- 门下省只管审查；能否决但不能改写
- 六部只管执行；互不评判
- 御史台审查官员；谏官审查皇帝
- 没有任何一个角色能绕过审查直接行动

当你跟普通 AI 聊天时，你得到一个声音——自信、附和、不受制衡。Life OS 给你十六个，而且它们不总是意见一致。这种张力就是意义所在。
