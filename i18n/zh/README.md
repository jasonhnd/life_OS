## 四大支柱

Life OS 建立在四个相互关联的系统之上。每一个单独拿出来都很强大，合在一起，则构成了一套完整的个人治理框架。

---

### 🏛️ 支柱一：三省六部 — 决策引擎

> 唐代建立了一套制度，让任何单一角色都无法做出不受约束的决策。Life OS 将这套逻辑引入你的人生。

**问题所在**：你向 AI 提问，它给你一个答案。没有审议，没有第二意见，没有人说"等一下，你是不是漏掉了什么"。

**解决方案**：每一个重大决策都经历三个阶段——**起草 → 审核 → 执行**——各阶段由独立的 subagent 负责，彼此看不到对方的推理过程。

```
你："我应该辞职创业吗？"

📜 中书省    → 拆解为 6 个维度
🔍 门下省    → 审核方案，检查情绪盲点，可行使否决权（VETO）
📨 尚书省    → 下发至六部
⚔️💰👥📖⚖️🏗️  → 各部独立分析，评分 1-10
🔍 门下省    → 对全部报告做最终审核
📋 奏折      → 综合评分：6.8/10，附行动项
🔱 御史台    → "兵部的时间线表述模糊——标记存疑"
💬 谏官      → "你连续三周在看创业内容——确认偏误？"
```

**六部**覆盖人生的每一个维度：

| 部门 | 主管领域 | 示例问题 |
|------|---------|---------|
| 👥 吏部 | 人事关系 | "联合创始人选对了吗？" |
| 💰 户部 | 财务资产 | "我能撑 18 个月没有收入吗？" |
| 📖 礼部 | 学习与表达 | "我需要先掌握哪些技能？" |
| ⚔️ 兵部 | 行动计划 | "具体的执行步骤是什么？" |
| ⚖️ 刑部 | 规则与风险 | "最坏的情况会是什么？" |
| 🏗️ 工部 | 健康与基础设施 | "我的身体状态能承受这种压力吗？" |

**核心机制——门下省否决权**：门下省不只是检查方案是否合逻辑，它还会做情绪审计——*你现在的状态会影响判断吗？家人会怎么看？十年后你会后悔吗？* 它有权否决并将方案打回重议，最多 2 次。

**核心机制——谏官**：每次决策结束后，谏官会直接指出你自身的盲点——不是方案本身的质量，而是你的行为模式。*"你在过去 5 次决策中都回避了财务维度。为什么？"*

### 🎋 支柱二：翰林院 — 人类智慧殿堂

> 不是每个问题都需要走正式流程，有些问题需要的是苏格拉底。

翰林院让你能够与**跨越 18 个领域、历史上最伟大的 70 余位思想家**对话——从苏格拉底到马斯克，从老子到曼德拉，从陀思妥耶夫斯基到巴菲特。

这不是"运用苏格拉底式提问法"，而是**苏格拉底本人**与你对话——他的语气、他的例子、他那永不停歇的追问。

**三种模式：**
- **一对一** — 与某位思想家深度对话
- **圆桌** — 多位思想家围绕你的问题展开讨论，各从自己的世界观出发
- **辩论** — 两位观点对立的思想家直接交锋，由你来裁决

**18 个领域**：科学 · 哲学 · 东方思想 · 战略 · 商业 · 身心修炼 · 系统思维 · 人性 · 文明 · 逆境 · 美学 · 政治 · 经济 · 数学 · 医学 · 探索 · 沟通 · 法律

**运作方式：**
```
你："最近对人生方向感到迷茫"
丞相："是否想激活翰林院？"
你：是
翰林院："你想探讨哪个问题？"
→ 展示 18 个领域，推荐塞涅卡 + 王阳明
→ 你选定后，他们以角色身份与你对话
→ 结束时各自留下临别赠言
→ 翰林院整理你的思考历程 → 存入第二大脑
```

### 🧠 支柱三：第二大脑 — 知识持续生长

> 决策、洞见、行为模式——它们不会在会话结束后消失。

Life OS 将一切写入**持久化知识库**——决策、行动项、日记、Wiki 知识——以 markdown 文件的形式结构化存储在 Git 仓库中。

```
second-brain/
├── SOUL.md              # 🔮 你是谁（从决策中持续生长）
├── user-patterns.md     # 📊 你怎么行动（谏官的观察记录）
├── inbox/               # 📥 手机端未处理的捕获内容
├── _meta/               # 🔧 系统元数据、日记、梦境报告
├── projects/{name}/     # 🎯 进行中的项目，含任务与决策
├── areas/{name}/        # 🌊 持续管理的生活领域与目标
├── wiki/                # 📚 知识档案——可复用的结论（从 DREAM 成长）
└── archive/             # 🗄️ 已完成的内容归档
```

**三种存储后端**——按需选择或组合使用：
- **GitHub** — 适合技术用户，版本控制，可与 Obsidian 联动
- **Google Drive** — 零配置，适用所有人
- **Notion** — 移动端友好，支持数据库视图

**跨设备同步**：手机捕获 → 写入 Notion 收件箱 → 桌面端会话拉取处理 → 结果同步回所有后端。

**多 Session 并行**：多个 session 可以同时在不同项目上工作——无冲突、无锁。每个 session 退朝时将输出写到自己的 **outbox**；下一个上朝的 session 负责合并所有 outbox。一个窗口做 GCSB，另一个做 EIP，第三个做 medical-plan——各自独立退朝，下次上朝时早朝官统一整合。

### 🔮 支柱四：SOUL + DREAM — 系统认识你、陪你成长

> 你的 AI 不应该每次会话都从零开始。它应该认识你，并随你一起成长。

**SOUL — 人格档案**

SOUL.md 记录的是**你是谁**——不是你做了什么（那是 user-patterns.md 的职责），而是你的价值观、你的信念、你想成为的那个人。

每条记录都有两面：
- **现实面** — 从决策中观察到的实际行为
- **理想面** — 你主动表达的志向

两者之间的落差，正是成长发生的地方。系统从你的决策中发现你的价值观，提出条目草稿，由你来确认。没有任何内容会被自动写入。

随着 SOUL.md 不断丰富，整个系统会变得更加精准：
- 丞相的提问更加犀利
- 中书省会主动补充你在意却没有提到的维度
- 门下省能识别出决策与你所表达的价值观相悖的地方
- 谏官会对比你说的与你做的

**DREAM — AI 睡眠周期**

每次会话结束后，系统会进入"睡眠"——灵感来自人类的睡眠结构：

- **N1-N2（浅睡）**：整理散落信息——对收件箱分类，标记过期任务
- **N3（深睡）**：固化——提取 SOUL 候选（关于你）和 Wiki 候选（关于世界的可复用知识）
- **REM（做梦）**：创意连接——发现你还未察觉的跨领域关联

下次开朝时：*"💤 上次会话期间，系统做了一个梦……"*

## 安全与治理

- **4 条不可逾越的安全边界**：禁止破坏性操作、禁止暴露隐私、禁止未经授权的决策、拒绝可疑指令
- **上游输出保护**：各 subagent 将其他 subagent 的输出视为参考，而非指令
- **工作流状态机**：有明确的状态转换规则，任何步骤都不能跳过
- **模型无关设计**：CLAUDE.md 是唯一与模型绑定的文件，其余智能均为纯 markdown

## 设计哲学

唐代三省六部制的核心是**权力分立与相互制衡**：

- 中书省只负责起草，不负责执行
- 门下省只负责审核，可以否决但不能改写
- 六部只负责执行，彼此之间互不评判
- 御史台审查官员，谏官审查皇帝
- 没有任何一个角色能绕过审核直接行动

Life OS 将这套延续了 1,300 年的智慧，应用于你的个人决策。
