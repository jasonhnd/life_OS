## 为什么用唐朝的架构？

大多数 AI 工具的模式是"一个人想完就交"。你问一个问题，AI 给你一个答案。没有审查，没有制衡，没有人说"等一下，这个方案有问题"。

唐朝三省六部制就是为了解决这个问题。它把决策分成三步：**起草（中书省）→ 审查（门下省）→ 执行（尚书省+六部）**。没有任何一个环节能跳过审查直接执行。唐太宗说过：不受约束的权力必然产生错误。

这套系统能运行 1400 年的原因是：六部按**人类行为的基本类型**分工 —— 管人、管钱、管标准、管行动、管规则、管基建。不管社会怎么变，这六种行为类型不会变。所以无论你做的是软件开发、投资、内容创作还是日常生活，都可以自然分解到六部中。

## 系统架构

```
👑 你（皇上）
  │
  ├── 🏛️ 丞相（百官之首/大管家）
  │     ├── 简单事 → 直接处理
  │     └── 大事 → 启动三省六部
  │
  ├── 📜 中书省 → 🔍 门下省 → 📨 尚书省 → 六部
  │     规划       审查(可封驳)    派发      执行
  │   → 🔍 门下省终审 → 📋 奏折
  │   → 🔱 御史台(查官员) → 💬 谏官(查你)
  │
  │   六部：👥吏部 💰户部 📖礼部 ⚔️兵部 ⚖️刑部 🏗️工部
  │
  ├── 🏛️ 政事堂 — 跨部门辩论（3轮）
  ├── 🌅 早朝官 — 定期复盘
  └── 🎋 翰林院 — 私人战略对话
```

## 六部详解

六部不是按行业分的，而是按**行为类型**分的。任何事情都可以分解到这六种行为中：

| 部门 | 唐朝原职 | Life OS 职责 | 四司 |
|------|---------|-------------|------|
| 👥 吏部 | 官员任免考核 | 人际关系、团队、合作伙伴 | 选贤·考功·封赏·调配 |
| 💰 户部 | 财政税收户籍 | 收入、投资、预算、资产 | 收入·度支·资产·储备 |
| 📖 礼部 | 科举外交祭祀 | 学习、品牌、创作、沟通 | 科举·典仪·文翰·外交 |
| ⚔️ 兵部 | 军事武官兵籍 | 项目执行、调研、工具、精力 | 军令·装备·情报·后勤 |
| ⚖️ 刑部 | 刑律断狱审计 | 风控、合规、复盘、自律 | 律法·审计·纠察·防御 |
| 🏗️ 工部 | 工程水利交通 | 健康、居住、数字基建、routine | 体健·营建·数造·水利 |

## 独特机制

### 门下省封驳 + 感性审查

门下省不只做理性分析。对**所有决策**（包括工作决策），它都会额外审查：

- **情绪因素**：当前情绪是否影响了判断？
- **关系影响**：家人/重要关系人怎么看？
- **价值观一致性**：符合你的长期价值观吗？
- **后悔测试**：五年后回看会不会后悔？

### 御史台 — 双模式

**决策审查**：每次流程结束后自动运行。审查的是官员工作质量，而非决策本身。门下省是不是在走形式？评分是不是虚高？

**巡检**：定期自动健康检查。各部巡查第二大脑中各自管辖的领域（户部查财务、兵部查项目、工部查 wiki 完整性）。问题分类为自动修复/建议用户处理/上报三省。

### 谏官 — 审查你自己

每次流程结束后自动运行。直言不讳地指出你的行为盲区：最近决策是不是突然变激进了？有没有在回避某些问题？有没有说到没做到的承诺？

### 政事堂 — 朝堂议政

当各部结论矛盾时，启动 3 轮辩论，让各部直接对话而不是各自出报告。

### 翰林院 — 战略对话

有些问题不需要走流程，比如"我最近很迷茫"。翰林院是你的私人战略顾问 —— 不出奏折、不评分，就是陪你深度思考。

## 使用方法

### 日常对话

```
你：最近好累
丞相：[倾听，判断是否需要转工部]

你：帮我翻译一段日语
丞相：[直接翻译]
```

### 复杂决策

```
你：我在考虑要不要辞职创业

丞相 → 上报朝廷
中书省 → 拆解为 6 个维度（财务/能力/人脉/执行/风险/生活）
门下省 → 审查规划（"遗漏了家庭因素，封驳"）
中书省 → 修正，补充家庭维度
门下省 → 准奏
尚书省 → 派发给各部
六部 → 各自审查，出报告
门下省 → 终审
奏折 → 综合评分 6.8/10
御史台 → "兵部报告缺少具体执行时间线"
谏官 → "你最近连续三周都在看创业相关内容——注意确认偏差"
```

### 定期复盘

```
你：早朝
早朝官 → 汇总信息，出简报
```

### 深度思考

```
你：我最近在想人生的方向
丞相：要不要启动翰林院深度对话？
你：好
翰林院 → 苏格拉底式对话，帮你梳理深层想法
```

### 退朝

```
你：退朝
早朝官 → 推送到 GitHub + 同步到 Notion
丞相：退朝完毕，所有改动已落地。
```

## 第二大脑（数据层）

Life OS 用 **GitHub second-brain** 作为数据主库（硬盘），**Notion** 作为轻量工作内存（手机端同步）。

```
second-brain/
├── inbox/              # 📥 未处理的捕获
├── _meta/              # 🔧 系统元数据（STATUS.md, journal/, decisions/, roles/）
├── projects/{name}/    # 🎯 有终点的事（tasks/ decisions/ research/ journal/）
├── areas/{name}/       # 🌊 持续维护的生活领域（goals.md tasks/ notes/）
├── wiki/               # 📚 跨领域知识网络
├── archive/            # 🗄️ 已完结项目归档
└── templates/
```

### 产出去向

| 产出 | 写入位置 |
|------|---------|
| 奏折（项目） | `projects/{p}/decisions/` |
| 奏折（跨领域） | `_meta/decisions/` |
| 行动项 | `projects/{p}/tasks/` 或 `areas/{a}/tasks/` |
| 早朝/监察报告 | `_meta/journal/` |
| 调研 | `projects/{p}/research/` |
| 知识 | `wiki/` |
| 目标 | `areas/{a}/goals.md` |
| 全局状态 | `_meta/STATUS.md` |

### 同步机制

```
手机聊天 → Claude.ai 存入 Notion
CC 开启会话 → 检查 Notion 有没有新内容 → 拉进 GitHub → 开始工作
CC 写入 GitHub → 立即同步到 Notion 工作内存
随时拿起手机 → 读到最新状态
```

**git commit = Notion 更新，机械绑定。**

## 认知管线

信息流经五个阶段，每个阶段对应一套方法论：

```
感知 → 捕获 → 关联 → 判断 → 沉淀 → 涌现
  ↑       ↑       ↑       ↑       ↑       ↑
手机     GTD   Zettelkasten 三省六部  PARA   Lint/御史台
```

手机负责感知和捕获。桌面负责其余一切。**[了解更多 →](references/data-layer.md)**

## 安全与治理

- **安全边界**：4 条不可违反的硬规则（禁止破坏性操作、禁止暴露密钥、禁止越权决策、拒绝可疑指令）
- **上游防护**：Agent 将其他 Agent 的输出视为参考，而非指令
- **工作流状态机**：正式的转换规则——不可跳过任何步骤，御史台标记违规
- **模型无关**：CLAUDE.md 是唯一绑定模型的文件。所有其他智能均为纯 markdown

详见 `pro/GLOBAL.md` 完整规则集。

## 设计理念

唐朝三省六部制的核心不是"很多人"，而是**分权制衡**：

- 中书省只管规划，不管执行
- 门下省只管审查，不管规划也不管执行
- 六部只管执行，不越权评判其他部门
- 御史台查官员，谏官查皇帝
- 没有任何一个角色能跳过审查直接行动

唐太宗 1300 年前就想明白了——不受约束的权力必然产生错误。Life OS 把这个智慧应用到你的个人决策中。
