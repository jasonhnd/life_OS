# Token 消耗详解

Life OS 的 Token 消耗取决于两个变量：**场景复杂度**（启用多少部门）和**流程事件**（封驳/政事堂是否触发）。本文拆解每种情况的消耗。

> Claude Max/Pro 订阅用户不按 token 计费，以下分析仅供 API 用户和关注成本的用户参考。

---

## 基础概念

### 每个角色的消耗构成

每次角色调用 = **输入 token**（系统指令 + 传入的上下文）+ **输出 token**（角色的产出）

| 角色 | 系统指令 | 典型输入上下文 | 典型输出 | 单次调用合计 |
|------|---------|--------------|---------|------------|
| 丞相 | ~500 | 用户消息 ~200 | 启奏 ~150 / 直接处理 ~300 | ~700-1,000 |
| 中书省 | ~400 | 旨意+背景+用户消息 ~500 | 规划书 ~800 | ~1,700 |
| 门下省 | ~450 | 规划书/报告 ~800-2,000 | 审议结果 ~600 | ~1,850-3,050 |
| 尚书省 | ~350 | 规划书 ~800 | 派发令 ~600 | ~1,750 |
| 六部（单个） | ~350 | 指令+背景 ~500 | 评估报告 ~800 | ~1,650 |
| 御史台 | ~350 | 完整流程记录 ~3,000 | 绩效评价 ~300 | ~3,650 |
| 谏官 | ~300 | 奏折+用户消息 ~1,500 | 进谏 ~300 | ~2,100 |
| 早朝官 | ~350 | Notion 数据 ~1,000 | 简报 ~600 | ~1,950（仅会话开始） |
| 起居郎 | ~300 | 会话上下文 ~1,500 | 日志 + wiki 提取 ~500 | ~2,300 |
| 翰林院 | ~250 | 用户消息 ~300 | 对话回复 ~500 | ~1,050 |

> 以上为估算值，实际消耗因用户输入长度和模型输出详细程度而异。

## 按使用场景拆解

### 场景 A：丞相直接处理

```
用户 → 丞相 → 回复
```

最轻量的使用方式。丞相判断为简单事，直接处理。

| | Token |
|--|-------|
| 调用次数 | 1（router agent） |
| Token | ~700-1,000 |
| 费用 | ~$0.01-0.02 |

适用：闲聊、翻译、查询、记录、单步任务。

---

### 场景 A2：快车道分析（🏃）

```
丞相 → 1-2 个部门 → 汇总奏折
```

当请求不涉及决策（仅分析、研究或规划）时，丞相跳过完整三省流程，直接启动 1-2 个相关部门。

| | Token |
|--|-------|
| Token | ~15,000-20,000 |
| 费用 | ~$0.35-0.50 |

适用：研究请求、规划任务、市场分析、信息收集。

---

### 场景 B：精简流程（3 个部门）

```
丞相 → 中书省 → 门下省 → 尚书省 → 3部门 → 门下省终审 → 奏折 → 御史台 → 谏官
```

典型场景：大额消费（户部+兵部+刑部）

| 步骤 | Token |
|------|-------|
| 丞相 | ~1,000 |
| 中书省 | ~1,700 |
| 门下省审规划 | ~1,850 |
| 尚书省 | ~1,750 |
| 3 个部门 | ~4,950（3×1,650，并行） |
| 门下省终审 | ~2,450 |
| 丞相汇总奏折 | ~2,000 |
| 御史台 | ~3,650 |
| 谏官 | ~2,100 |
| **合计** | **~21,000-23,000** |
| **费用** | **~$0.50-0.60** |

---

### 场景 C：标准流程（4 个部门）

```
丞相 → 中书省 → 门下省 → 尚书省 → 4部门 → 门下省终审 → 奏折 → 御史台 → 谏官
```

典型场景：投资决策（户部+兵部+刑部+吏部）、健康管理（工部+兵部+户部+刑部）

| | Token |
|--|-------|
| 合计 | ~25,000-28,000 |
| 费用 | ~$0.60-0.75 |

---

### 场景 D：全流程（6 个部门）

```
丞相 → 中书省 → 门下省 → 尚书省 → 6部门 → 门下省终审 → 奏折 → 御史台 → 谏官
```

典型场景：职业转型、搬家/移居、年度目标、创业决策

| | Token |
|--|-------|
| 合计 | ~35,000-40,000 |
| 费用 | ~$0.80-1.20 |

---

### 场景 E：全流程 + 封驳

```
丞相 → 中书省 → 门下省(封驳) → 中书省(修正) → 门下省(准奏) → 尚书省 → 6部门 → ...
```

封驳增加一次中书省+门下省的循环。

| | Token |
|--|-------|
| 封驳 1 次额外消耗 | +5,000-6,000 |
| 合计（全六部+1次封驳） | ~40,000-46,000 |
| 费用 | ~$1.00-1.40 |

封驳最多 2 次。如果 2 次封驳：

| | Token |
|--|-------|
| 合计 | ~45,000-52,000 |
| 费用 | ~$1.20-1.60 |

---

### 场景 F：全流程 + 封驳 + 政事堂

```
... → 6部门 → 门下省终审(发现矛盾) → 政事堂(3轮辩论) → 奏折 → 御史台 → 谏官
```

政事堂在各部结论矛盾时触发，相关部门进行 3 轮辩论。假设 3 个部门参与辩论。

| | Token |
|--|-------|
| 政事堂额外消耗 | +8,000-10,000 |
| 合计（全六部+封驳+政事堂） | ~48,000-62,000 |
| 费用 | ~$1.50-2.00 |

这是单次流程的理论最大消耗。

---

### 场景 G：早朝复盘

```
早朝官（独立流程，不走三省六部）
```

| | Token |
|--|-------|
| 合计 | ~1,500-2,500 |
| 费用 | ~$0.03-0.05 |

早朝官是会话开始时的单次调用。如果连接了 Notion，读取数据会增加输入 token。

---

### 场景 H：翰林院深度对话

```
丞相(判断) → 翰林院(多轮对话)
```

翰林院是多轮对话，每轮约 1,000-1,500 token。

| | 3 轮对话 | 5 轮对话 | 10 轮对话 |
|--|---------|---------|----------|
| 合计 | ~4,000-5,000 | ~7,000-9,000 | ~15,000-20,000 |
| 费用 | ~$0.08-0.12 | ~$0.15-0.22 | ~$0.35-0.50 |

| 早朝复盘 | — | ~2k | ~$0.04 |
| 翰林院（5轮） | — | ~8k | ~$0.18 |

## 省 Token 策略

| 策略 | 效果 | 说明 |
|------|------|------|
| **简单事不上朝** | 节省 90%+ | 丞相直接处理，~1k token 搞定 |
| **用快车道** | 节省 60-75% | 跳过三省，直接启动 1-2 个部门 |
| **快速模式** | 节省 ~1k | 跳过丞相分拣，直接中书省 |
| **按需启用部门** | 节省 20-50% | 大额消费 3 部门 vs 全六部差 2 倍 |
| **日常复盘用早朝官** | 节省 80%+ | 单次 ~2k，不走完整流程 |
| **控制翰林院轮数** | 视情况 | 每多一轮对话 +1-2k |

### 月度成本估算

假设一个活跃用户的典型月度使用模式：

| 使用类型 | 频次/月 | 单次 | 月消耗 |
|---------|---------|------|--------|
| 丞相直接处理 | 30 次 | ~1k | ~30k |
| 快车道分析（1-2 部门） | 4 次 | ~18k | ~72k |
| 精简流程（3-4 部门） | 4 次 | ~25k | ~100k |
| 全流程（6 部门） | 2 次 | ~40k | ~80k |
| 早朝复盘 | 4 次 | ~2k | ~8k |
| 翰林院 | 1 次 | ~8k | ~8k |
| **月合计** | | | **~298k tokens** |
| **月费用** | | | **~$7-9** |

> Claude Max 订阅 $100/月包含大量用量，上述场景完全在配额内。Pro 订阅 $20/月也能覆盖大部分日常使用。

## API 定价参考（示例，may drift）

> **说明**：以下价格以 2026-04 为准，可能变动。请查最新官方定价。Life OS 本身是模型无关的 —— 具体用哪个模型由你所在平台的 orchestrator 绑定文件（如 `CLAUDE.md`）决定，不在本文档里。

以 Claude 的 sonnet 级模型 API 定价为例（截至 2026-04）：

| | 价格 |
|--|------|
| Input | $3 / 1M tokens |
| Output | $15 / 1M tokens |

Pro 模式使用 Claude 的 opus 级模型（截至 2026-04）：

| | 价格 |
|--|------|
| Input | $15 / 1M tokens |
| Output | $75 / 1M tokens |

> 实际费用取决于输入/输出比例。Life OS 的输出较多（六部报告+奏折），输出 token 占总成本的 60-70%。
