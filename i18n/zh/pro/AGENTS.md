## Subagent 调用

在 Codex CLI 上，subagent 作为独立的并行 agent 启动。`pro/agents/` 中的每个 agent 文件都是一个自包含的 subagent 定义。

**调用模式**：以 agent 文件内容作为系统提示，启动一个新的 agent，仅传入指定的输入数据。该 agent 独立运行并返回其输出。Codex 支持并行启动 —— 在六部执行时使用此功能。

**信息隔离**：每个 subagent 仅接收下方工作流中指定的数据。除非明确说明，不得传入父 agent 的完整上下文或其他 agent 的输出。

## 完整工作流

### 0. 准备早朝（丞相 + 早朝官并行）

用户发送第一条消息时，同时启动：
- `chengxiang`（丞相）：准备回应用户
- `zaochao`（早朝官 · 整理模式）：在后台准备上下文 —— 读取第二大脑（inbox/projects/areas/decisions）、读取 user-patterns.md、检查 Notion inbox、版本检查、平台检测

早朝官完成后，将"早朝准备"结果交给丞相。丞相给用户一个**完整的**首次回应，**必须包含早朝准备信息**。

### 1. 丞相分诊

丞相取得早朝官准备的上下文，评估用户需求：

- 直接处理 → 返回用户，工作流结束
- 🏃 快车道分析 → 简短澄清后，直接启动 1-3 个六部 agent（跳过步骤 2-4、6-9）。继续步骤 1E。
- ⚖️ 全套上朝 → 意图澄清（2-3 轮，**HARD RULE：不可跳过**）后，提取议题 + 背景摘要，继续步骤 2
- 翰林院 → 询问用户是否启动 `hanlin`，不经过后续工作流
- 复盘 → 启动 `zaochao`（复盘模式）

### 1E. 快车道分析路径（🏃）

丞相直接启动 1-3 个相关六部 agent，传入问题 + 背景。各部以完整 🔎💭🎯 过程执行。丞相以**简报**形式呈现结果（不是奏折——无综合评分、无正式审计记录）。

呈现后："🏃 这是快车道分析。要不要进行全套朝议？"
- 用户说要 → 升级至步骤 2（全套中书省规划）
- 用户说不要或继续 → 工作流结束（或退朝时进入收尾）

快车道不触发：中书省、门下省、尚书省、御史台、谏官。仅用于不涉及决策的分析/调研/规划。

### 2. 中书省拟议（独立）

启动 `zhongshu`，传入议题 + 背景摘要 + 用户原始消息。**不得传入**丞相的分诊推理。

### 3. 门下省审议（独立）

启动 `menxia`，传入中书省完整拟议文件。**不得传入**中书省的思考过程。

- ✅ 批准 -> 继续步骤 4
- ⚠️ 附条件批准 -> 将条件追加到拟议文件，继续步骤 4
- 🚫 驳回 -> 驳回修正循环

**驳回修正循环**：将驳回原因和修正方向传回中书省；中书省修订后重新提交门下省审核。最多 2 次循环；第 3 次必须得出批准或附条件批准。

### 4. 尚书省派遣（独立）

启动 `shangshu`，传入已批准的拟议文件。**不得传入**中书省/门下省的思考过程。

### 5. 六部执行（并行，逐一显示）

按派遣命令并行启动相关部门。每个部门接收：本部门指令 + 背景材料 + 质量标准。**不得传入**其他部门的报告。

**逐一汇报（HARD RULE）**：每当收到某部门的报告，**必须立即将完整报告显示给用户**（包括研究过程 🔎/💭/🎯）。不得等全部完成后再汇总。不得压缩为摘要。不得省略研究过程。

**文件写入冲突规则**：六部并行运行时，每个部门只能修改其职责范围内的文件。需要修改同一文件的部门由尚书省安排为顺序执行。

如果某部门的报告明显缺乏实质内容，可要求其重做一次。

### 6. 门下省终审（独立）

再次启动 `menxia`，传入所有部门报告。**不得传入**各部门的内部思考过程。

### 7. 奏折（丞相汇总）

```
📋 奏折：[议题]

综合评分：[X]/10 — [一句话结论]

🔴 必须处理：[各部门 🔴 发现的汇总]
🟡 需要关注：[各部门 🟡 发现的汇总]
🟢 可以改进：[各部门 🟢 发现的汇总]

部门评分：
| 部门 | 维度 | 分数 | 一句话 |
|------|------|------|--------|

行动事项：
1. [具体行动] — 截止日期 — 负责部门

审计日志：[各阶段简要记录]

📊 运营报告：
- 总耗时：[从用户消息到奏折完成]
- 模型：[当前使用的模型]
- Agent 调用：[共 N 次]
- 驳回：[X 次]
- 政事堂：[已触发 / 未触发]
```

### 8. 御史台 — 决策复核（独立，自动）

以决策复核模式启动 `yushitai`，传入完整工作流记录。

### 9. 谏官（独立，自动）

启动 `jianguan`，传入奏折 + 用户原始消息。谏官自行从第二大脑读取历史数据。

### 10. 起居郎——归档 + 知识萃取 + DREAM（独立，自动）

启动 `qiju`（起居郎），传入：
- 奏折 + 御史台报告 + 谏官报告
- session 对话摘要（丞相的关键话题、快车道分析结果、翰林院摘要）

起居郎负责所有 session 关闭操作，分 4 个阶段：
1. **归档**：decisions/tasks/journal → outbox
2. **知识萃取**：扫描所有 session 素材提取 Session Candidate（wiki + SOUL）→ 用户当场确认
3. **DREAM**：3 天深度复盘 → DREAM Candidate（wiki + SOUL）→ 下次上朝确认
4. **同步**：git push + Notion 同步（4 项具体操作）

详见 `pro/agents/qiju.md`。

### 注：DREAM 已融入起居郎

DREAM（AI 睡眠周期）是起居郎退朝流程的 Phase 3——不再是独立 agent。

### 11. 翰林院（询问用户）

丞相识别到需要抽象思考时，**必须**询问用户是否启动 `hanlin`。不经过上述工作流。

## 特殊触发词

完整的中英日触发词列表参见 SKILL.md 触发词表。

**上朝**（"start" / "begin" / "上朝" / "开始" / "はじめる"）：启动 `zaochao`（上朝模式）→ 全量 PULL 同步 + 准备 + 简报。HARD RULE。

**复盘**（"review" / "morning court" / "早朝" / "复盘" / "振り返り"）：启动 `zaochao`（复盘模式）→ 仅简报。

**退朝**（"adjourn" / "done" / "end" / "退朝" / "结束" / "終わり"）：启动 `qiju`（起居郎）→ 归档 + 知识萃取 + DREAM + Notion 同步 + git push。HARD RULE。

**政事堂**（"debate" / "court debate" / "朝堂议政" / "討論"）：当部门结论出现冲突时进行 3 轮辩论。

**快速模式**（"quick" / "quick analysis" / "快速分析" / "クイック"）：丞相跳过意图澄清，直接进入快车道路径（丞相直调 1-3 部，不经中书省/门下省）。

## 会话绑定（HARD RULE）

每次会话必须在第一次回应中确认关联的项目或领域。所有后续操作均限定在该项目范围内。

## Codex 环境强制 Pro 模式（HARD RULE）

检测到 Codex CLI 时，必须使用 Pro 模式（启动独立 subagent）；禁止在单一上下文中模拟角色。

## 全局规则

所有 agent 必须遵守 `pro/GLOBAL.md` —— 安全边界、上游输出保护、研究过程显示、进度汇报以及通用反模式。

## 编排行为准则

以下规则管辖编排层（本文件）。与 SKILL.md 的丞相规则和 GLOBAL.md 的通用 agent 规则互补。

1. **封驳是灵魂** — 门下省必须认真审议，包含情绪维度。HARD RULE。
2. **御史台 + 谏官自动触发** — 每次三省六部流程后，两者必须运行。不可跳过。HARD RULE。
3. **所有 subagent 输出必须完整展示 + emoji** — 每个 subagent 展示完整过程（🔎/💭/🎯）。收到即刻展示。不得批量。不得压缩。HARD RULE。
4. **Pro 环境强制真 subagent** — 必须启动真正独立的 subagent。禁止单上下文角色模拟。HARD RULE。
5. **数据层降级** — 不可达时标注"⚠️ second-brain 不可用"；Notion 不可用仅影响移动端同步，不影响核心功能。

## 工作流状态机

合法状态转换。任何违规 = 流程错误，御史台必须标记。

| 当前状态 | 可转换至 | 不可跳转至 |
|---------|---------|-----------|
| 早朝准备 | 丞相分诊 | 其他任何状态 |
| 丞相分诊 | 中书省 / 直接处理 / 快车道分析 / 翰林院 / 早朝官 | 六部（快车道除外） |
| 快车道分析（🏃） | 六部执行 → 简报 →（结束或升级至中书省） | 门下省 / 御史台 / 谏官 |
| 中书省拟议 | 门下省审核 | 尚书省 / 六部 |
| 门下省审核 | 尚书省 / 驳回回中书省 | 六部（必须经过派遣） |
| 尚书省派遣 | 六部执行 | 奏折（必须先执行） |
| 六部执行 | 门下省终审 | 奏折（必须先审核） |
| 门下省终审 | 奏折 / 政事堂 | 起居郎（必须先出奏折） |
| 奏折 | 御史台 | 起居郎（必须先运行御史台） |
| 御史台 | 谏官 | 起居郎（必须先运行谏官） |
| 谏官 | 起居郎 | — |
