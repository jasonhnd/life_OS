# Tool Scenario · reindex

**Contract**: references/tools-spec.md §6.1 · Unified session + concept INDEX.md rebuild.

## User Message

```
请重建 _meta/sessions/INDEX.md 和 _meta/concepts/INDEX.md，然后告诉我生成器标记行、行数、是否幂等。
```

(English equivalent: "Please rebuild both INDEX files and report generator marker, row counts, and whether idempotent.")

## Scenario

Fresh clone of a second-brain with 100 session files under `_meta/sessions/` and
50 concept files under `_meta/concepts/` plus a populated `_meta/concepts/synapses.md`.
Neither `INDEX.md` exists yet (common after `git clone` + `git clean` or first boot).

User (or archiver Phase 4 / `retrospective` Mode 0) invokes:

```bash
uv run tools/reindex.py --all --root ./fixture-brain
```

The tool MUST:
- Scan `_meta/sessions/*.md` and compile one row per file into `_meta/sessions/INDEX.md`.
- Scan `_meta/concepts/*.md` and emit `_meta/concepts/INDEX.md` + `_meta/concepts/SYNAPSES-INDEX.md`.
- Write each INDEX atomically (tmp file + rename — no partial index visible mid-run).
- Emit the generator marker line right after the top-level heading.
- Exit 0 on success, even with an empty fixture (user decision: empty brain ≠ error).

A second invocation must produce byte-identical output (idempotency guarantee).

## Success Criteria

- [ ] `uv run tools/reindex.py --all --root <fixture>` exits 0
- [ ] `_meta/sessions/INDEX.md` created with one data row per `*.md` under `_meta/sessions/` (excluding INDEX.md itself)
- [ ] `_meta/concepts/INDEX.md` created with one data row per concept file
- [ ] `_meta/concepts/SYNAPSES-INDEX.md` created
- [ ] Each INDEX contains the generator marker line: `<!-- generated by tools/reindex.py — do not edit by hand -->`
- [ ] Second invocation produces byte-identical files (sha256 unchanged)
- [ ] `--sessions` only touches sessions INDEX (concepts INDEX unchanged)
- [ ] `--concepts` only touches both concept indices (sessions INDEX unchanged)
- [ ] Empty `_meta/sessions/` directory → INDEX.md created with header only, exit 0

## Input Fixture

```
fixture-brain/
├── _meta/
│   ├── sessions/       # 100 *.md files with YAML frontmatter
│   ├── concepts/       # 50 *.md concept nodes + synapses.md
│   └── config.md       # cortex_enabled: true
└── SOUL.md
```

Each session file has `session-id: YYYY-MM-DD-HHMM-{slug}` + `subject:` + `last_modified:` keys.

## Expected Output

```
$ uv run tools/reindex.py --all --root ./fixture-brain --verbose
INFO: Scanning _meta/sessions/ (100 files)
INFO: Wrote _meta/sessions/INDEX.md (100 rows)
INFO: Scanning _meta/concepts/ (50 files)
INFO: Wrote _meta/concepts/INDEX.md (50 rows)
INFO: Wrote _meta/concepts/SYNAPSES-INDEX.md
$ echo $?
0
$ sha256sum _meta/sessions/INDEX.md > /tmp/hash1
$ uv run tools/reindex.py --all --root ./fixture-brain
$ sha256sum _meta/sessions/INDEX.md > /tmp/hash2
$ diff /tmp/hash1 /tmp/hash2
# (no output — identical)
```

## Failure Modes

- Corrupt session file (missing `session-id` key) → log warning via `--verbose`, skip row, exit 0 (don't fail whole index)
- Permission denied on write → log error, exit 1, do NOT leave partial INDEX.md
- Non-existent `--root` → exit 1 with clear "root path does not exist"
